# 决策记录

## D-001 runtime 作为横切层单独建目录

- 决策：在 `quwoquan_service/specs/runtime/` 维护横切面设计。
- 原因：避免把公共能力散落到各业务服务 spec 中。

## D-002 媒体与同步分别成册

- 决策：`media/` 与 `sync/` 分目录编写。
- 原因：便于独立评审与增量演进。

## D-003 业务归属与底层托管分离

- 决策：用户头像归 `user-service`，群头像归 `chat-service`，底层存储归 runtime。
- 原因：保持 DDD 业务边界清晰。

## D-004 不做端侧群头像拼图兜底

- 决策：服务端预合成失败时客户端显示默认群图标。
- 原因：降低低端机复杂度与不确定性。
