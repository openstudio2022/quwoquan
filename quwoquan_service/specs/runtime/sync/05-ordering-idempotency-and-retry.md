# 排序、幂等与重试

## 1. 排序

同一用户同步流内，`syncSeq` 是唯一排序依据。

要求：

- 单调递增
- 不回退
- 同一 `syncSeq` 不得复用

## 2. 幂等

客户端对已处理过的 `syncSeq` 必须可安全跳过。

服务端对以下动作必须幂等：

- patch 投递
- 用户头像更新广播
- 群头像更新广播

## 3. 重试

### 3.1 服务端

- 事件消费失败可重试
- patch 生成失败可重试
- 群头像异步重算失败可重试

### 3.2 客户端

- 拉取失败可按 cursor 重试
- 实时通知丢失可通过 gap fill 恢复

## 4. 去重

建议使用：

- `eventId`
- `dedupKey`
- `syncSeq`

三层组合保障。

## 5. 禁止项

- 依赖客户端本地时间做主排序
- 通过“最后一条消息时间”替代同步游标
