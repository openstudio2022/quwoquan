# 图片与视频派生策略

## 1. 目标

统一原图、缩略图、封面、转码结果等派生规格，避免各服务自行生成不一致的变体。

## 2. 图片派生

建议至少支持：

- `origin`
- `thumb_64`
- `thumb_128`
- `thumb_256`

适用：

- 用户头像
- 群头像
- 聊天图片
- 内容图片

## 3. 视频派生

建议至少支持：

- `origin`
- `cover_1s`
- `preview_small`

若后续需要更强播放能力，可扩展：

- `h264_720p`
- `h264_1080p`

## 4. 派生生成时机

- 用户上传媒体：在 `complete` 后异步派生
- 群头像：服务端生成时直接输出目标规格

## 5. 派生状态

建议记录：

- `pending`
- `ready`
- `failed`

客户端对未 ready 的派生资源必须有兜底展示。

## 6. 统一策略

- 头像优先 WebP
- 视频封面优先 WebP/JPEG
- 长视频转码必须异步执行

## 7. 不推荐做法

- 由客户端自行猜测缩略图 URL
- 每个业务服务维护自己的派生命名方式
