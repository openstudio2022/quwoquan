# Media 总览

## 1. 目标

为所有媒体类对象提供统一的 runtime 能力，包括：

- 上传会话管理
- 对象存储接入
- CDN 分发
- 统一 URL 与 object key
- 统一资产引用模型

## 2. 媒体对象范围

首批纳入：

- 用户头像
- 群头像
- 聊天图片
- 聊天视频
- 聊天语音
- 聊天文件
- 内容图片
- 内容视频

## 3. 设计原则

- 业务表仅存媒体引用，不存二进制
- 业务归属与底层存储分离
- 对象存储厂商可替换
- URL 统一由 runtime 构建
- 媒体原文件与派生文件统一纳管

## 4. 与现有代码的衔接

当前 `runtime/media` 已具备：

- `MediaCategory`
- `UploadSession`
- `MediaAsset`
- `MediaStore`

后续设计应在现有抽象上扩展，而不是平行再造一套媒体系统。
