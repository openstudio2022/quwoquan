# 趣我圈云侧服务工程目录设计（云侧附录）

> 根目录权威入口：`specs/00_MASTER_DEVELOPMENT_FLOW.md`（含工程目录）  
> 本文件保留云侧实现细节作为附录，避免与根目录权威规范重复维护。

依据 `design.md` 与各服务 `specs/`。**11 个服务**（含业务单体、网关、编排、运营、推荐训练工程与模型服务）；网关与编排为独立服务；静态资源单独存储，CDN 加速。

## 一、架构概览

```
                    端侧 App
                        │
                        ▼
              ┌─────────────────────┐
              │   API Gateway       │  独立服务，路由/认证/限流
              └─────────┬───────────┘
                        │
        ┌───────────────┼───────────────┐
        │               │               │
        ▼               ▼               ▼
┌───────────────┐ ┌───────────────┐ ┌──────────────────┐
│  编排服务      │ │  各业务单体    │ │  Ops（运营）服务  │
│  Orchestrator │ │  直接路由     │ │  运营管理接口      │
└───────┬───────┘ └───────────────┘ └─────────┬────────┘
        │                   ▲               │
        │ 调用              │               │ 调用运营管理接口
        └───────────────────┴───────────────┘

静态资源：业务服务 → 对象存储 → CDN → 端侧（缓存命中快取，未命中回源）

运维/平台能力（非业务服务）：`platform/observability`（可观测性）、`platform/config`（系统配置），统一规范见 `contracts/*`。
```

## 十三、说明

- **网关独立**：不内置在各业务服务中，端侧统一访问 Gateway
- **编排独立**：编排服务位于各单体之上，跨服务流程由编排层完成，业务服务间不直接调用
- **静态资源**：单独存储于对象存储，CDN 加速；端侧访问 CDN，未命中时回源存储
- **接口分层**：各单体有面向 App 的业务接口、面向运营（Ops）的管理接口；运维/平台能力见 `platform/*` 与 `contracts/*`

## 十四、端云一体化工程目录重整（DDD + 元数据 + 契约）

本节用于将 `quwoquan_app` 与 `quwoquan_service` 的工程组织拉齐为同一语义，便于 AI agent 与研发团队以特性粒度协同开发。

### 14.1 统一分层语义（端云同构）

两端统一采用以下语义层：
- `domain`：实体/值对象/聚合/领域规则
- `application`：用例编排、事务边界、领域服务调用
- `adapters`：HTTP/gRPC/MQ/UI 适配层
- `infrastructure`：DB/Cache/MQ/配置/观测实现

### 14.2 推荐目录（端侧）

```text
quwoquan_app/lib/
  features/<feature>/
  cloud/services/<service>/
    domain/
    application/
    data/
      remote/
      mock/
  shared/
    cross_cutting/
      observability/
      governance/
      ops/
```

### 14.3 推荐目录（云侧）

```text
quwoquan_service/services/<service>/
  internal/
    domain/
    application/
    adapters/
      http/
      mq/
    infrastructure/
      persistence/
      cache/
      messaging/
    bootstrap/
```

### 14.4 契约与元数据目录（单一事实来源）

```text
quwoquan_service/contracts/
  openapi/
  metadata/
    entity_catalog.yaml
    field_policy.yaml
    event_catalog.yaml
```

说明：
- OpenAPI 描述接口契约；
- metadata 描述字段策略（分类、日志、观测、运营）；
- 两者共同驱动端云代码与测试生成，减少 hard-coding。

### 14.5 横切能力统一接入（运维线 + 运营线）

- 运维线（platform-ops）：日志、指标、trace、告警、SLO、配置治理、服务治理
- 运营线（product-ops）：行为事件、实验分桶、反馈闭环、自动优化
- 统一入口：`runtime/*` + `platform/*` + `contracts/*`

### 14.6 云厂商接入（阿里云/火山引擎）

统一协议层：
- 可观测：OpenTelemetry（接入阿里云/火山引擎 APM/日志/指标 SaaS）
- 配置中心：`ConfigProvider` 抽象（本地 file/env，云侧 Nacos 或等价托管）
- 告警与通知：统一告警模板，目标通道可按厂商实现切换
