# 系统配置（运维配置）平台模块

本模块定义“运维/系统配置（Platform/Runtime Config）”的落地方式：配置中心、Secrets、环境变量、变更治理与本地测试策略。
运营配置（业务策略/实验/活动）不在此模块，由 `product-ops`（产品运营服务）管理为业务数据。

---

## 1. 配置来源分层（优先级）

建议统一遵循如下优先级（从高到低）：
1. **环境变量**：启动参数、紧急开关（可快速回滚）
2. **Secrets**：凭据与密钥（数据库密码、MQ 访问密钥等）
3. **配置中心**：Nacos/云托管配置中心（可动态下发、可灰度）
4. **配置文件**：alpha/beta 本地开发验证与 gamma 集成测试（`configs/alpha/*.yaml / configs/beta/*.yaml`）
5. **代码默认值**：安全默认（最小权限、保守超时/重试）

> 统一要求：所有服务的配置读取应通过同一套 `ConfigProvider` 抽象，保证 local/test 与 cloud 路径一致。

---

## 2. 典型系统配置范围（示例）

- **网络与调用**：下游超时、重试、熔断、并发上限、连接池
- **资源与性能**：批处理大小、队列并发、缓存 TTL、热点保护
- **网关防护**：限流阈值、黑白名单、风控联动开关
- **可观测性**：采样率、日志级别、导出端点、指标开关

统一命名建议见 `contracts/configuration.md`。

---

## 3. 变更治理（强制）

- **审计**：记录操作者/时间/范围/旧值新值/关联发布号
- **回滚**：支持一键回滚上一版本
- **灰度**：对高风险项（超时/重试/限流/降级/采样）必须灰度

---

## 4. 本地测试策略

- local/test：文件 + 环境变量（零云依赖）
- cloud：配置中心 + Secrets

建议目录：

```
configs/
  local/
    default.yaml
    gateway-service.yaml
    orchestrator-service.yaml
    content-service.yaml
    ...
```

---

## 5. 云厂商配置中心适配（阿里云 / 火山引擎）

通过统一 `ConfigProvider` 抽象屏蔽厂商差异：
- local/test：file + env
- cloud（阿里云）：MSE Nacos + Secrets
- cloud（火山引擎）：托管配置服务 + Secrets（或等价能力）

强制要求：
- 业务配置与系统配置分离（`ops.*` vs `sys.*`）
- 高风险系统配置（超时/重试/限流/降级/采样）必须灰度与可回滚
- 配置变更可审计（操作者/时间/范围/旧值/新值）
