# 趣我圈云侧技术选型（云侧附录）

> 根目录权威入口：`specs/runtime_framework_spec.md`（含技术选型）  
> 本文件保留云侧选型细节作为附录，避免与根目录权威规范重复维护。

目标：弹性、高并发低时延；可部署于阿里云或火山引擎；**优先使用云厂商托管 SaaS**，减少自建运维。

**测试与部署流程**：本地完成单元/集成测试 → 验证通过后部署到云侧 → 对接云 SaaS。

---

## 1. 数据库

### 选型原则：**MongoDB 为主、PostgreSQL 为辅，最终一致性优先**

- **主数据存储**：MongoDB 承担绝大部分业务数据，支持弹性扩展、文档模型、schema 灵活演进。
- **关系型补充**：仅在必须强一致、唯一约束、跨文档事务时使用 PostgreSQL，尽量减少关系型依赖。
- **一致性策略**：业务主体采用最终一致性；身份、认证、状态机等核心场景采用强一致性。

### 数据划分

| 存储 | 适用领域 | 说明 |
|------|----------|------|
| **MongoDB（主）** | Content、Circle、Chat、Assistant 学习、Ops 事件、用户画像/偏好 | 文档模型，最终一致性，适合高并发、灵活 schema |
| **PostgreSQL（辅）** | 用户身份、认证、profileUpdateProposal 状态、全局约束 | 强一致性、唯一约束、MVCC 并发 |
| **Redis** | 缓存、会话、热点数据 | KV，托管即可 |

### 为何关系型选 PostgreSQL

- JSONB 支持：可与 MongoDB 互补，必要时在关系层存半结构化数据。
- 开源、标准遵从好，无厂商归属顾虑。
- MVCC 并发模型适合身份、状态机等高并发更新场景。
- 阿里云、火山引擎均有托管（PolarDB for PostgreSQL / veDB PostgreSQL）。

### 云托管产品

| 能力 | 阿里云 | 火山引擎 |
|------|--------|----------|
| 文档型（主库） | 云数据库 MongoDB | veDB MongoDB |
| 关系型（辅库） | PolarDB for PostgreSQL / RDS PostgreSQL | veDB PostgreSQL |
| 缓存 KV | Tair（Redis 兼容） | 云数据库 Redis |

### 建议

- **MongoDB**：Content、Circle、Chat、Assistant 学习、Ops 埋点、用户画像等，云 MongoDB 托管。
- **PostgreSQL**：User 身份、Auth、profileUpdateProposal 流转，云 PostgreSQL 托管。
- **Redis**：会话、热点缓存，云 Redis 托管。

## 2. 消息队列

### 场景划分

| 场景 | 特点 | 推荐 | 说明 |
|------|------|------|------|
| **业务解耦、异步任务**（反馈上报、profile 回流、通知） | 可靠投递、顺序、重试 | **RocketMQ 云版** | 阿里云、火山引擎均有托管 |
| **大数据流、日志、埋点流** | 高吞吐、流式 | **Kafka 云版** | 云消息队列 Kafka 版 |
| **轻量、简单队列** | 低复杂度 | **RocketMQ** 或云厂商队列 | 够用即可 |

### 建议

- **主选**：**云消息队列 RocketMQ**（阿里云 / 火山引擎）
  - 低延迟、事务消息、顺序消息
  - 适合：反馈上报、profile 回流、行为/埋点解耦
- **备选**：若需要与大数据 / 流计算打通，增加 **云消息队列 Kafka 版**，用于埋点流、实时 ETL。

## 3. 服务通信框架（网关 + 内部协议）

### 网关职责（已确定）

- 统一入口、HTTPS 终端、路由、认证、限流

### 内部通信：**HTTP + gRPC 混合**

| 调用方 | 被调方 | 协议建议 | 理由 |
|--------|--------|----------|------|
| 网关 → 编排/业务服务 | 编排层、各单体 | **HTTP/1.1** | 简单、易调试、多语言通用 |
| 编排层 → 业务服务 | Content/Circle/User/Chat/Ops/Assistant | **gRPC**（首选）或 **HTTP** | 内部调用、性能优先 |
| Ops → 各业务服务 | 管理接口 | **HTTP** | 管理类、调用频次低 |

### 方案 A（推荐）：**内部以 gRPC 为主**

```
端侧 HTTPS → 网关（HTTPS 终端）→ HTTP 转 gRPC（或直连编排）→ 编排/业务服务 gRPC
```

- **编排 → 业务服务**：gRPC，低延迟、强类型、流式可选
- **网关 → 编排**：HTTP 或 gRPC，取决于网关能力
- **Ops → 业务**：HTTP REST，便于工具、脚本

### 方案 B：**全 HTTP**

- 编排与业务服务之间全部 HTTP REST
- 实现简单，跨语言友好，性能略逊于 gRPC

### 建议

- **编排 ↔ 业务服务**：**gRPC**，符合 design.md 中「内部可辅以 gRPC」。
- **网关**：选用支持 gRPC 转发的云网关（如 MSE 微服务网关、ALB + gRPC）或自建 Kong/APISIX，网关做 HTTP→gRPC 转换。
- **运维/管理接口**：保持 HTTP REST。

## 4. 配置中心（运维/系统配置，区别于运营配置）

### 需求（运维/系统配置）

- 系统级配置（运维配置）：DB/MQ/Redis 连接、服务发现、超时/重试/熔断、限流阈值、可观测性采样与导出端点等
- 明确区分：运营配置（实验、策略、活动、治理规则等业务数据），由 `product-ops`（产品运营服务）管理

### 选型

| 方案 | 说明 | 适用 |
|------|------|------|
| **MSE Nacos**（阿里云） | 托管 Nacos，配置 + 服务发现 | 阿里云部署 |
| **火山引擎对应配置中心** | 若提供 Nacos/Config 托管 | 火山引擎部署 |
| **Apollo** | 自建，需维护 | 多云/混合云、已有 Apollo |
| **云原生 ConfigMap +  Secrets** | K8s 内置 | 纯 K8s、配置简单 |

### 云侧建议

- **阿里云**：**MSE Nacos** 作为配置中心，托管、免运维，与 MSE 服务发现一致。
- **火山引擎**：优先使用其托管配置中心（如有 Nacos 或等价服务）。
- **多云/混合云**：可选 **Apollo**，但需自建运维；若上 K8s，可先用 ConfigMap/Secrets + 外部配置服务。

### 本地测试策略（配置抽象）

配置来源**抽象为统一接口**，按环境切换实现，保证同一代码路径在本地与云侧均可运行：

| 环境 | 配置来源 | 说明 |
|------|----------|------|
| **alpha / beta** | 文件 + 环境变量 | `configs/alpha/*.yaml / configs/beta/*.yaml`、`.env`，无 Nacos 依赖 |
| **cloud** | MSE Nacos 等 | 云部署时连接托管 Nacos |

**实现要点**：

1. **ConfigProvider 抽象**：`FileConfigProvider`（本地）与 `NacosConfigProvider`（云侧）实现同一接口。
2. **环境变量驱动**：`CONFIG_SOURCE=file` 或 `APP_ENV=alpha` 时使用文件配置，否则走 Nacos。
3. **目录约定**：`configs/alpha/ 与 configs/beta/` 存放本地 YAML，各服务共享或按服务拆分。
4. **单元测试**：注入内存 ConfigProvider 或直接读测试用 YAML，零外部依赖。

更多统一规范与边界约束见：`contracts/configuration.md`。

```
configs/
├── local/                    # alpha/beta 本地开发验证与 gamma 集成测试
│   ├── default.yaml         # 通用默认
│   ├── content-service.yaml
│   └── user-service.yaml
└── ...
```

## 5. 云厂商 SaaS 优先原则

### 直接用云托管、避免自建

| 组件 | 阿里云 | 火山引擎 | 说明 |
|------|--------|----------|------|
| 文档型数据库（主） | 云数据库 MongoDB | veDB MongoDB | 托管，弹性、分片 |
| 关系型数据库（辅） | PolarDB for PostgreSQL / RDS PostgreSQL | veDB PostgreSQL | 托管，身份/约束/状态机 |
| 缓存 | Tair / Redis | 云 Redis | 托管 |
| 消息队列 | RocketMQ / Kafka 云版 | BMQ / Kafka | 全托管 |
| 配置中心 | MSE Nacos | 对应配置服务 | 托管 |
| 对象存储 | OSS | TOS | 托管 |
| CDN | 阿里云 CDN | 火山 CDN | 托管 |
| 网关 | MSE 微服务网关 / ALB | CLB / 应用型负载均衡 | 按需选用 |
| 服务网格/注册中心 | MSE | 对应 MSE | 若有需要 |

### 避免自建

- 不自建 Nacos/Apollo 集群（优先 MSE Nacos 等托管）
- 不自建 Kafka/RocketMQ 集群（用云消息队列）
- 不自建 Redis 集群（用云 Redis）
- 不自建 MongoDB / PostgreSQL（用云托管）

## 7. 与 design.md 的衔接

- **接口风格**：对外 REST，内部 gRPC，与 design.md 一致。
- **语言栈**：Go（Content/Circle/User/Chat/Ops）、Python（Assistant），上述组件均有 SDK 支持。
- **编排层**：通过 gRPC 调用各业务服务，通过消息队列做异步解耦（反馈、profile 回流等）。
- **数据库策略**：MongoDB 为主、PostgreSQL 为辅，最终一致性优先；业务主体在 MongoDB，身份/认证/状态机在 PostgreSQL。

## 8. 本地开发与测试（云侧部署前验证）

### 原则

- **本地先测、云后部署**：单元/集成测试在本机完成，通过后再部署到云侧对接 SaaS。
- **环境隔离**：local 使用本地/Docker 资源，不依赖云账号；cloud 使用云托管服务。
- **配置/依赖可切换**：通过环境变量或配置选择数据源、配置源、MQ 等。

### 各组件本地方案

| 组件 | 本地（单元/集成测试） | 云侧 |
|------|------------------------|------|
| **配置中心** | 文件（YAML/env），见 §4 配置抽象 | MSE Nacos |
| **MongoDB** | Docker 或 Docker Compose | 云 MongoDB |
| **PostgreSQL** | Docker 或 Docker Compose | 云 PostgreSQL |
| **Redis** | Docker 或本地 Redis | 云 Redis |
| **消息队列** | Docker RocketMQ，或集成测试时 Mock/内存队列 | 云 RocketMQ |
| **对象存储** | 本地目录/MinIO Docker | OSS/TOS |

### Docker Compose 示例（集成测试）

```yaml
# deployments/docker-compose.local.yaml（示例）
services:
  mongodb:
    image: mongo:7
    ports: ["27017:27017"]
  postgres:
    image: postgres:16
    environment: { POSTGRES_DB: quwoquan, POSTGRES_USER: test, POSTGRES_PASSWORD: test }
    ports: ["5432:5432"]
  redis:
    image: redis:7
    ports: ["6379:6379"]
  # 可选：RocketMQ、MinIO 等
```

### 测试分层

| 层级 | 依赖 | 说明 |
|------|------|------|
| **单元测试** | Mock 全部外部依赖（DB、MQ、Config） | 快速、无 Docker |
| **集成测试** | Docker Compose + 文件配置 | 真实 DB/MQ，本地配置 |
| **E2E / 预发** | 云侧环境 + 云 SaaS | 部署后验证 |

### 环境变量约定

| 变量 | local | cloud |
|------|-------|-------|
| `APP_ENV` | `local` | `dev` / `staging` / `prod` |
| `CONFIG_SOURCE` | `file` | `nacos` |
| `MONGODB_URI` | `mongodb://localhost:27017` | 云连接串 |
| `POSTGRES_DSN` | `postgres://test:test@localhost:5432/quwoquan` | 云连接串 |
