## Context

当前图片浏览器与沉浸式媒体查看器已存在基础工具栏与交互，但顶部间距、工具栏布局、图文排版与纯图模式不一致，且与参考视觉不匹配。核心组件集中在 `lib/components/image_viewer.dart` 与 `lib/components/immersive_media_viewer.dart`，需要在不破坏现有交互与状态管理的前提下重构布局与显示逻辑，并遵循语义 token 约束。

## Goals / Non-Goals

**Goals:**
- 统一顶部/底部工具栏布局与间距，并符合参考视觉（返回、位置指示、作者信息、关注、更多等）。
- 图文排版适配横竖屏：图片上下留白尽量对称，标题/配文与底部工具栏对齐。
- 提供“纯图模式”切换与低干扰状态栏呈现策略。
- 保持现有互动能力（点赞、收藏、评论、分享、小趣入口）与数据来源不变。

**Non-Goals:**
- 不修改业务数据结构或新增服务接口。
- 不引入新的第三方依赖（除非现有 `BackdropFilter` 无法满足需求）。
- 不调整评论系统、更多弹窗的内容逻辑。

## Decisions

1. **以 `ImmersiveMediaViewer` 为主入口统一布局策略**  
   - 选择：将顶部/底部工具栏与图文区域的布局能力集中在 `ImmersiveMediaViewer`，图片-only 的 `ImageViewer` 复用相同布局组件或抽取共用 widget。  
   - 备选：分别在两个组件内各自实现。  
   - 理由：减少双实现导致的视觉偏差与维护成本。

2. **图文区域使用“可测量内容高度”的布局**  
   - 选择：通过 `LayoutBuilder` + 文本测量（或 `TextPainter`）计算标题/配文占用高度，再给图片区域分配剩余高度，保证上下留白对称。  
   - 备选：简单 Column + Expanded，依赖自然布局。  
   - 理由：需要保证图片上下留白可控，并支持横竖屏切换。

3. **无留白时使用半透明毛玻璃遮罩**  
   - 选择：当图片区域无法提供留白时，对标题/配文所在区域叠加 `BackdropFilter` + 半透明背景。  
   - 备选：直接遮挡图片或强制缩放图片。  
   - 理由：保持图片主体完整性，且符合参考视觉。

4. **纯图模式下状态栏策略**  
   - 选择：进入纯图模式时优先隐藏系统状态栏；若平台或安全区域限制不可隐藏，则使用深色系统样式（`SystemUiOverlayStyle.dark`）降低干扰。  
   - 备选：始终显示状态栏或固定半透明遮罩。  
   - 理由：符合“沉浸式”诉求，兼容不同设备与系统行为。

## Risks / Trade-offs

- **布局计算复杂** → 通过统一的布局函数与测试样例验证横竖屏与多行文本的边界情况。
- **纯图模式切换时闪烁** → 控制动画时序，使用 `AnimatedOpacity`/`AnimatedSlide` 平滑隐藏/显示。
- **毛玻璃性能开销** → 仅在无留白条件下启用，限制遮罩高度。
