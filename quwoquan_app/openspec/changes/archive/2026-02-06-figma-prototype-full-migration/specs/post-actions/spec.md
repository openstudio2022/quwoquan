# post-actions

## 新增需求

### 需求：帖子支持点赞、收藏、分享与更多

每条帖子卡片与沉浸式查看器须提供操作：点赞、收藏、分享、更多。点击更多须打开操作面板（PostActionSheet），选项与 Figma/设计一致。

#### 场景：点赞切换

- **当** 用户在帖子上点击点赞
- **则** 点赞状态切换，点赞数在 UI 中更新（乐观更新或请求后更新）

#### 场景：收藏切换

- **当** 用户在帖子上点击收藏
- **则** 收藏状态切换，UI 反映新状态

#### 场景：更多打开操作面板

- **当** 用户点击「更多」操作
- **则** 打开操作面板，展示 Figma 规定的选项（如复制链接、举报、不感兴趣）

### 需求：操作面板与 Figma 一致

帖子操作面板须为底部抽屉（或弹窗），包含操作列表与正确样式。点击某项须执行对应操作并关闭面板；点击外部须关闭面板。

#### 场景：操作面板行为

- **当** 操作面板打开时
- **则** 用户可点击某项执行，或点击外部关闭

### 需求：卡片与查看器状态一致

点赞与收藏状态须一致：若用户在查看器中点赞某帖，同一帖在 feed 卡片上须显示已点赞，反之亦然（共享状态或回调同步）。

#### 场景：状态同步

- **当** 用户在沉浸式查看器中切换点赞或收藏
- **则** 返回 feed 后，该帖的卡片展示更新后的点赞/收藏状态

### 需求：帖子操作使用语义 token

所有操作按钮、面板与文案须使用 AppColors、AppSpacing、UITextConstants。禁止使用硬编码值。

#### 场景：帖子操作的 token

- **当** 帖子操作与操作面板被构建时
- **则** 所有样式与文案使用设计系统常量
