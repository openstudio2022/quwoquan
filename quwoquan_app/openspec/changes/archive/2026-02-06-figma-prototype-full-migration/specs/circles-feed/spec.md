# circles-feed

## 新增需求

### 需求：圈子频道与设计文档一致

「圈子」频道须与 InformationArchitecture v3.1 及 CIRCLE_DESIGN v2.0 一致：按**兴趣维度**展示内容或圈子入口；支持二级圈子架构与多分身切换。提供创建圈子入口，点击圈子可进入圈子主页（含创作/讨论/生活/聊天子频道）。

#### 场景：圈子频道内容可见

- **当** 用户处于「圈子」频道
- **则** 展示兴趣分类下的发现视图或圈子列表；布局与交互与原型一致

#### 场景：创建圈子与进入圈子

- **当** 用户点击创建圈子入口
- **则** 打开创建圈子向导（CreateCircleWizard）
- **当** 用户点击某个圈子
- **则** 打开该圈子的圈子主页叠加层

### 需求：圈子频道使用语义 token

圈子频道所有 UI 须使用设计系统常量。禁止使用硬编码值。

#### 场景：圈子频道的 token

- **当** 圈子频道被构建时
- **则** 颜色、间距与文案仅使用语义 token
