## Context

当前编辑器已有“滤镜”入口，但分类、模板与强度链路不完整，难以支撑普通用户“一眼选风格、一步出片”的核心诉求。  
本次目标是构建“消费者优先”的滤镜体系：高辨识分类、可预期模板、低学习成本交互、预览与导出一致。

约束：
- 需复用现有图片编辑器三段式操作面板，不新增独立页面。
- 全链路使用语义 token（`AppColors`、`AppSpacing`、`UITextConstants`）并适配纯黑背景。
- 支持多图编辑场景，滤镜参数对当前图片独立生效，不污染其它图片。
- 首版仅从本地资源加载滤镜配置；需预留启动后定期检查云端更新的位点（本次不实现云端拉取逻辑）。

## Goals / Non-Goals

**Goals:**
- 提供 20 个消费者友好分类（推荐 + 质感、美颜、人像、蓝调、美食、复古、胶片、自然、风景、梦幻、油画、电影、清新、黑白、春夏秋冬）及每类 3~5 个模板。
- 推荐分类默认聚合“最近使用模板”，并支持追加本地预置推荐模板。
- 交付统一面板结构：顶部分类工具栏（含“去滤镜”）+ 中部模板列表（上图下名）+ 底部强度滑条。
- 实现分类与模板列表双向联动：点分类可定位；滑模板可反向更新分类选中态。
- 形成可扩展的滤镜参数模型：模板参数 + 强度插值 + 预览/导出同源渲染。
- 配置模型可扩展到云端动态下发，但首版仅实现本地配置加载与本地版本化存储。
- 模板列表在滑动时对可视项异步生成“基于当前图片”的效果缩略图，计算中展示 loading。
- 对齐业界（剪映/小红书/Snapseed）在“分类可理解性”和“模板可复用性”上的最佳实践。

**Non-Goals:**
- 本次不实现 LUT 文件导入、用户自定义模板发布、云端风格市场（Marketplace）。
- 本次不实现专业分区的 HSL/曲线级滤镜编辑（仍由专业修图承接）。
- 本次不做跨端算法完全一致性（先保证 Flutter 端预览与导出一致）。

## Decisions

### 1) 分类组织采用“用户意图优先”而非“参数术语优先”

- 决策：使用消费者能直接理解的分类词（如“美食/风景/复古/电影”），不以“色相/曲线/胶片响应曲线”等专业术语作为一级入口。
- 原因：社交与内容平台核心链路是“快速选择风格”，用户先按场景与情绪筛选，再看模板细节。
- 备选方案：按参数维度分组（亮度组、色彩组、纹理组）。
  - 不选原因：学习成本高，不符合普通用户决策路径。

### 2) 模板组织采用“分类段落 + 横向模板流”

- 决策：中部采用单一横向模板列表，按分类分段；每段包含该分类 3~5 个模板。
- 原因：与剪映/小红书常见交互一致，减少跳转；支持滑动浏览时平滑切换分类高亮。
- 备选方案：顶部 tab + 每个 tab 独立列表容器。
  - 不选原因：切换成本高，且双向联动复杂（需要多滚动容器同步）。

### 2.1) 推荐分类采用“最近使用优先 + 运营补位”

- 决策：`推荐` 分类由两段数据合并生成：最近使用模板（按时间倒序，去重）优先，其次补齐本地预置推荐模板到目标数量。
- 原因：最近使用是用户真实偏好，运营推荐可保证新风格曝光，二者组合兼顾命中率与探索性。
- 备选方案：仅展示运营推荐。
  - 不选原因：个性化弱，用户复用成本高。

### 3) 强度模型采用“模板目标参数 × 统一强度插值”

- 决策：每个模板定义目标参数向量（亮度、对比、饱和、色温、色调、高光、阴影、褪色、颗粒、晕影、锐化等），最终效果为 `mix(Identity, TemplateParams, strength)`。
- 原因：便于统一滑条、可解释、可回放，且保证 0 强度即原图。
- 备选方案：每个模板单独编写不同强度曲线。
  - 不选原因：维护成本高，调参不可控，难以规模化扩展。

### 4) “去滤镜”作为顶部首位显式操作

- 决策：分类栏左侧固定“去滤镜”按钮，一键恢复到 identity 模板（强度重置为默认值，通常 100 对 identity 无影响）。
- 原因：普通用户最常见撤销诉求是“整组取消滤镜”，不应要求先找到原分类再回退。
- 备选方案：仅依赖全局撤销。
  - 不选原因：路径更长，且会误撤销其它工具操作。

### 5) 双向联动采用“段锚点映射 + 滚动观察”

- 决策：建立 `categoryId -> firstTemplateIndex` 的锚点映射；点击分类滚动到首模板；模板列表滚动时根据可视主模板索引反查当前分类。
- 原因：实现简单稳定，适合移动端高频滑动场景。
- 备选方案：根据像素偏移做连续插值高亮。
  - 不选原因：调试成本高，收益有限。

### 5.1) 配置加载采用“本地资源优先 + 本地持久化 + 云端更新预留”

- 决策：滤镜配置模型抽象为可序列化结构（分类、模板、参数向量、默认强度、排序、启用标记），首版仅从本地资源读取；应用启动后保留“定期检查云端更新并写回本地”的接口位点与调度占位，但不实现网络获取。
- 原因：先保证首版稳定与可控上线，同时不阻塞后续动态运营演进。
- 备选方案：首版直接接入云端拉取。
  - 不选原因：联调与容灾复杂度高，不符合当前迭代节奏。

### 5.2) 缩略图渲染采用“可视项任务队列 + 取消机制”

- 决策：仅对当前可视模板卡片发起缩略图计算任务，滚出可视区时可取消任务；计算完成写入缓存并刷新卡片。计算期间卡片显示 loading（转圈）。
- 原因：避免一次性全量计算造成卡顿，确保滑动流畅。
- 备选方案：进入面板即全量计算全部模板缩略图。
  - 不选原因：首屏等待长、内存与 CPU 峰值高。

### 6) 业界能力拆解与本产品落地策略

- 社交平台（小红书、Instagram 类）：强调“氛围词 + 人像友好 + 一键可用”  
  → 落地：分类名可感知、模板命名短词、默认强度可直接发布。
- 内容制作平台（剪映、CapCut 类）：强调“风格丰富 + 模板预览清晰 + 强度细调”  
  → 落地：每类 3~5 个模板、上图下名字卡片、底部统一滑条。
- 专业制作平台（Snapseed/Lightroom 类）：强调“可控参数 + 可重复结果”  
  → 落地：模板参数向量化、强度插值标准化、预览导出同源。

### 7) 分类与模板清单（每类 3~5）

- `推荐`：最近使用聚合 + 本地预置推荐补齐（如：最近A、最近B、本周精选、节气推荐）
- `质感`：清透、柔雾、立体、微锐
- `美颜`：柔肤、净颜、通透肌、奶油肌
- `人像`：人像柔光、人像冷白、人像胶片、人像电影
- `蓝调`：浅蓝、深海、暮蓝、冰川蓝
- `美食`：鲜味、暖食、炭烤、甜点高光
- `复古`：旧时光、褪色色、港风、棕调复古
- `胶片`：胶片N、胶片W、柯达暖、富士绿、颗粒黑边
- `自然`：原生、轻校正、自然高光、空气感
- `风景`：山野、海岸、日落、天空增强
- `梦幻`：朦胧、粉雾、青紫梦境、柔焦高光
- `油画`：油彩、厚涂、画布纹、复古油画
- `电影`：青橙、低饱和电影、暗黑悬疑、暖调剧情
- `清新`：薄荷、晨光、白净、浅绿
- `黑白`：经典黑白、高对比黑白、银盐、哑黑白
- `春天`：新芽、樱粉、晴春、花田
- `夏天`：海风、汽水蓝、热浪、晴夏
- `秋天`：金叶、琥珀、雾秋、暖棕
- `冬天`：冷霜、雪景、冷灰、冬日清晨

> 实现策略：首版每类先上线 3 个模板（含推荐共 60 个），其余模板作为第二批开关放量，保证性能与调参质量。

## Risks / Trade-offs

- [Risk] 分类数较多导致首屏认知负担偏高  
  → Mitigation：默认优先展示“热门/最近使用”在最前，并保留完整分类入口。

- [Risk] 模板总量增加后滚动性能下降  
  → Mitigation：卡片懒加载、缩略图缓存、参数计算在可见项触发。

- [Risk] 后续接入云端时配置兼容性不足  
  → Mitigation：首版即定义配置 schema 与版本字段，并在本地加载流程执行校验。

- [Risk] 缩略图异步计算导致列表闪烁或错位更新  
  → Mitigation：任务绑定模板唯一 id + 可见性校验 + 主线程最小刷新粒度。

- [Risk] 不同设备上滤镜观感差异（屏幕色域）  
  → Mitigation：参数先校准到 sRGB 目标，重点模板做真机抽样验收。

- [Risk] 双向联动产生“分类抖动”  
  → Mitigation：只在主可视模板跨分类阈值时更新高亮，并做节流。

- [Risk] 模板数量大，命名与维护成本高  
  → Mitigation：采用配置驱动（id/分类/名称/参数/默认强度/排序）与批量校验脚本。

## Migration Plan

1. 定义滤镜分类与模板数据结构（含推荐分类、分类锚点、模板参数向量、默认强度）。  
2. 抽象配置协议并实现本地资源加载、校验、缓存与版本管理。  
3. 预留“启动后定期检查云端更新并回写本地”接口与任务调度占位（不实现云端请求）。  
4. 重构滤镜面板为三段式：顶部分类栏（含去滤镜）、中部模板卡片流、底部强度滑条。  
5. 接入双向联动：分类点击定位 + 列表滚动反向选中分类。  
6. 接入“可视项异步缩略图”渲染链路（loading、缓存、取消任务）。  
7. 接入滤镜效果链路：预览层参数生效与导出层同源实现。  
8. 完成体验校准：黑底对比、卡片文案可读性、强度手感。  
9. 回归测试：分类定位、滑动联动、去滤镜、推荐最近使用、缩略图 loading、保存一致性。  

回滚策略：保留旧滤镜面板入口开关，若出现高风险回归可快速切回占位版。
