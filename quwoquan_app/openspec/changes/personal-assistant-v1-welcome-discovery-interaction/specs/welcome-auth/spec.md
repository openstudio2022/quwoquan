## MODIFIED Requirements

### Requirement: 首次进入时展示欢迎页

应用须在用户打开应用且尚未进入主框架时展示欢迎页。欢迎页须与现有视觉骨架和动效一致，并在文案叙事上升级为单主句策略，突出“兴趣圈 + 小趣陪伴”。

#### Scenario: 用户首次打开应用
- **WHEN** 应用启动且当前会话未标记为“已通过欢迎”
- **THEN** 系统全屏展示欢迎页并保持现有骨架与核心动效

#### Scenario: 欢迎主句展示
- **WHEN** 欢迎页渲染主文案
- **THEN** 系统仅展示一条主句并使用“小趣陪伴”语义，不出现冗长双副句叠加

#### Scenario: 小趣轻量形象融入
- **WHEN** 欢迎页展示兴趣圈叙事区域
- **THEN** 系统在不破坏原版式的前提下展示小趣轻量形象或标识，并保持页面视觉重心在兴趣圈

#### Scenario: 用户完成欢迎
- **WHEN** 用户在欢迎页执行完成操作（如点击按钮或滑动）
- **THEN** 欢迎页按既定动效关闭，并展示主应用框架

## ADDED Requirements

### Requirement: 欢迎页主句与语义 token 可配置

欢迎页主句文案与小趣相关语义文本 MUST 通过语义常量集中管理，以支持后续品牌文案迭代，不得在页面中硬编码。

#### Scenario: 文案常量化
- **WHEN** 开发者更新欢迎主句或小趣语义文本
- **THEN** 系统可通过常量层完成替换，页面代码无需新增硬编码字符串
