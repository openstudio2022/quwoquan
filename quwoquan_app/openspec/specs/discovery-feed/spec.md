# discovery-feed

## 新增需求

### 需求：发现页与设计文档一致

「发现」频道须与 CONTENT_SPECIFICATION v3.0 及 InformationArchitecture v3.1 一致：**Tab 1: 推荐**（跨类别精品混合流）、**Tab 2: 图片/视频/文章**（垂直品类内容池）。小趣参与：在发现页底部提供小趣悬浮球，协助用户快速总结感兴趣的长内容。

#### 场景：Tab 可见且可切换

- **当** 用户处于「发现」频道
- **则** Tab 为推荐与图片/视频/文章；切换 Tab 时内容流随之更新

#### 场景：内容类型与卡片

- **当** 内容流被渲染时
- **则** 帖子类型与当前 Tab 一致，卡片布局与 Figma 一致

### 需求：卡片头像与点击作者

发现卡片上的**作者头像**须使用圆形（avatar-user-md），**圈子标识**须使用圆角正方形（avatar-circle-sm）。当用户点击作者（头像或名称）时，应用须以全屏叠加层打开作者主页。

#### 场景：作者主页打开

- **当** 用户在发现卡片上点击作者区域
- **则** 展示作者主页叠加层，用户可关闭后回到发现页

### 需求：点击圈子打开圈子主页叠加层

当用户在发现卡片上点击圈子时，应用须打开圈子主页叠加层。

#### 场景：圈子主页打开

- **当** 用户在发现卡片上点击圈子区域
- **则** 展示圈子主页叠加层，用户可关闭

### 需求：点击帖子打开查看器或文章详情

当用户点击帖子时，应用须对图片/视频类打开沉浸式媒体查看器（工具栏中心含小趣入口），对文章类打开文章详情页。

#### 场景：媒体帖打开查看器

- **当** 用户点击图片或视频帖子
- **则** 沉浸式查看器打开，工具栏中心显示小趣入口

#### 场景：文章打开详情

- **当** 用户点击文章帖子
- **则** 打开文章详情页，工具栏含小趣入口

### 需求：发现页使用语义 token

发现页所有 UI 须使用 AppColors、AppSpacing、UITextConstants、ContentTypeConstants。禁止使用硬编码值。

#### 场景：发现页 token

- **当** 发现页与卡片被构建时
- **则** 所有样式与文案使用设计系统常量
