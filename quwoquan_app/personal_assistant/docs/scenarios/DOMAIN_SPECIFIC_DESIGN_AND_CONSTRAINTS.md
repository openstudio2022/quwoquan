# 垂类专项设计与约束

> **从属**：`../PERSONAL_ASSISTANT_DESIGN_AND_CONSTRAINTS.md`

## 1. 适用场景

当针对某个垂类做专项优化、迁移、治理或体验提升时，必须阅读本文。

## 2. 设计原则

- 优先增强对应 skill 资产，而不是增强 runtime 特判
- 垂类的知识、示例、状态机、检索策略与边界都应沉淀在该垂类目录
- 通用默认链路必须同样可工作，垂类只做增强，不替代默认主链路

## 3. 实现约束

- 禁止在 runtime 中新增该垂类专属 if/switch/关键词匹配
- 禁止围绕该垂类样例文案写长期回归断言
- 若需要兼容层，必须记录清理条件与时限

## 4. 验收要点

- 垂类变更是否全部回到对应 skill / config / metadata
- 通用兜底链路是否仍保持可用
- 是否新增了字符串驱动行为逻辑
- 测试是否以合同、阶段、输出结构为主，而不是以个别垂类句式为主
