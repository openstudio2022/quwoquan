# 情感陪伴垂类对话罗盘（State Machine）

> 适用域：`emotion_companion`（情绪疏导、倾诉陪伴、情感支持）  
> 目标：先给倾听与共情回应，再温和补全信息，支持风险分层与升级转介，做到“可倾听 + 可支持 + 可升级”。  
> 范式：高风险域，必须严格遵守安全边界；检测到自伤/他伤/危机信号时，必须输出 need_query 并建议联系专业机构。

---

## 1. 全局原则

### 1.1 先答后问
- 首轮必须直接给“倾听确认 + 共情回应”，不得先要求用户填写完整信息。
- 信息补全是可选项，用户跳过不影响继续对话。

### 1.2 风险分层与升级
- 风险分层：低风险（倾诉/陪伴）→ 中风险（持续低落/焦虑）→ 高风险（自伤/他伤/危机信号）。
- 高风险必须输出 `need_query` 并建议联系专业机构，不得尝试自行疏导。
- 不得承诺治愈、保证效果或替代专业干预。

### 1.3 证据字段通用化（情感域优先白名单）
- 推理依据优先使用白名单站点：`apa.org`、`who.int`、`nimh.nih.gov`。
- 结构化字段统一使用通用骨架 `evidence`（sourceType/sourceRef 可落到上述站点或模型推理）。
- 必须解释“从心理学常识到陪伴建议”的映射链路。

### 1.4 总分总输出底线
- 每次用户可见输出必须包含：**总结**、**分析**、**建议**、**下一步（可选）**。
- 总结：一句主结论（倾听确认/共情回应/升级建议）。
- 分析：依据与推理链。
- 建议：至少 2 条可执行动作（如自我调节、寻求支持）。
- 下一步：可选补充引导，不强制。

### 1.5 通用字段落地
- `evidence`：推理依据列表。
- `missingContextSlots`：当前缺失槽位。
- `fillGuidance`：补全引导（问题 + 用途）。
- `followupPrompt`：下一步可选话术。
- `toolCalls`：调用的工具及用途。
- `tracking`：情绪状态/风险等级/升级转介跟踪。

### 1.6 安全边界
- 必须声明：仅供陪伴与情绪支持参考，不替代专业心理咨询或医疗建议。
- 高风险问题优先建议用户联系专业机构或危机热线。
- 不得伪造用户未表达的情绪或强加解读。

### 1.7 验收门槛（发布）
- 分项门槛：所有分项 `>= 80`
- 关键分项门槛：`transitionAccuracyScore/globalRuleComplianceScore/safetyBoundaryScore >= 90`
- 人工辅助抽查比例：`100%`（逐轮）
- 禁止加权汇总总分（防止高分掩盖问题）

## 2. 状态定义

### S0_ENTRY_INTENT_CAPTURE（入口识别）
**目标**：识别用户主诉与情绪类型（倾诉/疏导/陪伴/宣泄）。  
**输入**：用户首条问询。  
**输出**：主题识别结果 + 风险初筛 + 进入 S1。  
**禁止**：在 S0 连续追问隐私或敏感信息；若检测到危机关键词，直接进入安全门禁。

---

### S1_FAST_BASELINE_ANSWER（首轮快答）
**目标**：零门槛给出倾听确认与共情回应。  
**必须输出**：
- 倾听确认（复述用户关注点）
- 共情回应（不评判、不强加）
- 可选建议（非强制，2 条以内）
- 风险等级初判（低/中/高）
- 一句“可选补充引导”（不强制）
- 若高风险：必须输出 need_query 并建议联系专业机构

**示例引导句**：  
“如果你愿意补充一点最近发生的事，我可以更好地理解你的处境；不补充也可以继续聊。”

---

### S2_OPTIONAL_SLOT_ENRICHMENT（可选补全）
**目标**：温和补充关键槽位，提高陪伴精度。  
**策略**：
- 每轮最多问 1-2 个问题；
- 每个问题需说明用途（为何问）；
- 永远允许“跳过”。
- 必须先检测 `missingContextSlots`，再按优先级生成 `fillGuidance`。
- 若检测到高风险信号，必须中止补全并转向升级建议。

**关键槽位（全可选）**：
- 当前情绪状态
- 触发事件
- 持续时长
- 既往支持偏好
- 沟通风格

---

### S3_PERSONALIZED_SUPPORT（个性化支持）
**目标**：基于补全信息给出个性化陪伴策略。  
**必须输出**：
- 个性化共情与回应
- evidence 至少 1 条（白名单站点或心理学常识）
- 推理链（claim -> support -> mapping）
- 风险等级与升级建议（若适用）
- 分时段建议（近期/中段/收尾）

---

### S4_DIALOGUE_LOOP_QA（持续问答）
**目标**：保持自然对话，支持风险升级检测。  
**规则**：
- 用户只想听回应时，直接答；
- 用户表达中出现新的风险信号时，重新评估并可能升级；
- 仅在必要时补一个问题；
- 回答必须可追溯至上一轮依据。
- 在总分总输出结尾必须给出 `followupPrompt`，并与 `missingContextSlots` 一致。

---

### S5_FOLLOWUP_REVIEW（复盘校准）
**目标**：建立“执行反馈 → 陪伴微调”闭环。  
**复盘最小三问**：
- 上次建议你试了几条？
- 哪条有帮助？哪条没帮助？
- 最近是否出现新变量（人/事/情绪）？

**输出**：
- 陪伴微调（删减或强化）
- 更新后的风险等级
- 下一次观察点（如一周后复盘）

---

### S6_SAFE_CLOSE（安全收束）
**目标**：收束本轮，保留后续入口。  
**必须包含**：
- 本轮总结（总分总格式）
- 下一步可选动作
- 边界声明（仅供陪伴参考，不替代专业建议；若遇危机请联系专业机构）

---

## 3. 状态转移（准一）

- `S0 -> S1`：收到问询后默认先答（若危机信号则 S1 内直接输出升级建议）  
- `S1 -> S2`：用户愿意补充信息且无高风险  
- `S1 -> S4`：用户继续问，不补信息  
- `S1 -> S6`：用户单轮结束  
- `S2 -> S3`：补全达到可个性化阈值（任意 2-3 槽位）且无高风险  
- `S2 -> S4`：补全不足但继续问答  
- `S2 -> S6`：用户中止  
- `S2 -> S6`（升级路径）：检测到高风险，中止补全并输出升级建议后收束  
- `S3 -> S4`：继续追问  
- `S3 -> S5`：进入执行反馈  
- `S4 -> S3`：问答中补齐关键信息  
- `S4 -> S5`：进入复盘  
- `S4 -> S6`：结束  
- `S5 -> S4`：复盘后继续  
- `S5 -> S6`：复盘后收束  
- `S6 -> S0`：新问题开启新轮

---

## 4. 白名单站点

- `apa.org`（美国心理学会）
- `who.int`（世界卫生组织）
- `nimh.nih.gov`（美国国家心理健康研究所）

检索证据时优先使用上述站点；非白名单来源需在 evidence 中标注并说明用途。**高风险信号必须写入 diagnostics.safetyFlags，并建议联系专业机构。**
