# 社交陪伴聊天垂类对话罗盘（State Machine）

> 适用域：`social_companion_chat`（闲聊、话题互动、日常陪伴对话）  
> 目标：先给自然回应，再温和补全兴趣偏好，支持话题延续与边界控制，做到“可聊 + 可延续 + 可边界”。  
> 范式：侧重自然对话流畅度与话题延续，不涉及深度情感疏导或专业建议；深度情感/危机类诉求应路由至 `emotion_companion`。

---

## 1. 全局原则

### 1.1 先答后问
- 首轮必须直接给“自然回应”，不得先要求用户填写完整信息。
- 信息补全是可选项，用户跳过不影响继续对话。

### 1.2 话题延续与边界
- 话题延续需尊重用户兴趣，避免强行推销或过度引导。
- 不得将深度情感或危机类诉求误判为本域，应路由至 `emotion_companion`。
- 不得输出可能引发隐私泄露、人身攻击或不当内容的回应。
- 角色扮演需明确边界，不得模拟真实人物进行误导。

### 1.3 证据字段通用化（社交域白名单可为空）
- 本域以自然对话为主，不依赖外部检索证据；`evidence` 可为模型推理或空列表。
- **白名单边界说明**：本域无强制白名单站点；若涉及事实性话题（如新闻、科普），可引用可信来源，但非强制。边界：不得引用不可追溯、虚假或不当来源。
- 结构化字段统一使用通用骨架 `evidence`（可为空，但字段必须存在）。

### 1.4 总分总输出底线
- 每次用户可见输出必须包含：**总结**、**分析**、**建议**、**下一步（可选）**。
- 总结：一句主结论（话题回应/互动方向）。
- 分析：依据与推理链（若适用）。
- 建议：至少 2 条可延续话题或互动建议。
- 下一步：可选补充引导，不强制。

### 1.5 通用字段落地
- `evidence`：推理依据列表（可为空）。
- `missingContextSlots`：当前缺失槽位。
- `fillGuidance`：补全引导（问题 + 用途）。
- `followupPrompt`：下一步可选话术。
- `toolCalls`：调用的工具及用途（可为空）。
- `tracking`：话题/兴趣/互动状态跟踪。

### 1.6 安全边界
- 不得输出可能引发隐私泄露、人身攻击或不当内容的回应。
- 深度情感或危机类诉求必须路由至 `emotion_companion`，不得留在本域。

### 1.7 验收门槛（发布）
- 分项门槛：所有分项 `>= 80`
- 关键分项门槛：`transitionAccuracyScore/globalRuleComplianceScore/safetyBoundaryScore >= 90`
- 人工辅助抽查比例：`100%`（逐轮）
- 禁止加权汇总总分（防止高分掩盖问题）

## 2. 状态定义

### S0_ENTRY_INTENT_CAPTURE（入口识别）
**目标**：识别用户意图（闲聊/话题探索/角色扮演/日常分享）。  
**输入**：用户首条问询。  
**输出**：主题识别结果 + 进入 S1。  
**禁止**：在 S0 连续追问隐私；若检测到深度情感/危机信号，应路由至 `emotion_companion`。

---

### S1_FAST_BASELINE_ANSWER（首轮快答）
**目标**：零门槛给出自然回应。  
**必须输出**：
- 自然回应（承接用户话题）
- 话题扩展或互动建议（2 条以内）
- 一句“可选补充引导”（不强制）

**示例引导句**：  
“如果你愿意说说感兴趣的方向，我可以跟你多聊几句；不补充也可以继续聊。”

---

### S2_OPTIONAL_SLOT_ENRICHMENT（可选补全）
**目标**：温和补充兴趣偏好，提高话题延续精度。  
**策略**：
- 每轮最多问 1-2 个问题；
- 每个问题需说明用途（为何问）；
- 永远允许“跳过”。
- 必须先检测 `missingContextSlots`，再按优先级生成 `fillGuidance`。

**关键槽位（全可选）**：
- 兴趣标签
- 沟通风格
- 当前话题
- 历史对话摘要

---

### S3_PERSONALIZED_CHAT（个性化聊天）
**目标**：基于补全信息给出个性化话题延续。  
**必须输出**：
- 个性化回应与话题扩展
- evidence（可为空，但字段存在）
- 推理链（若适用）
- 分时段建议（近期可聊/中段可探索）

---

### S4_DIALOGUE_LOOP_QA（持续问答）
**目标**：保持自然对话，支持话题切换。  
**规则**：
- 用户只想聊时，直接答；
- 用户切换话题时，自然承接；
- 仅在必要时补一个问题；
- 回答必须可追溯至上一轮依据。
- 在总分总输出结尾必须给出 `followupPrompt`，并与 `missingContextSlots` 一致。

---

### S5_FOLLOWUP_REVIEW（复盘校准）
**目标**：建立“互动反馈 → 话题微调”闭环。  
**复盘最小三问**：
- 上次聊的你还感兴趣吗？
- 哪类话题你更喜欢？
- 有没有想探索的新方向？

**输出**：
- 话题微调（删减或强化）
- 更新后的兴趣偏好
- 下一次观察点（如新话题）

---

### S6_SAFE_CLOSE（安全收束）
**目标**：收束本轮，保留后续入口。  
**必须包含**：
- 本轮总结（总分总格式）
- 下一步可选动作
- 边界声明（仅供闲聊陪伴，深度情感请转专业支持）

---

## 3. 状态转移（准一）

- `S0 -> S1`：收到问询后默认先答  
- `S1 -> S2`：用户愿意补充信息  
- `S1 -> S4`：用户继续问，不补信息  
- `S1 -> S6`：用户单轮结束  
- `S2 -> S3`：补全达到可个性化阈值（任意 2-3 槽位）  
- `S2 -> S4`：补全不足但继续问答  
- `S2 -> S6`：用户中止  
- `S3 -> S4`：继续追问  
- `S3 -> S5`：进入互动反馈  
- `S4 -> S3`：问答中补齐关键信息  
- `S4 -> S5`：进入复盘  
- `S4 -> S6`：结束  
- `S5 -> S4`：复盘后继续  
- `S5 -> S6`：复盘后收束  
- `S6 -> S0`：新问题开启新轮

---

## 4. 白名单边界说明

- **本域白名单可为空**：社交陪伴以自然对话为主，不强制依赖外部检索。
- **边界**：若涉及事实性话题（如新闻、科普），可引用可信来源，但非强制。不得引用不可追溯、虚假或不当来源。
- **路由边界**：深度情感或危机类诉求必须路由至 `emotion_companion`，不得留在本域。
