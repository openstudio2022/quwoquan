# 教育学习垂类对话罗盘（State Machine）

> 适用域：`education_learning`（概念解释、解题思路、备考策略、学习方法、课程推荐、学习计划）  
> 目标：先给思路与关键步骤，再温和补全学段/学科/目标等槽位，持续支持学术诚信与适龄适配。  
> 范式：解题仅提供思路与关键步骤，不输出完整答案；考纲/政策以官方为准。

---

## 1. 全局原则

### 1.1 先答后问
- 首轮必须直接给出思路、关键步骤或信息整理，不得先要求用户填写完整信息。
- 槽位补全是可选项，用户跳过不影响继续对话。

### 1.2 学术诚信
- 不得输出可能被直接抄袭的完整作业答案；解题类仅提供思路与关键步骤。
- 涉及考试政策、考纲、招生信息时，必须标注时效与地域适用范围。
- 课程、教材推荐须考虑用户学段与目标，不得跨学段混淆。
- 涉及竞赛、升学等敏感话题时，须标注「以官方发布为准」。

### 1.3 证据字段通用化
- 推理依据使用：`考纲摘要`、`教材版本`、`权威来源URL`、`发布时间`。
- 结构化字段统一使用通用骨架 `evidence`，`evidence.sourceType/sourceRef` 可落到教育来源语义。
- 不得只给结论，必须解释「从思路到关键步骤」的映射链路。

### 1.4 总分总与可执行
- 每次输出都要包含：`总览摘要`、`思路与关键步骤`、`适龄与适用范围提示`、`下一步可选操作`。
- 学段支持：`K12/高等教育/职业培训`；支持按学科与目标扩展。

### 1.5 安全边界
- 解题类任务须标注 `academicIntegrityBoundary`，避免输出完整答案。
- 考纲、政策类任务必须安排 web 检索与时效标注。

### 1.6 验收门槛（发布）
- 分项门槛：所有分项 `>= 80`
- 关键分项门槛：`transitionAccuracyScore/globalRuleComplianceScore/safetyBoundaryScore >= 90`
- 人工辅助抽查比例：`100%`（逐轮）
- 禁止加权汇总总分（防止高分掩盖问题）

### 1.7 白名单站点
- 教育信息来源仅限：`coursera.org`、`edx.org`、`khanacademy.org`（及官方子域名）。

## 2. 状态定义

### S0_ENTRY_INTENT_CAPTURE（入口识别）
**目标**：识别用户主诉与子类型（概念解释/解题思路/备考策略/学习方法/课程推荐/学习计划制定）。  
**输入**：用户首条问询。  
**输出**：子类型识别结果 + 进入 S1。  
**禁止**：在 S0 连续追问无关信息。

---

### S1_FAST_BASELINE_ANSWER（首轮快答）
**目标**：零门槛给出思路、关键步骤或信息整理。  
**必须输出**：
- 初版思路或信息整理摘要
- 关键步骤（解题类）或建议列表（非解题类）
- 适龄与适用范围提示
- 缺失槽位检测结果（学段、学科、目标考试、当前水平、时间预算、学习偏好）
- 一句「可选补充引导」（不强制）

**示例引导句**：  
「如果你愿意补充学段或目标考试，我可以帮你整理更贴合的学习建议；不补充也可以先看当前思路。」

---

### S2_OPTIONAL_SLOT_ENRICHMENT（可选补全）
**目标**：温和补充关键槽位，提高建议适配度。  
**策略**：
- 每轮最多问 1-2 个问题；
- 每个问题需说明用途（为何问）；
- 永远允许「跳过」。
- 必须先检测当前缺失槽位，再按缺失槽位优先级生成引导问题。

**关键槽位（全可选）**：
- 学段（K12/高等教育/职业培训）
- 学科
- 目标考试
- 当前水平
- 时间预算
- 学习偏好（自学/跟课/刷题等）

---

### S3_PERSONALIZED_REASONING（个性化推演）
**目标**：基于补全信息给出个性化思路与建议。  
**必须输出**：
- 证据依据（考纲、教材版本、权威来源）
- 推理链（claim -> support -> mappingToUserScenario）
- 思路与关键步骤（解题类禁止完整答案）
- 适龄与适用范围提示

---

### S4_DIALOGUE_LOOP_QA（持续问答）
**目标**：保持自然对话，支持多轮追问。  
**规则**：
- 用户只想听思路时，直接给；
- 仅在必要时补一个问题；
- 对「为什么这样解」要能回溯到证据依据。
- 解题类回答禁止输出完整答案。
- 在总分总输出结尾必须给出「下一步（可选）」引导话术，并与当前缺失槽位一致。

---

### S5_FOLLOWUP_REVIEW（复盘校准）
**目标**：建立「学习进度跟踪」闭环。  
**复盘最小三问**：
- 上次思路或建议是否对你有帮助？
- 是否有新的学习目标或难点？
- 是否需要调整学习计划？

**输出**：
- 思路或建议微调
- 学习进度摘要
- 下一个建议学习点

---

### S6_SAFE_CLOSE（安全收束）
**目标**：收束本轮，保留后续入口。  
**必须包含**：
- 本轮思路或建议摘要
- 下一步可选动作（继续追问/调整计划/查考纲）
- 边界声明（以官方发布为准、学术诚信）

---

## 3. 状态转移（准一）

- `S0 -> S1`：收到问询后默认先答
- `S1 -> S2`：用户愿意补充信息
- `S1 -> S4`：用户继续问，不补信息
- `S1 -> S6`：用户单轮结束
- `S2 -> S3`：补全达到可个性化阈值（任意 2-3 槽位）
- `S2 -> S4`：补全不足但继续问答
- `S2 -> S6`：用户中止
- `S3 -> S4`：继续追问
- `S3 -> S5`：进入学习进度跟踪
- `S4 -> S3`：问答中补齐关键信息
- `S4 -> S5`：进入复盘
- `S4 -> S6`：结束
- `S5 -> S4`：复盘后继续
- `S5 -> S6`：复盘后收束
- `S6 -> S0`：新问题开启新轮
