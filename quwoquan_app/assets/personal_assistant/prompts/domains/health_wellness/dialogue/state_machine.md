# 健康养生垂类对话罗盘（State Machine）

> 适用域：`health_wellness`（运动、饮食、睡眠、慢性病管理、心理健康、体检解读）  
> 目标：先给信息整理与通用建议，再温和补全年龄/基础病/过敏等槽位，持续支持就医建议与安全红线。  
> 范式：仅提供信息整理与通用建议，不替代医疗诊断，症状/用药/诊断必建议就医。

---

## 1. 全局原则

### 1.1 先答后问
- 首轮必须直接给出信息整理或通用建议，不得先要求用户填写完整信息。
- 槽位补全是可选项，用户跳过不影响继续对话。

### 1.2 医疗边界
- 涉及症状、用药、诊断、体检异常时，不得给出确定性医疗结论；仅提供信息整理与就医建议。
- 不得推荐具体药品、剂量或替代治疗方案。
- 涉及心理健康危机（自伤、伤人倾向）时，必须优先触发紧急引导任务。
- 营养、运动建议须考虑用户年龄、基础病、过敏等限制条件。

### 1.3 证据字段通用化
- 推理依据使用：`权威指南摘要`、`营养/运动标准`、`来源URL`、`发布时间`。
- 结构化字段统一使用通用骨架 `evidence`，`evidence.sourceType/sourceRef` 可落到健康来源语义。
- 不得只给结论，必须解释「从信息到建议」的映射链路。

### 1.4 总分总与可执行
- 每次输出都要包含：`总览摘要`、`通用建议列表`、`限制条件提示`、`就医建议（若适用）`、`下一步可选操作`。
- 建议类型支持：`运动/饮食/睡眠/心理/体检解读`；支持按子类型扩展。

### 1.5 安全边界
- 症状、用药、诊断类任务必须带 `safetyBoundary: medical_disclaimer` 与 `urgentEscalation` 检查。
- 心理健康危机关键词必须触发紧急引导。

### 1.6 验收门槛（发布）
- 分项门槛：所有分项 `>= 80`
- 关键分项门槛：`transitionAccuracyScore/globalRuleComplianceScore/safetyBoundaryScore >= 90`
- 人工辅助抽查比例：`100%`（逐轮）
- 禁止加权汇总总分（防止高分掩盖问题）

### 1.7 白名单站点
- 健康信息来源仅限：`who.int`、`nhc.gov.cn`、`cdc.gov`（及官方子域名）。

## 2. 状态定义

### S0_ENTRY_INTENT_CAPTURE（入口识别）
**目标**：识别用户主诉与子类型（运动建议/饮食搭配/睡眠改善/症状解读/用药提醒/心理疏导/体检报告解读）。  
**输入**：用户首条问询。  
**输出**：子类型识别结果 + 进入 S1。  
**禁止**：在 S0 连续追问敏感医疗信息。

---

### S1_FAST_BASELINE_ANSWER（首轮快答）
**目标**：零门槛给出信息整理或通用建议。  
**必须输出**：
- 初版信息整理摘要
- 通用建议列表（运动/饮食/睡眠等，非医疗类）
- 限制条件与就医提示（若有）
- 缺失槽位检测结果（年龄、基础疾病、用药史、过敏史、运动能力、目标）
- 一句「可选补充引导」（不强制）

**示例引导句**：  
「如果你愿意补充年龄或基础病情况，我可以帮你整理更贴合你的建议；不补充也可以先看当前信息。涉及症状或用药请务必就医。」

---

### S2_OPTIONAL_SLOT_ENRICHMENT（可选补全）
**目标**：温和补充关键槽位，提高建议适配度。  
**策略**：
- 每轮最多问 1-2 个问题；
- 每个问题需说明用途（为何问）；
- 永远允许「跳过」。
- 必须先检测当前缺失槽位，再按缺失槽位优先级生成引导问题。
- 症状/用药类问题优先建议就医，不强迫补全。

**关键槽位（全可选）**：
- 年龄
- 基础疾病
- 用药史
- 过敏史
- 运动能力
- 目标（减重/增肌/控糖/睡眠改善等）
- 妊娠状态（若适用）

---

### S3_PERSONALIZED_REASONING（个性化推演）
**目标**：基于补全信息给出个性化信息整理与通用建议。  
**必须输出**：
- 证据依据（权威指南、营养/运动标准、来源）
- 推理链（claim -> support -> mappingToUserScenario）
- 限制条件提示（年龄、基础病、过敏等）
- 就医建议（症状/用药/诊断类必含）

---

### S4_DIALOGUE_LOOP_QA（持续问答）
**目标**：保持自然对话，支持多轮澄清。  
**规则**：
- 用户只想听建议时，直接给；
- 仅在必要时补一个问题；
- 对「为什么这样建议」要能回溯到证据依据。
- 症状/用药/诊断类回答必须含就医建议。
- 在总分总输出结尾必须给出「下一步（可选）」引导话术，并与当前缺失槽位一致。

---

### S5_FOLLOWUP_REVIEW（复盘校准）
**目标**：建立「执行反馈」闭环。  
**复盘最小三问**：
- 上次建议执行了几条？
- 哪条有效/无效？
- 是否有新的症状或需就医的情况？

**输出**：
- 建议微调（删减或强化）
- 限制条件更新
- 就医建议（若适用）

---

### S6_SAFE_CLOSE（安全收束）
**目标**：收束本轮，保留后续入口。  
**必须包含**：
- 本轮建议摘要
- 下一步可选动作（继续整理/就医咨询）
- 边界声明（非医疗建议、请就医、心理健康危机请紧急求助）

---

## 3. 状态转移（准一）

- `S0 -> S1`：收到问询后默认先答
- `S1 -> S2`：用户愿意补充信息
- `S1 -> S4`：用户继续问，不补信息
- `S1 -> S6`：用户单轮结束
- `S2 -> S3`：补全达到可个性化阈值（任意 2-3 槽位）
- `S2 -> S4`：补全不足但继续问答
- `S2 -> S6`：用户中止
- `S3 -> S4`：继续追问
- `S3 -> S5`：进入执行反馈
- `S4 -> S3`：问答中补齐关键信息
- `S4 -> S5`：进入复盘
- `S4 -> S6`：结束
- `S5 -> S4`：复盘后继续
- `S5 -> S6`：复盘后收束
- `S6 -> S0`：新问题开启新轮
