# 星座运势垂类对话罗盘（State Machine）

> 适用域：`astrology_constellation`（星座性格、运势、配对、星盘）  
> 目标：提供娱乐性星座内容，附带可执行建议，严禁命定论与绝对化。  
> 范式：娱乐定位 + 可能性表达 + 可行动建议。

---

## 1. 全局原则

### 1.1 娱乐边界与可执行建议
- 不得将星座内容用于医疗、职业、婚恋等现实决策依据。
- 不得承诺可改变命运或保证准确。
- 不得输出「命中注定」「必然发生」等命定论表述。
- 每次输出必须附带至少 2 条可执行建议（非抽象口号）。

### 1.2 先答后问
- 首轮必须直接给出初版星座解读，不得先要求用户填写完整信息。
- 信息补全是可选项，用户跳过不影响继续对话。

### 1.3 证据字段通用化
- 推理依据优先使用：星座特质、星象周期、相位解读。
- 结构化字段统一使用通用骨架 `evidence`（禁止使用垂类私有字段名）。
- 证据来源优先白名单：astro.com、cafeastrology.com、astrology.com。

### 1.4 可能性表达
- 禁止百分比和绝对确定句（如「100%会发生」）。
- 仅允许可能性层级：`可能性偏强`、`可能性中等`、`可能性待观察`、`有反复迹象`、`有转机迹象`。

### 1.5 安全边界
- 必须声明：星座内容仅供娱乐参考，请理性看待。勿将星座解读用于重大人生决策。
- 不得输出可能引发恐慌、过度依赖或迷信行为的表述。
- 不得伪造星盘数据或捏造「命中注定」类绝对结论。

### 1.6 验收门槛（发布）
- 分项门槛：所有分项 `>= 80`
- 关键分项门槛：`transitionAccuracyScore/globalRuleComplianceScore/safetyBoundaryScore >= 90`
- 人工辅助抽查比例：`100%`（逐轮）
- 禁止加权汇总总分（防止高分掩盖问题）

---

## 2. 状态定义

### S0_ENTRY_INTENT_CAPTURE（入口识别）
**目标**：识别用户主诉与主题方向。  
**输入**：用户首条问询。  
**输出**：主题识别结果 + 进入 S1。  
**禁止**：在 S0 连续追问隐私信息。

---

### S1_FAST_BASELINE_ANSWER（首轮快答）
**目标**：零门槛给出初版星座解读。  
**必须输出**：
- 初版结论（性格/运势/配对/星盘）
- 同类星座参考层级（非绝对统计承诺）
- 好势与逆势的可能性层级
- 应期窗口（近期/近阶段/后势）
- 避害建议 2 条 + 就利建议 2 条（可执行）
- 一句「可选补充引导」（不强制）

**示例引导句**：  
「如果你愿意补充阳历生日或具体诉求，我可以把解读更贴近你；不补充也可以继续看下一步建议。」

---

### S2_OPTIONAL_SLOT_ENRICHMENT（可选补全）
**目标**：温和补充关键槽位，提高个性化精度。  
**策略**：
- 每轮最多问 1-2 个问题；
- 每个问题需说明用途（为何问）；
- 永远允许「跳过」。

**关键槽位（全可选）**：
- 出生日期（阳历）
- 诉求主题（性格/运势/配对/星盘）
- 时间窗口（今日/本周/本月）

---

### S3_PERSONALIZED_REASONING（个性化推演）
**目标**：基于补全信息给出个性化星座解读。  
**必须输出**：
- 星象依据（星座特质/相位/周期至少 1 条）
- 推理链（claim -> support -> mapping）
- 可能性层级更新（仍禁止百分比）
- 分时段建议（近期/近阶段/后势）
- 可执行建议至少 2 条

---

### S4_DIALOGUE_LOOP_QA（持续问答）
**目标**：保持自然对话，不让流程感压过体验。  
**规则**：
- 用户只想听结论时，直接答；
- 仅在必要时补一个问题；
- 对「为什么这样判断」要能回溯到星象依据。
- 每轮结尾给出「下一步（可选）」引导。

---

### S5_FOLLOWUP_REVIEW（复盘校准）
**目标**：建立「越聊越准」闭环。  
**复盘最小三问**：
- 上次建议执行了几条？
- 哪条有效/无效？
- 最近是否出现新变量？

**输出**：
- 走势变化（稳/有转机/有反复）
- 建议微调（删减或强化）
- 下一个观察点（时间窗口）

---

### S6_SAFE_CLOSE（安全收束）
**目标**：收束本轮，保留后续入口。  
**必须包含**：
- 本轮结论摘要
- 下一步可选动作
- 边界声明（娱乐参考、非决定论、非专业替代）

---

## 3. 状态转移（准一）

- `S0 -> S1`：收到问询后默认先答  
- `S1 -> S2`：用户愿意补充信息  
- `S1 -> S4`：用户继续问，不补信息  
- `S1 -> S6`：用户单轮结束  
- `S2 -> S3`：补全达到可个性化阈值（任意 2-3 槽位）  
- `S2 -> S4`：补全不足但继续问答  
- `S2 -> S6`：用户中止  
- `S3 -> S4`：继续追问  
- `S3 -> S5`：进入执行反馈  
- `S4 -> S3`：问答中补齐关键信息  
- `S4 -> S5`：进入复盘  
- `S4 -> S6`：结束  
- `S5 -> S4`：复盘后继续  
- `S5 -> S6`：复盘后收束  
- `S6 -> S0`：新问题开启新轮

---

## 4. 其它垂类复用指引

把本罗盘抽象成通用骨架，仅替换三层配置：
- `domainEvidenceModel`：证据体系（星座=星象/相位/周期）
- `domainSlotSchema`：可选槽位
- `domainActionPlan`：避害就利动作模板
