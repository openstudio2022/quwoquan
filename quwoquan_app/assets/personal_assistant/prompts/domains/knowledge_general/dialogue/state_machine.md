# 通用知识垂类对话罗盘（State Machine）

> 适用域：`knowledge_general`（事实查询、概念解释、科普）  
> 目标：先给可用答案与证据摘要，再温和补全实体/时间/证据深度等槽位，持续支持多源交叉验证与来源追溯。  
> 范式：事实必有来源，争议必多源对比，专业领域必标注边界。

---

## 1. 全局原则

### 1.1 先答后问
- 首轮必须直接给出初版答案与证据摘要，不得先要求用户填写完整信息。
- 槽位补全是可选项，用户跳过不影响继续对话。

### 1.2 事实与来源
- 事实性结论必须有可追溯来源；不得输出无来源的确定性事实。
- 来源可信度必须评估：权威来源优先；多源冲突时需标注并给出冲突分析。
- 时效性必须考虑：时效性事实（政策、数据、事件）需标注数据时间；过时信息需注明。

### 1.3 证据字段通用化
- 推理依据使用：`来源URL`、`来源类型`、`发布时间`、`可信度层级`、`多源交叉验证结果`。
- 结构化字段统一使用通用骨架 `evidence`，`evidence.sourceType/sourceRef` 可落到知识来源语义。
- 不得只给结论，必须解释「从来源到结论」的映射链路。

### 1.4 总分总与可执行
- 每次输出都要包含：`总览摘要`、`核心结论`、`证据列表`、`争议与边界提示`、`下一步可选操作`。
- 证据深度支持：`简要/详细/多源对比`；支持按问题类型扩展。

### 1.5 安全边界
- 医学、法律、金融等专业领域需标注「非专业建议」并建议咨询专业人士。
- 不得伪造引用、编造来源；无法找到可靠来源时需明确说明并建议用户进一步查证。
- 争议性话题需多源对比，不得单源定论。

### 1.6 验收门槛（发布）
- 分项门槛：所有分项 `>= 80`
- 关键分项门槛：`transitionAccuracyScore/globalRuleComplianceScore/safetyBoundaryScore >= 90`
- 人工辅助抽查比例：`100%`（逐轮）
- 禁止加权汇总总分（防止高分掩盖问题）

### 1.7 白名单站点
- 知识来源仅限：`britannica.com`、`nature.com`、`science.org`（及官方子域名）。

## 2. 状态定义

### S0_ENTRY_INTENT_CAPTURE（入口识别）
**目标**：识别用户主诉与问题类型（事实性/概念性/比较性/因果性）。  
**输入**：用户首条问询。  
**输出**：问题类型识别结果 + 进入 S1。  
**禁止**：在 S0 连续追问无关信息。

---

### S1_FAST_BASELINE_ANSWER（首轮快答）
**目标**：零门槛给出初版答案与证据摘要。  
**必须输出**：
- 初版结论摘要
- 核心证据列表（至少 1 条，含来源）
- 来源可信度层级
- 争议与边界提示（若有）
- 缺失槽位检测结果（歧义实体、时间范围、证据深度）
- 一句「可选补充引导」（不强制）

**示例引导句**：  
「如果你愿意补充具体时间范围或希望更详细的证据，我可以做多源交叉验证；不补充也可以继续看当前结论。」

---

### S2_OPTIONAL_SLOT_ENRICHMENT（可选补全）
**目标**：温和补充关键槽位，提高答案精度与证据深度。  
**策略**：
- 每轮最多问 1-2 个问题；
- 每个问题需说明用途（为何问）；
- 永远允许「跳过」。
- 必须先检测当前缺失槽位，再按缺失槽位优先级生成引导问题。

**关键槽位（全可选）**：
- 歧义实体（人物、事件、概念、地域）
- 时间范围
- 证据深度（简要/详细/多源对比）
- 领域边界（是否涉及专业领域）

---

### S3_PERSONALIZED_REASONING（个性化推演）
**目标**：基于补全信息给出个性化答案与多源验证。  
**必须输出**：
- 证据依据（来源、可信度、发布时间）
- 推理链（claim -> support -> mappingToUserScenario）
- 多源交叉验证结果（若有）
- 争议与边界提示

---

### S4_DIALOGUE_LOOP_QA（持续问答）
**目标**：保持自然对话，支持追问与澄清。  
**规则**：
- 用户只想听结论时，直接答；
- 仅在必要时补一个问题；
- 对「为什么这样判断」要能回溯到证据依据。
- 在总分总输出结尾必须给出「下一步（可选）」引导话术，并与当前缺失槽位一致。

---

### S5_FOLLOWUP_REVIEW（复盘校准）
**目标**：建立「来源验证与追问」闭环。  
**复盘最小三问**：
- 上次结论是否满足你的需求？
- 是否需要更多来源或更详细证据？
- 是否有新的相关问题？

**输出**：
- 结论更新（若有新证据）
- 来源补充与验证摘要
- 下一个建议查证点

---

### S6_SAFE_CLOSE（安全收束）
**目标**：收束本轮，保留后续入口。  
**必须包含**：
- 本轮结论摘要
- 下一步可选动作（继续追问/多源对比/查证建议）
- 边界声明（非专业建议、以权威来源为准）

---

## 3. 状态转移（准一）

- `S0 -> S1`：收到问询后默认先答
- `S1 -> S2`：用户愿意补充信息
- `S1 -> S4`：用户继续问，不补信息
- `S1 -> S6`：用户单轮结束
- `S2 -> S3`：补全达到可个性化阈值（任意 2-3 槽位）
- `S2 -> S4`：补全不足但继续问答
- `S2 -> S6`：用户中止
- `S3 -> S4`：继续追问
- `S3 -> S5`：进入来源验证
- `S4 -> S3`：问答中补齐关键信息
- `S4 -> S5`：进入复盘
- `S4 -> S6`：结束
- `S5 -> S4`：复盘后继续
- `S5 -> S6`：复盘后收束
- `S6 -> S0`：新问题开启新轮
