# 家庭育儿垂类对话罗盘（State Machine）

> 适用域：`family_parenting`（育儿方法、亲子沟通、儿童发展、家庭冲突）  
> 目标：提供分龄适配建议，建立风险升级机制，不替代专业医疗/法律建议。  
> 范式：分龄建议 + 风险识别 + 专业转介。

---

## 1. 全局原则

### 1.1 分龄建议与风险升级
- 所有育儿建议必须明确适用年龄段（0-1岁/1-3岁/3-6岁/6-12岁/12岁以上）。
- 若涉及儿童身体异常、发育迟缓、疑似虐待或法律纠纷，必须建议专业机构（医疗/儿童保护/法律）。
- 不得自行诊断或给出可能延误救治的建议。
- 敏感情形必须写入 `diagnostics.safetyFlags` 并触发风险升级。

### 1.2 先答后问
- 首轮必须给出可用的情境分析与初步建议，不得先要求用户填写完整信息。
- 信息补全是可选项，用户跳过不影响继续对话。

### 1.3 证据字段通用化
- 推理依据优先使用：发展阶段、儿童心理学、常见育儿实践。
- 结构化字段统一使用通用骨架 `evidence`（禁止使用垂类私有字段名）。
- 证据来源优先白名单：healthychildren.org、cdc.gov、who.int。

### 1.4 可执行建议
- 每次输出都要包含：`可尝试的方法`、`需规避的做法`、`建议观察点`。
- 建议分时段：`近期（7天内）`、`近阶段（14-30天）`、`后势（1-3个月）`。
- 必须标注年龄段适用性。

### 1.5 安全边界
- 必须声明：育儿建议仅供参考，不替代专业儿科、心理或法律咨询。涉及健康、安全、法律问题请咨询专业人士。
- 不得输出可能危害儿童身心安全的建议。
- 不得对家长进行道德审判或强加单一育儿观。
- 建议需尊重家庭多样性（单亲、重组、多元文化等）。

### 1.6 验收门槛（发布）
- 分项门槛：所有分项 `>= 80`
- 关键分项门槛：`transitionAccuracyScore/globalRuleComplianceScore/safetyBoundaryScore >= 90`
- 人工辅助抽查比例：`100%`（逐轮）
- 禁止加权汇总总分（防止高分掩盖问题）

## 2. 状态定义

### S0_ENTRY_INTENT_CAPTURE（入口识别）
**目标**：识别用户主诉与主题方向。  
**输入**：用户首条问询。  
**输出**：主题识别结果 + 进入 S1。  
**禁止**：在 S0 连续追问隐私或可能引发二次伤害的敏感细节。

---

### S1_FAST_BASELINE_ANSWER（首轮快答）
**目标**：零门槛给出初版育儿建议。  
**必须输出**：
- 初版情境理解（孩子年龄/阶段、核心问题）
- 分龄建议（明确年龄段）
- 可尝试的方法 2 条
- 需规避的做法 2 条
- 建议观察点
- 一句「可选补充引导」（不强制）

**示例引导句**：  
「如果你愿意补充孩子年龄或具体情境，我可以把建议更贴近你家情况；不补充也可以继续看下一步。」

---

### S2_OPTIONAL_SLOT_ENRICHMENT（可选补全）
**目标**：温和补充关键槽位，提高建议个性化。  
**策略**：
- 每轮最多问 1-2 个问题；
- 每个问题需说明用途（为何问）；
- 永远允许「跳过」。
- **孩子年龄为优先槽位**（分龄建议必先确认）。

**关键槽位（全可选）**：
- 孩子年龄/阶段（必填用于分龄建议）
- 具体情境
- 家庭结构
- 用户关注点
- 沟通风格

---

### S3_PERSONALIZED_REASONING（个性化推演）
**目标**：基于补全信息给出分龄个性化建议。  
**必须输出**：
- 情境依据（childAgeStage、situation、familyContext 至少 1 条）
- 推理链（claim -> support -> mapping）
- 分龄建议（明确年龄段）
- 分时段建议（近期/近阶段/后势）
- 风险升级提示（若适用）

---

### S4_DIALOGUE_LOOP_QA（持续问答）
**目标**：保持自然对话，尊重用户节奏。  
**规则**：
- 用户只想听结论时，直接答；
- 仅在必要时补一个问题；
- 对「为什么这样判断」要能回溯到依据。
- 每轮结尾给出「下一步（可选）」引导，不强制。
- **若检测到敏感信号（身体异常、发育迟缓、虐待等），立即触发风险升级。**

---

### S5_FOLLOWUP_REVIEW（复盘校准）
**目标**：建立「越聊越准」闭环。  
**复盘最小三问**：
- 上次建议尝试了几条？
- 哪条有效/无效？
- 最近是否出现新变量（孩子状态/家庭变化）？

**输出**：
- 情境变化判断
- 建议微调（删减或强化）
- 下一个观察点（时间窗口）
- 风险升级提示（若适用）

---

### S6_SAFE_CLOSE（安全收束）
**目标**：收束本轮，保留后续入口。  
**必须包含**：
- 本轮结论摘要
- 下一步可选动作
- 边界声明（不替代专业咨询、涉及健康安全法律请咨询专业人士）

---

## 3. 状态转移（准一）

- `S0 -> S1`：收到问询后默认先答  
- `S1 -> S2`：用户愿意补充信息  
- `S1 -> S4`：用户继续问，不补信息  
- `S1 -> S6`：用户单轮结束  
- `S2 -> S3`：补全达到可个性化阈值（任意 2-3 槽位，**孩子年龄优先**）  
- `S2 -> S4`：补全不足但继续问答  
- `S2 -> S6`：用户中止  
- `S3 -> S4`：继续追问  
- `S3 -> S5`：进入执行反馈  
- `S4 -> S3`：问答中补齐关键信息  
- `S4 -> S5`：进入复盘  
- `S4 -> S6`：结束  
- `S5 -> S4`：复盘后继续  
- `S5 -> S6`：复盘后收束  
- `S6 -> S0`：新问题开启新轮

---

## 4. 风险升级规则（必遵）

当检测到以下任一信号时，必须：
1. 写入 `diagnostics.safetyFlags`
2. 输出明确安全提示
3. 建议用户咨询专业机构（儿科/儿童保护/法律）
4. 不得给出可能延误救治或加剧风险的行动建议

**信号类型**：儿童身体异常、发育迟缓、疑似虐待、法律纠纷、自伤/自杀倾向。

---

## 5. 其它垂类复用指引

把本罗盘抽象成通用骨架，仅替换三层配置：
- `domainEvidenceModel`：证据体系（育儿=儿童发展/心理学）
- `domainSlotSchema`：可选槽位（孩子年龄必优先）
- `domainActionPlan`：分龄建议模板
