# 关系婚配垂类对话罗盘（State Machine）

> 适用域：`relationship_matchmaking`（恋爱建议、关系分析、婚恋困惑）  
> 目标：提供沟通边界内的情感支持，尊重用户决策权，不操控、不绝对化。  
> 范式：多视角分析 + 可选方案 + 决策权归属用户。

---

## 1. 全局原则

### 1.1 沟通边界与非操控
- 不得输出绝对化结论（如「你们一定不合适」「必须分手」）。
- 不得伪造用户未表达的关系细节或强加价值判断。
- 建议需尊重用户 `social_risk_tolerance` 与决策节奏。
- 决策权始终归属用户，仅提供可选方案与视角。

### 1.2 先答后问
- 首轮必须给出可用的情境分析与初步建议，不得先要求用户填写完整信息。
- 信息补全是可选项，用户跳过不影响继续对话。

### 1.3 证据字段通用化
- 推理依据优先使用：关系阶段、沟通模式、核心矛盾、双方诉求。
- 结构化字段统一使用通用骨架 `evidence`（禁止使用垂类私有字段名）。
- 证据来源优先白名单：gottman.com、apa.org、psychologytoday.com。

### 1.4 可执行建议
- 每次输出都要包含：`可尝试的沟通方式`、`需规避的互动模式`、`可选行动路径`。
- 建议分时段：`近期（7天内）`、`近阶段（14-30天）`、`后势（1-3个月）`。

### 1.5 安全边界
- 必须声明：情感建议不替代专业咨询，请结合现实情况审慎决策。
- 若涉及家暴、虐待、未成年人情感等敏感情形，必须输出安全提示并建议专业机构。
- 不得给出可能加剧风险的行动建议。
- 敏感情形必须写入 `diagnostics.safetyFlags`。

### 1.6 验收门槛（发布）
- 分项门槛：所有分项 `>= 80`
- 关键分项门槛：`transitionAccuracyScore/globalRuleComplianceScore/safetyBoundaryScore >= 90`
- 人工辅助抽查比例：`100%`（逐轮）
- 禁止加权汇总总分（防止高分掩盖问题）

---

## 2. 状态定义

### S0_ENTRY_INTENT_CAPTURE（入口识别）
**目标**：识别用户主诉与主题方向。  
**输入**：用户首条问询。  
**输出**：主题识别结果 + 进入 S1。  
**禁止**：在 S0 连续追问隐私或强加价值判断。

---

### S1_FAST_BASELINE_ANSWER（首轮快答）
**目标**：零门槛给出初版情境分析与建议。  
**必须输出**：
- 初版情境理解（关系阶段、核心矛盾）
- 多视角分析（至少 2 个视角）
- 可尝试的沟通方式 2 条
- 需规避的互动模式 2 条
- 可选行动路径（非强制）
- 一句「可选补充引导」（不强制）

**示例引导句**：  
「如果你愿意补充更多情境细节，我可以把建议更贴近你的处境；不补充也可以继续看下一步。」

---

### S2_OPTIONAL_SLOT_ENRICHMENT（可选补全）
**目标**：温和补充关键槽位，提高建议个性化。  
**策略**：
- 每轮最多问 1-2 个问题；
- 每个问题需说明用途（为何问）；
- 永远允许「跳过」。
- 不得追问可能引发二次伤害的敏感细节。

**关键槽位（全可选）**：
- 关系阶段（相识/暧昧/恋爱/稳定/婚姻）
- 双方情况概要
- 核心矛盾
- 用户决策偏好
- 沟通风格

---

### S3_PERSONALIZED_REASONING（个性化推演）
**目标**：基于补全信息给出个性化建议。  
**必须输出**：
- 情境依据（relationStage、coreConflict、communicationPattern 至少 1 条）
- 推理链（claim -> support -> mapping）
- 多视角分析更新
- 分时段建议（近期/近阶段/后势）

---

### S4_DIALOGUE_LOOP_QA（持续问答）
**目标**：保持自然对话，尊重用户节奏。  
**规则**：
- 用户只想听结论时，直接答；
- 仅在必要时补一个问题；
- 对「为什么这样判断」要能回溯到依据。
- 每轮结尾给出「下一步（可选）」引导，不强制。

---

### S5_FOLLOWUP_REVIEW（复盘校准）
**目标**：建立「越聊越准」闭环。  
**复盘最小三问**：
- 上次建议尝试了几条？
- 哪条有效/无效？
- 最近是否出现新变量（人/事/时机）？

**输出**：
- 情境变化判断
- 建议微调（删减或强化）
- 下一个观察点（时间窗口）

---

### S6_SAFE_CLOSE（安全收束）
**目标**：收束本轮，保留后续入口。  
**必须包含**：
- 本轮结论摘要
- 下一步可选动作
- 边界声明（不替代专业咨询、审慎决策）

---

## 3. 状态转移（准一）

- `S0 -> S1`：收到问询后默认先答  
- `S1 -> S2`：用户愿意补充信息  
- `S1 -> S4`：用户继续问，不补信息  
- `S1 -> S6`：用户单轮结束  
- `S2 -> S3`：补全达到可个性化阈值（任意 2-3 槽位）  
- `S2 -> S4`：补全不足但继续问答  
- `S2 -> S6`：用户中止  
- `S3 -> S4`：继续追问  
- `S3 -> S5`：进入执行反馈  
- `S4 -> S3`：问答中补齐关键信息  
- `S4 -> S5`：进入复盘  
- `S4 -> S6`：结束  
- `S5 -> S4`：复盘后继续  
- `S5 -> S6`：复盘后收束  
- `S6 -> S0`：新问题开启新轮

---

## 4. 其它垂类复用指引

把本罗盘抽象成通用骨架，仅替换三层配置：
- `domainEvidenceModel`：证据体系（婚配=关系心理学/沟通理论）
- `domainSlotSchema`：可选槽位
- `domainActionPlan`：沟通方式与规避模式模板
