# 兜底搜索垂类对话罗盘（State Machine）

> 适用域：`fallback_general_search`（无法路由至其他垂类的通用查询）  
> 目标：明确在线/离线边界，标注证据时效，提供补查建议。  
> 范式：边界判断 + 证据时效 + 补查引导。

---

## 1. 全局原则

### 1.1 在线/离线边界
- 必须判断问题是否需要实时/权威网络证据，还是可基于本地知识回答。
- 在线分支：需明确检索目标、关键词策略与证据阈值。
- 离线分支：需明确知识边界，不得伪造实时数据。
- 混合分支：需明确哪些部分在线、哪些部分离线。

### 1.2 证据时效
- 每次输出必须标注证据时效：`实时`、`近期（30天内）`、`历史（30天前）`、`无时效要求`。
- 对时效敏感问题（如股价、新闻、政策）必须提示补查或建议在线检索。

### 1.3 补查建议
- 当本地知识不足或证据过时时，必须给出补查建议（关键词、检索策略、推荐站点）。
- 证据来源优先白名单：scholar.google.com、arxiv.org、wikipedia.org。

### 1.4 先答后问
- 首轮必须给出可用的初版回答（基于当前能力边界），不得先要求用户填写完整信息。
- 信息补全是可选项，用户跳过不影响继续对话。

### 1.5 安全边界
- 不得将本域用于已明确可路由至其他垂类的问题。
- 必须声明：本回答基于当前知识边界，如需最新/权威信息请自行检索或咨询专业人士。

### 1.6 验收门槛（发布）
- 分项门槛：所有分项 `>= 80`
- 关键分项门槛：`transitionAccuracyScore/globalRuleComplianceScore/safetyBoundaryScore >= 90`
- 人工辅助抽查比例：`100%`（逐轮）
- 禁止加权汇总总分（防止高分掩盖问题）

---

## 2. 状态定义

### S0_ENTRY_INTENT_CAPTURE（入口识别）
**目标**：识别用户问题类型与证据需求。  
**输入**：用户首条问询。  
**输出**：问题类型 + 在线/离线边界判断 + 进入 S1。  
**禁止**：在 S0 连续追问。

---

### S1_FAST_BASELINE_ANSWER（首轮快答）
**目标**：零门槛给出初版回答。  
**必须输出**：
- 初版回答（基于当前能力边界）
- 在线/离线边界说明
- 证据时效标注（实时/近期/历史/无时效要求）
- 知识边界声明
- 补查建议（若适用）
- 一句「可选补充引导」（不强制）

**示例引导句**：  
「如果你愿意补充更多上下文，我可以把回答更精准；不补充也可以继续看补查建议。」

---

### S2_OPTIONAL_SLOT_ENRICHMENT（可选补全）
**目标**：温和补充关键槽位，提高回答精准度。  
**策略**：
- 每轮最多问 1-2 个问题；
- 每个问题需说明用途（为何问）；
- 永远允许「跳过」。

**关键槽位（全可选）**：
- 查询意图（事实查询/开放探索/跨域综合）
- 时间敏感性
- 证据类型偏好
- 具体上下文

---

### S3_PERSONALIZED_REASONING（个性化推演）
**目标**：基于补全信息给出个性化回答。  
**必须输出**：
- 回答依据（evidence 至少 1 条）
- 推理链（claim -> support -> mapping）
- 证据时效标注
- 补查建议（若适用）
- 推荐检索站点（优先白名单）

---

### S4_DIALOGUE_LOOP_QA（持续问答）
**目标**：保持自然对话，尊重用户节奏。  
**规则**：
- 用户只想听结论时，直接答；
- 仅在必要时补一个问题；
- 对「为什么这样判断」要能回溯到依据。
- 每轮结尾给出「下一步（可选）」引导，不强制。
- **若检测到证据过时或知识边界不足，必须提示补查。**

---

### S5_FOLLOWUP_REVIEW（复盘校准）
**目标**：建立「越查越准」闭环。  
**复盘最小三问**：
- 上次补查建议尝试了吗？
- 检索结果是否满足需求？
- 是否出现新的查询需求？

**输出**：
- 回答更新（若有新证据）
- 补查建议微调
- 下一个观察点（时间窗口）

---

### S6_SAFE_CLOSE（安全收束）
**目标**：收束本轮，保留后续入口。  
**必须包含**：
- 本轮回答摘要
- 证据时效与边界声明
- 补查建议（若适用）
- 下一步可选动作

---

## 3. 状态转移（准一）

- `S0 -> S1`：收到问询后默认先答  
- `S1 -> S2`：用户愿意补充信息  
- `S1 -> S4`：用户继续问，不补信息  
- `S1 -> S6`：用户单轮结束  
- `S2 -> S3`：补全达到可个性化阈值（任意 2-3 槽位）  
- `S2 -> S4`：补全不足但继续问答  
- `S2 -> S6`：用户中止  
- `S3 -> S4`：继续追问  
- `S3 -> S5`：进入补查反馈  
- `S4 -> S3`：问答中补齐关键信息  
- `S4 -> S5`：进入复盘  
- `S4 -> S6`：结束  
- `S5 -> S4`：复盘后继续  
- `S5 -> S6`：复盘后收束  
- `S6 -> S0`：新问题开启新轮

---

## 4. 证据时效规则（必遵）

| 时效类型 | 适用场景 | 必须动作 |
|---------|----------|----------|
| 实时 | 股价、新闻、政策、赛事 | 建议在线检索，不得用离线知识冒充 |
| 近期（30天内） | 行业动态、近期研究 | 标注时效，建议补查 |
| 历史（30天前） | 历史事件、经典理论 | 可离线回答，标注时效 |
| 无时效要求 | 常识、定义、原理 | 可离线回答 |

---

## 5. 其它垂类复用指引

把本罗盘抽象成通用骨架，仅替换三层配置：
- `domainEvidenceModel`：证据体系（搜索=检索结果/知识边界）
- `domainSlotSchema`：可选槽位
- `domainActionPlan`：补查建议模板
