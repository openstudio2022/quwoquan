# 轮次状态机

## S0_意图识别
- 识别用户意图，判定问题层级（L1-L4）与子类型。
- 同时判定服务阶段（售前/售中/售后/通用）。
- 若意图模糊（如"华为云怎么样"），进入 S1 补全具体诉求。

## S1_槽位补全
- 关键槽位：product（目标产品）、phase（服务阶段）、level（问题层级）、subtype（子类型）。
- 优先从问句与历史对话提取；缺失时 ask_user，问题简短且指向明确。
- 对比场景额外需要 competitor（对比方产品）槽位。
- 故障场景额外需要 symptom（现象描述）与 errorCode（报错码，可选）槽位。

## S2_知识检索
- 按问题层级执行差异化检索策略。
- L1：site:huaweicloud.com + 产品名，3-5 片段。
- L2：多产品/多文档，8-15 片段；对比场景检索双方。
- L3：定价/案例/趋势，5-10 片段。
- L4：方案/架构/最佳实践，10-15 片段。
- 首轮检索不足时，自动扩大检索范围。

## S3_推理整合
- L1：直接提取事实。
- L2：多源关联推理，构建对比矩阵或诊断链。
- L3：基于检索数据 + 模式推断。
- L4：在事实基础上生成方案。
- 竞品对比：客观呈现各方优势，单独说明华为云差异化。

## S4_结果渲染
- 按服务阶段选择对应 Markdown 卡片模板。
- 机器轨与用户轨必须同时可用。
- 参考链接完整不截断。
- 追问建议与当前话题紧密关联。
