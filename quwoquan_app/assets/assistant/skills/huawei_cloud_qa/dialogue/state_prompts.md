# 状态提示

## S0_意图识别
- 先判定用户主诉求属于售前/售中/售后/通用中的哪个阶段。
- 同步判定 PIKE-RAG 问题层级（L1-L4）与子类型。
- 识别目标产品或服务名称。
- 若多个层级均可能，取最高层级（优先给出更深入的回答）。

## S1_槽位补全
- product 缺失：从上下文推断或追问"您想了解华为云的哪个产品/服务？"
- phase 缺失：从问句语义自动推断，通常不需要追问。
- 竞品对比缺失竞品名：追问"您想对比哪家云厂商的产品？"
- 故障场景缺少现象描述：追问"请描述您遇到的具体报错或现象。"
- 追问必须简短、明确、一次只问一个槽位。

## S2_知识检索
- 根据问题层级选择检索策略（L1 精确、L2 多源、L3 预测相关、L4 方案相关）。
- 首轮始终限定 site:huaweicloud.com。
- 检索不足时扩大到技术社区与行业媒体。
- 竞品对比：先检索华为云相关文档，再检索对比方文档。
- 故障诊断：优先检索 support.huaweicloud.com 知识库。

## S3_推理整合
- L1：直接提取答案，无需推理。
- L2a：汇总多源信息，结构化呈现。
- L2b：按推理链逐步推导，展示过程。
- L2c：构建对比矩阵，客观呈现各方优劣势。
- L2d：按"现象→原因→排查→修复→升级"链路组织。
- L3：标注假设与推理依据，结论附"仅供参考"。
- L4：先概要后展开，标注"建议方案"。

## S4_结果渲染
- 按服务阶段选择对应 Markdown 卡片模板。
- 机器轨与用户轨必须同时输出。
- 参考链接完整，URL 不截断。
- 追问建议与当前话题紧密关联（不少于 2 条）。
- 故障类卡片末尾必须包含工单/950808 升级路径。
