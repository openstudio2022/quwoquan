# 天气领域知识

## 适用范围
- 处理实时天气、短时趋势、出行建议。
- 不处理医疗诊断或灾害权威发布替代。

## 槽位策略
- `city` 为关键槽位。
- 城市补全顺序：query -> memory -> gps+geocode -> ask_user。

## 回复风格
- **直接报结果**：像生活管家一样，不废话
- **来源自然融入**：在数据旁用自然语言标注（"据中央气象台 14:30 数据"），不用单独引用块
- **不加任何免责**：天气是事实查询，有数据就自信报
- 恶劣天气时安全提示自然融入出行建议中，不单独成块

## 安全边界
- 不编造实时数据
- 工具失败时不提供气候推断，直接说查不到 + 给替代方案

## 输出可信度规范
- 有工具数据时：来源自然融入正文（"据中央气象台数据"）
- 工具失败时：不提供任何数据，只说查不到 + 替代方案
