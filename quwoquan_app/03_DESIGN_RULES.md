# 趣我圈App 设计规则

## 📋 文档概述

### 基本信息
- **项目名称**: 趣我圈 (QuWoQuan)
- **文档版本**: v1.0
- **创建日期**: 2024年12月19日
- **适用范围**: 趣我圈App端侧UI/UX设计
- **设计风格**: Instagram风格 + 中国社交功能

### 设计原则
- **一致性**: 保持设计语言的一致性
- **可用性**: 确保良好的用户体验
- **可访问性**: 支持无障碍访问
- **响应式**: 适配多设备和屏幕尺寸
- **性能**: 优化视觉性能

#### 1.3 颜色使用规范与混用避免规则
```yaml
主品牌色使用规则 (趣我圈蓝 #405DE6):
  必须使用场景:
    - 主要操作按钮 (登录、发布、确认等)
    - 品牌Logo和标识
    - 导航栏激活状态
    - 链接文字和可点击元素
    - 进度条和加载指示器
    - 表单焦点状态

  使用限制:
    - 单页面主品牌色使用不超过3处
    - 避免大面积背景使用
    - 避免与其他主色调同时出现
    - 保持品牌一致性

  禁止使用场景:
    - 与辅助品牌色同时使用
    - 与特殊装饰色混用
    - 作为大面积背景色
    - 在错误或警告场景中使用

辅助品牌色使用规则 (渐变蓝 #5851DB):
  允许使用场景:
    - 次要操作按钮
    - 图标和装饰元素
    - 标签和徽章
    - 辅助信息展示

  使用限制:
    - 不与主品牌色同时出现
    - 使用频率低于主品牌色
    - 避免与功能色冲突

特殊装饰色使用规则 (紫色、粉色、红色、橙色、黄色):
  严格限制使用:
    - 仅用于特殊节日主题
    - 仅用于个性化定制
    - 仅用于特殊功能标识
    - 需要设计总监审批

  使用禁令:
    - 禁止在常规界面使用
    - 禁止与品牌色混用
    - 禁止大面积使用
    - 禁止用于核心功能

颜色组合规则:
  推荐组合:
    - 主品牌色 + 中性色 (推荐)
    - 辅助品牌色 + 中性色 (允许)
    - 功能色 + 中性色 (按需)

  禁止组合:
    - 主品牌色 + 辅助品牌色 (禁止)
    - 主品牌色 + 特殊装饰色 (禁止)
    - 多个特殊装饰色混用 (禁止)
    - 功能色之间混用 (禁止)

品牌一致性检查:
  每页面检查清单:
    - [ ] 主品牌色使用不超过3处
    - [ ] 未使用特殊装饰色
    - [ ] 颜色组合符合规范
    - [ ] 品牌识别度保持
    - [ ] 色彩层级清晰

颜色使用示例:
  正确示例:
    登录页面:
      - 背景: 白色/浅灰色
      - 主按钮: 趣我圈蓝 (#405DE6)
      - 文字: 深灰色
      - 链接: 趣我圈蓝 (#405DE6)

    主页:
      - 导航栏: 趣我圈蓝 (#405DE6)
      - 内容区域: 白色背景
      - 次要按钮: 渐变蓝 (#5851DB)
      - 文字: 中性色系

    错误页面:
      - 背景: 白色
      - 错误提示: 错误红 (#FF4444)
      - 主按钮: 趣我圈蓝 (#405DE6)
      - 文字: 深灰色

  错误示例:
    避免混用:
      - ❌ 主按钮用紫色，链接用粉色
      - ❌ 同一页面出现多种装饰色
      - ❌ 品牌色与功能色混用
      - ❌ 大面积使用品牌色作为背景

设计检查工具:
  自动检查:
    - 颜色使用频率统计
```

#### 3.3 间距使用规范
```yaml
使用原则:
  语义优先: 优先使用语义化间距 (intraGroup/interGroup/container)
  响应式: 自动适配不同屏幕尺寸
  一致性: 相同语义使用相同间距等级
  层次性: 不同语义保持清晰的视觉层次
  
使用方式:
  自动适配:
    context.getIntraGroupSpacing('md')
    context.getInterGroupSpacing('lg')
    context.getContainerSpacing('sm')
    
  指定屏幕类型:
    AppSpacing.getSpacing('intraGroup', 'md', screenType: 'tablet')
    
  语义常量:
    AppSpacing.getSpacing(
      DesignSemanticConstants.intraGroup,
      DesignSemanticConstants.md,
      context: context,
    )
```
