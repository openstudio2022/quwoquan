# 如何获取Figma工作区文件URL（详细步骤）

## 🎯 目标

获取可以API访问的工作区文件URL（格式：`/file/FILE_KEY/...`）

## 📋 详细步骤

### 方法1：从Figma界面获取（最简单）

#### 步骤1：登录Figma
1. 打开 https://www.figma.com/
2. 登录你的账号

#### 步骤2：找到工作区中的文件
1. 在左侧边栏，找到你的**团队**或**个人**工作区
   - 团队工作区：显示团队名称
   - 个人工作区：显示 "Drafts" 或你的名字
2. 在工作区中查找 "趣我圈2026-126" 文件
3. **如果找不到文件**：
   - 检查是否真的复制成功
   - 尝试在搜索框中搜索文件名

#### 步骤3：打开文件并获取URL
1. **点击文件打开**
2. **查看浏览器地址栏**
3. **复制完整的URL**

正确的URL格式应该是：
```
https://www.figma.com/file/XXXXXXXXXXXXX/趣我圈2026-126
```

**注意**：
- ✅ 正确：`/file/` 
- ❌ 错误：`/make/`

#### 步骤4：提取文件ID
从URL中提取文件ID（`/file/` 后面的部分，到下一个 `/` 之前）

例如：
- URL: `https://www.figma.com/file/ABC123XYZ/趣我圈2026-126`
- 文件ID: `ABC123XYZ`

### 方法2：从文件菜单获取

1. **在Figma中打开文件**（工作区中的文件）
2. **点击右上角 "..." 菜单**
3. **选择 "Copy link" 或 "复制链接"**
4. **检查链接格式**
   - 应该是 `https://www.figma.com/file/...` 格式
   - 不是 `https://www.figma.com/make/...` 格式

### 方法3：从文件信息获取

1. **在Figma中打开文件**
2. **点击左上角的文件名称**
3. **查看文件信息面板**
   - 文件ID可能会显示在这里
   - 或者查看分享链接

## 🔍 如何确认文件在工作区

### 检查清单：

- [ ] 文件显示在你的团队或个人工作区中
- [ ] 你是文件的所有者或团队成员
- [ ] URL格式是 `/file/` 而不是 `/make/`
- [ ] 可以通过API访问（测试时会成功）

## ⚠️ 常见问题

### Q1: 文件复制后找不到？

**A**: 
1. 刷新Figma页面
2. 检查是否复制到了正确的工作区
3. 尝试重新复制文件

### Q2: URL仍然是 `/make/` 格式？

**A**: 
- 说明文件可能还在社区中
- 需要确认复制操作是否成功
- 尝试从工作区重新打开文件

### Q3: 如何确认复制成功？

**A**:
1. 在工作区中能看到文件
2. 打开文件后，URL是 `/file/` 格式
3. 文件显示你的名字作为所有者

## 📝 获取到URL后

### 使用脚本提取文件ID：

```bash
node scripts/extract_figma_file_id.js "你的工作区文件URL"
```

### 或手动更新配置：

编辑 `.env` 文件：
```bash
FIGMA_FILE_KEY=新的文件ID
```

### 然后测试：

```bash
npm run sync:figma:enhanced
```

## 💡 提示

如果文件已经复制到工作区，但URL仍然是 `/make/` 格式：

1. **确认你在查看工作区文件**
   - 不是社区文件的链接
   - 是从工作区中打开的

2. **尝试重新打开**
   - 关闭当前标签页
   - 从工作区重新打开文件

3. **检查文件位置**
   - 确认文件在正确的工作区
   - 不是仍在社区中

---

**关键步骤**：从Figma工作区中打开文件，然后复制浏览器地址栏中的URL（必须是 `/file/` 格式）！
