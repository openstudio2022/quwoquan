## 2. 里程碑 2 模板：路由与并行规划

### 目标

让第一阶段支持多 skill 路由、并行叙事和一次性澄清。

### 任务清单

- [ ] 冻结 `skill_route` 合同
- [ ] 接入目录树摘要输入
- [ ] 支持一次返回多个 skill
- [ ] 支持 primary / supporting 关系
- [ ] 支持一次性澄清列表
- [ ] 支持 fallback skill
- [ ] 输出并行处理叙事
- [ ] 为每个 skill 生成 `taskBrief`
- [ ] 为里程碑 3 产出每个 skill 的局部上下文种子
- [ ] 确认 `selectedTargets` 可直接进入子 agent 阶段

### 验收

- 一个问题可路由到 1 到 n 个 skill
- 澄清只在必要时发生
- 用户可看到并行处理说明
- 路由阶段输出足以直接进入里程碑 3

---

## 3. 里程碑 3 模板：并行 subagent 执行

### 目标

每个 skill 独立执行检索、工具调用和局部总结。

### 任务清单

- [ ] 冻结 `skill_subagent_turn` 合同
- [ ] 为每个 skill 独立生成检索设计
- [ ] 为每个 skill 独立执行工具调用
- [ ] 为每个 skill 独立筛选证据
- [ ] 为每个 skill 独立生成局部总结
- [ ] 支持失败 / 挂起 / 恢复
- [ ] 明确每个 subagent 的独立输入、独立输出、独立预算
- [ ] 明确每个 subagent 的挂起条件和恢复条件

### 验收

- 子代理之间互不污染
- 一个 skill 的失败不影响其他 skill
- 支持部分完成
- 每个 subagent 的输出可直接进入里程碑 4 汇总

## 6. 里程碑 6 模板：最终集成验收

### 目标

把里程碑 2 到 5 串成一条完整、可回放、可验收的主线。

### 任务清单

- [ ] 冻结 M6 最终集成验收定义
- [ ] 对齐 `skill_route`、并行 subagent、`skill_synthesis`、`Finalize` 的端到端数据流
- [ ] 明确连续叙事、局部完成、澄清补全、重新汇总的收口规则
- [ ] 明确跨轮接续与历史不污染的验收边界
- [ ] 补齐一条完整回放路径的最小测试矩阵
- [ ] 标记必须过门禁的回归测试与可选回归测试
- [ ] 整理回滚条件与失败降级策略

### 验收

- 一条完整问题链可以稳定完成路由、并行执行、汇总、收口
- 局部失败不会把其他 skill 一起拖死
- 用户补全后可以继续挂起的局部 skill 并重新统一成答
- 最终答案、过程叙事、历史保存三者一致
