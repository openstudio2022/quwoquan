# L2 规格：圈子协作工具

## 背景与动机

当前圈子仅支持浏览内容和简单互动（点赞/评论），缺少深度协作工具。用户在圈子中无法便捷地：
- 共享文件资料（教程、资源、模板等）。
- 通过即时消息快速沟通。
- 在圈子空间内直接创作发布。

本 L2 目标：为圈子增加存储空间、群聊集成、作品发布区三大协作工具，使圈子成为完整的社区协作空间。

## 目标用户

- 圈子成员：上传/下载共享资料，参与群聊讨论，在圈内发布作品。
- 圈主/管理员：管理存储空间权限，配置群聊设置，审核圈内发布。
- 访客：浏览存储空间（公开文件）和作品发布区，了解圈子活跃度。

## 功能范围

### L3: circle-storage（存储空间）
- 每个圈子拥有独立的云存储空间（对接 S3-compatible 对象存储）。
- 基础 CRUD：创建文件夹、上传文件（≤50MB 单文件）、下载、删除、重命名。
- 文件列表支持按类型/时间/大小排序，支持搜索。
- 容量限制：按圈子等级配置（基础 1GB、进阶 5GB、专业 20GB），超限提示。
- 权限模型：圈主完全控制、管理员可管理（增删改）、成员可上传下载、访客只读。
- 文件元数据存 MongoDB（CircleFile 实体），文件本体存对象存储。

### L3: circle-group-chat（群聊集成）
- 圈子可绑定一个或多个群聊（利用已有 chat 域 Conversation 实体，type=circle）。
- 加入圈子 → 自动加入默认群聊（CircleMemberJoined 事件 → chat-service 消费）。
- 退出圈子 → 自动退出群聊（CircleMemberLeft 事件 → chat-service 消费），可配置关闭。
- 群聊自定义：修改群名、群头像、群公告、@全体成员、消息置顶。
- 圈子详情页显示群聊入口卡片（最近消息预览 + 未读计数）。

### L3: circle-publishing-zone（作品发布区）
- 圈子内独立的内容 feed，聚合该圈子内发布的所有内容（图文/视频/文章）。
- 支持瀑布流和列表视图切换。
- 发布入口：圈子详情页 FAB → 跳转创作页（自动填充圈子 ID）。
- 内容排序：最新/最热/精选（圈主可置顶）。
- 复用已有 content 域的 PostBaseDto 体系，通过 circleId 关联筛选。

## 不做什么（Out of Scope）

- 不做在线文档编辑（Office/Markdown 在线协作）。
- 不做文件版本管理。
- 不做视频/音频在线播放预览（仅缩略图 + 下载）。
- 不实现 IM 协议层——依赖已有 chat 域能力。
- 不做多群聊管理 UI（v1 仅支持一个默认群聊）。

## 约束

- 存储空间实体（CircleFile）必须先在 contracts/metadata/social/circle/ 定义 fields.yaml，经 codegen 生成。
- 群聊集成通过事件驱动（非同步 RPC），保证松耦合。
- 发布区复用 content 域的内容模型，不新建内容实体——通过 circleId 字段关联。
- 文件上传必须有大小校验和类型白名单（防恶意文件）。

## 验收重点

- A5（L1）：存储空间基础 CRUD + 权限控制。
- A6（L1）：群聊集成可用，成员自动同步。
