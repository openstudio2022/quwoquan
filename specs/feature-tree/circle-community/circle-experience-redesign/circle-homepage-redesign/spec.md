# L3 规格：群组详情模板与圈子主页基线

## 背景与动机

当前 `CircleDetailPage` 只适合单一兴趣圈子场景，已经无法承接新的群组基线：

1. **全局入口已统一为群组**：详情页需要同时承接兴趣型圈子与组织型主页，而不是只服务“圈子”一种类型。
2. **主页目标不清**：当前详情页更像内容列表，无法同时回答“这是什么地方、我该去哪里、我能做什么、我属于哪里”。
3. **群与组织节点语义缺失**：现有详情页只把群看成聊天入口，无法承接公共群、自建群、院系、班级、部门等子单元。
4. **内容模型割裂**：详情页仍按旧的创作语义组织，不能和新的 `发布内容 + 笔记/作品/提问/口碑` 基线对齐。

本 L3 的目标，是冻结一套共享的详情基础壳层，并在其上给出 `通用圈子模板` 与 `组织主页模板` 两套落地规格。

## 目标用户

- **访客 / 成员**：快速判断这个群组是什么、是否要加入、先看什么内容、加入哪个群或组织节点。
- **圈主 / 圈管**：管理兴趣型圈子的首页结构、内容秩序、公共群与成员。
- **负责人 / 管理员**：管理学校、院系、班级、公司、部门等组织主页与节点结构。
- **群主 / 群管**：管理单个群或组织节点的交流、资料与公告。

## 功能范围

### F1：共享详情基础壳层

- 所有群组详情页共享统一头部信息区：名称、简介、类型徽章、成员数、内容数、群数或节点数。
- 所有群组详情页共享主操作区：`加入 / 关注 / 发布 / 进入群或组织 / 管理`。
- 所有群组首页共享模块化布局：公告、内容精选、子单元入口、成员摘要、可选的具体事物摘要卡。
- 所有首页模块都必须独立加载和独立降级，单模块失败不影响其他模块使用。

### F2：通用圈子模板

- 页签冻结为 `首页 / 内容 / 群 / 成员`。
- `首页` 默认模块顺序：头部信息 -> 公告与圈规 -> 我的群 / 公共群入口 -> 内容精选 -> 可选的具体事物摘要卡 -> 活跃成员。
- `内容` 承接群组层公开内容流，支持 `笔记 / 作品 / 提问 / 口碑` 筛选。
- `群` 承接 `我的群 / 公共群 / 自建群 / 推荐群`。
- `成员` 承接圈主、圈管、活跃成员、新成员与全部成员。

### F3：组织主页模板

- 页签冻结为 `首页 / 内容 / 组织 / 成员`。
- 前台直接显示具体名称，如学校、院系、班级、公司、部门，不强制显示“圈子”。
- `首页` 默认模块顺序：头部信息 -> 官方公告 -> 找到我的组织 -> 组织结构入口 -> 内容精选 -> 交流/资料快捷入口 -> 关键成员。
- `组织` 页承接 `我所在的组织 / 组织结构树 / 组织节点 / 公共交流群 / 兴趣小组`。
- `成员` 页承接负责人、管理员、节点负责人、同级成员与全部成员。

### F4：二级页面模式

- `通用群主页` 使用 `首页 / 交流 / 资料 / 成员`。
- `组织节点主页` 使用 `首页 / 内容 / 交流 / 资料 / 成员`。
- 组织节点（班级、院系、部门）底层可复用群能力，但前台优先表现为节点主页，而不是普通聊天群页。
- 群内能力至少包含 `交流 / 资料 / 公告`，聊天只属于 `交流` 页的一部分。

### F5：内容与发布模型

- 详情页层的统一动作叫 `发布内容`。
- 一级内容类型冻结为 `笔记 / 作品 / 提问 / 口碑`。
- 基础表达形式冻结为 `图文笔记 / 视频 / 文章`，总添加入口仍以 `相册 / 视频 / 长文` 作为开始方式。
- 全局入口创作完成后可选择发布到群组；群组内入口进入时，当前群组默认已选中，但使用同一发布页。
- `口碑` 必须绑定 1 个主具体事物；`笔记 / 作品 / 提问` 可绑定也可不绑定。

### F6：角色标签与操作语义

- 通用圈子模板详情页显示 `圈主 / 圈管`。
- 组织主页模板详情页显示 `负责人 / 管理员`。
- 群或组织节点详情页显示 `群主 / 群管`，组织节点可按具体场景映射为 `节点负责人 / 节点管理员`。
- 圈级或组织级角色拥有上位治理权，但不自动等于所有群或节点的群主。

### F7：具体事物摘要集成

- 群组详情页只消费具体事物的摘要，不拥有主档。
- 通用圈子模板可显示 `关联具体事物摘要卡` 与 `口碑精选`。
- 组织主页模板默认不突出具体事物，除非该主页本身承接明确的具体事物业务。
- 前台永远显示具体类目名，如 `关联学校 / 关联车型 / 关联酒店 / 关联景点`，不显示“实体”字样。

## 不做什么（Out of Scope）

- 不在本 L3 内冻结推荐算法和推荐训练链路。
- 不在本 L3 内定义具体事物的抓取、归一、展示和商业承接模型。
- 不在本 L3 内冻结复杂拖拽编排、实时推送与在线协作。
- 不在本 L3 内把群层升级为公开内容主时间线。

## 约束

- 全局入口层必须使用 `群组` 作为用户词。
- `圈子` 只作为兴趣型详情页名称，不再承担全局总称职责。
- 组织型详情页必须尊重学校、院系、班级、公司、部门等正式组织语义。
- 公开内容主分发面必须在群组层；群层以交流、资料、公告为主。
- 群组详情页的模板差异主要体现在第三个页签和首页模块权重，不得分裂为两套完全不同的产品。

## 对标输入与吸收结论

- 微信：吸收“一个入口下容纳不同关系对象”的全局认知收口方式。
- 小红书：吸收“统一发布，再按具体对象挂载”的内容模型，不吸收把所有详情页都做成同一种社区页。
- Discord / 豆瓣小组：吸收群内协作与子单元组织能力，但不直接照搬频道体系与命名。

## 验收重点

- A1：共享详情基础壳层成立，模块可独立加载和降级。
- A2：通用圈子模板可用。
- A3：组织主页模板可用。
- A4：通用群主页与组织节点主页区分成立。
- A5：详情页内容与发布模型对齐到 `发布内容 + 笔记/作品/提问/口碑`。
- A6：圈级、组织级与群级角色标签和治理边界清晰。
- A7：群组与具体事物的摘要集成边界冻结。
