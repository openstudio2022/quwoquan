# call-lifecycle-contract 设计

> status: specified

## 目标

固化 1v1 通话的状态机和超时规则，让发起、响铃、接听、挂断、拒绝在端云之间保持单一生命周期语义。

## 关键决策

- `CallSession` 状态机固定为 `INITIATED -> RINGING -> CONNECTING -> IN_CALL -> ENDED`，拒绝和超时都归并到结束态但保留原因字段。
- 发起、接听、拒绝、挂断由服务端串行推进状态，客户端不直接跳态，只提交动作意图并订阅回写结果。
- 无应答 30 秒超时由服务端定时治理触发，避免客户端离线导致状态悬挂。
- 通话结束后必须落会话记录与消息侧摘要事件，保证聊天列表、通话历史和系统通知看到同一结果。

## 验证策略

- T1：契约测试覆盖 Initiate/Answer/Reject/Hangup 与超时结束。
- T3：端云联调用例覆盖 1v1 音视频完整旅程和异常恢复。
