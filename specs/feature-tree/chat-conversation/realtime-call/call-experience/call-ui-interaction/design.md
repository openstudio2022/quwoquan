# call-ui-interaction 设计

> status: specified

## 目标

把通话界面的交互职责收敛为可实现的 UI 设计边界，确保不同通话页面、布局模式与系统能力接入遵循同一套状态来源和切换规则。

## 关键决策

- 页面入口固定为 `OutgoingCallPage`、`IncomingCallPage`、`VoiceCallPage`、`VideoCallPage`，状态统一来自实时通话会话模型，页面本身不持有第二套真相源。
- 多人布局优先按参与人数分段切换网格，发言人高亮由音量/活跃说话者信号驱动，切换延迟目标控制在 1 秒内。
- 控制栏统一承载静音、摄像头、翻转、邀请、扬声器、挂断 6 个动作，语音/视频差异只通过可见性裁剪，不分叉交互协议。
- PiP 与来电系统 UI 视为同一条会话生命周期上的展示层能力，进入后台、返回前台、系统来电拉起都必须复用现有会话状态。

## 验证策略

- T2：Widget/Journey 用例覆盖 7 种网格、演讲者视图、控制栏动作与 PiP 往返。
- T4：真机验证 CallKit / FullScreen Intent 拉起与恢复链路。
