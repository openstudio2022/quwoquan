# 语音消息（Voice Message）

> **层级**：L3_subfeature（隶属 L2 `list-detail-message-delivery`，L1 `chat-conversation`）
> **状态**：specified
> **依赖**：`runtime/runtime-media`（统一媒体运行时）、`gateway-orchestrator-foundation/realtime-gateway`（实时推送，Phase 2）

## 背景与动机

趣聊当前支持文本消息和占位式图片/视频消息，但不支持语音消息。现有的"语音"功能仅做 ASR（speech-to-text）将语音转为文字后以 `type=text` 发送，不保留音频文件。

语音消息是即时通讯的核心功能之一（微信日均语音消息量超过文字消息），尤其在移动端，语音输入效率远高于文字。缺乏语音消息严重限制了趣聊的通讯能力和用户体验。

本特性实现语音消息的端到端闭环：录音 → 上传 → 发送 → 云端存储 → 推送 → 接收方展示 → 点击播放。参考微信的交互模式，覆盖弱网、离线、并发等边界场景。

## 目标用户

- **趣聊用户**：所有使用趣聊聊天功能的用户（1v1 私聊和群聊）
- **AI 助手对话**：未来可扩展为助手语音回复（当前 Phase 不实现）

## 功能范围

### 核心功能

1. **录音**：按住录音按钮开始录音，松手发送；AAC (m4a) 编码，采样率 16kHz，最大时长 120 秒（可配置）
2. **录音交互**：录音中实时展示音量/波形、时长计时器、上滑取消手势
3. **上传**：录音完成后通过 `runtime/media` 上传音频文件到 OSS，获取 CDN URL
4. **发送**：上传完成后调用 `sendMessage(type: 'audio', media: {...})`，乐观插入本地消息列表
5. **消息模型扩展**：`MessageType` 枚举新增 `audio`；`Message` 实体新增 `media` 结构化字段（url, mimeType, durationMs, waveform, fileSizeBytes, codec）
6. **云端存储**：MongoDB 存储消息（含 media 字段），与现有消息同表
7. **消息推送/同步**：Phase 1 通过 `syncMessages` 轮询；Phase 2 通过 WebSocket 实时推送
8. **语音消息气泡**：独立的语音消息气泡 UI——播放/暂停按钮、波形可视化、时长显示、已读/未读标记
9. **播放**：点击气泡播放语音，支持暂停/恢复、进度条、扬声器/听筒切换
10. **本地缓存**：已播放的语音自动缓存到本地，再次播放无需重新下载
11. **离线语音**：断网时可录音，保存到本地队列，网络恢复后自动上传并发送

### 消息事件扩展

12. **MessageSent 事件**：`payload_fields` 新增 `mediaUrl`、`media` 字段，下游推送/通知可展示「[语音消息 0:05]」

## 不做什么（Out of Scope）

- **语音转文字（ASR）**：不实现发送时的 ASR 转文本附加功能——未来由供应商提供
- **变速播放**：不实现 0.5x/1.5x/2x 播放速度——后续增强
- **语音消息转发**：不实现将语音消息转发到其他会话——后续增强
- **连续播放**：不实现播完当前语音自动播放下一条——后续增强
- **视频消息**：不在本 Phase 实现——属于 `rich-media-message` 后续 L3
- **多图消息**：不在本 Phase 实现——属于 `rich-media-message` 后续 L3
- **文件消息**：不在本 Phase 实现——属于 `rich-media-message` 后续 L3
- **AI 助手语音回复**：不在本 Phase 实现
- **WebRTC 实时通话**：属于独立特性，不在本 Phase

## 约束

### 技术约束

- 录音库必须支持 AAC 编码、iOS/Android 双端，推荐 `record` 包
- 播放库必须支持流式播放（边下边播）、进度回调，推荐 `just_audio`
- 上传必须通过 `runtime/media.MediaStore`（`CategoryMessaging`），禁止直接调用 OSS
- 发送必须通过 `ChatRepository.sendMessage`，禁止绕过消息链路
- 语音气泡 UI 必须使用 `AppTypography`/`AppSpacing`/`AppColors`，禁止硬编码
- metadata 变更必须走 `metadata → verify → codegen` 流程
- 离线队列使用 Hive 持久化，不依赖网络

### 业务约束

- 最短录音时长 1 秒（低于 1 秒不发送，提示"录音时间太短"）
- 最大录音时长 120 秒（可配置），到达上限自动停止并发送
- 音频文件大小上限 5MB（120s AAC 约 2-3MB，余量充足）
- 波形数据由客户端录音时实时采集，存入 `media.waveform`（float 数组，约 50-100 采样点）

### 弱网与性能约束

| 场景 | 要求 |
|------|------|
| 强网（4G/WiFi） | 录音完成到对方可播放 ≤ 3 秒（含上传+发送+推送+下载） |
| 弱网（2G/100kbps） | 上传自动重试 3 次，指数退避；UI 显示上传进度 |
| 断网 | 录音保存到离线队列，UI 显示 ⏳ 待发送状态 |
| 网络恢复 | 自动从离线队列恢复上传，按 FIFO 顺序 |
| 播放弱网 | 支持流式播放（边下边播），下载超时 30 秒显示重试 |
| 并发录音 | 同时只允许一个录音任务，新录音覆盖旧录音 |
| 并发上传 | 同时最多 3 个语音上传任务（复用 MediaUploadManager 队列） |
| 并发播放 | 同时只允许一个语音播放，点击新语音自动停止旧语音 |

### 实时性约束

| 指标 | Phase 1（HTTP 轮询） | Phase 2（WebSocket） |
|------|---------------------|---------------------|
| 发送方感知延迟 | ≤ 2 秒（上传+发送） | ≤ 2 秒 |
| 接收方感知延迟 | ≤ 8 秒（含 5s 轮询间隔） | ≤ 1 秒 |
| 播放起播延迟 | ≤ 1 秒（强网/缓存命中） | ≤ 1 秒 |
| 播放起播延迟 | ≤ 5 秒（弱网/首次下载） | ≤ 3 秒 |

### 并发性能约束

| 指标 | 要求 |
|------|------|
| 云侧 SendMessage TPS | ≥ 1000/s（含 audio 类型） |
| 云侧 media InitUpload TPS | ≥ 500/s |
| 端侧离线队列容量 | ≥ 100 条未发送语音 |
| 本地缓存容量 | 默认 200MB（LRU 淘汰） |
| 消息列表含语音气泡渲染 | 列表滚动 60fps 无卡顿 |

### 部署约束

- 端侧通过 App Store / Google Play 发布，支持 iOS 15+ / Android API 26+
- 云侧 chat-service 已有部署拓扑，无需新增进程
- OSS bucket + CDN 域名按环境配置（dev/integration/prod）
- 灰度发布策略：先 integration 全量 → prod 10% → 50% → 100%

## 适用范围与约束

- **适用场景**：趣聊 1v1 私聊和群聊中发送/接收/播放语音消息
- **前置条件**：`runtime/media` MediaStore 接口已实现、端侧麦克风权限可获取
- **不适用**：AI 助手会话中的语音回复（需另行设计 TTS 链路）、实时语音/视频通话（属于 WebRTC 独立特性）

## 对标输入与吸收结论

| 对标 | 借鉴点 | 不借鉴点 | 适用边界 |
|------|--------|---------|---------|
| **微信** | 按住录音/松手发送/上滑取消交互范式；气泡宽度与时长正比；听筒/扬声器切换；最大 60s | 60s 上限（我们用 120s）；AMR/Silk 编码（我们用 AAC 兼容性更好） | 交互范式成熟度高，直接对标 |
| **Telegram** | 播放速度调节（0.5x/1.5x/2x）；圆形波形动画 | 语音消息可转发；圆形气泡设计（与趣聊气泡风格不一致） | 变速播放作为后续增强 |
| **Discord** | 波形动画展示；压缩+清晰度选项 | 基于 Opus 编码（兼容性不如 AAC） | 波形动画参考其视觉设计 |
| **飞书/钉钉** | 语音转文字附加功能；企业安全合规 | 过重的企业功能 | ASR 转文字作为后续由供应商提供 |

## 验收重点

### 端到端闭环（T4 旅程层）

- A1: 录音→上传→发送→接收→播放端到端闭环成功
- A2: 弱网/断网场景下语音消息可靠性

### 消息模型（T1 契约层）

- A3: MessageType 枚举含 audio，Message.media 结构化字段 codegen 通过
- A4: MessageSent 事件 payload 含 mediaUrl 和 media

### 录音与发送（T2 模块层）

- A5: 录音引擎 AAC 编码 + 波形提取 + 时长限制
- A6: 上传通过 MediaUploadManager，支持重试和离线队列

### 播放与缓存（T2 模块层）

- A7: 播放引擎支持流式播放、暂停/恢复、进度
- A8: 本地缓存 LRU 策略有效

### UI 渲染（T2 模块层）

- A9: 语音消息气泡渲染正确（波形、时长、播放状态）

### 性能与可靠性（T3 集成层 + T4 旅程层）

- A10: 强网端到端延迟 ≤ 3 秒
- A11: 弱网上传重试 + 离线队列恢复

### 部署灰度（T4 旅程层）

- A12: integration 全量验证 → prod 灰度 10%→50%→100% 无回滚

详细验收标准见 `acceptance.yaml`。
