# 视频消息端到端（Video Message E2E）

> **层级**：L4_story（隶属 L3 `rich-media-message`）
> **状态**：specified

## 功能说明

实现视频消息从选择/录制到对方接收播放的端到端闭环：

```
选择/录制视频 → 压缩(720p) → 提取封面 → 上传(OSS) → 发送(type=video)
  → 云端存储 → 推送/同步 → 接收方展示(缩略图气泡) → 点击全屏播放
```

### 发送链路

1. 从相册选择视频或调用相机录制
2. 端侧自动压缩至 720p/30fps/H.264/mp4（可选"原画质量"）
3. 提取首帧作为封面缩略图（JPEG, ≤200KB）
4. 封面图 + 视频文件分别通过 `MediaUploadManager` 上传
5. 上传完成后调用 `sendMessage(type='video', media={...})`
6. 本地乐观展示缩略图气泡 + 上传圆环进度

### 接收链路

7. 接收方通过 `syncMessages` 或 WebSocket 收到视频消息
8. 气泡展示缩略图（thumbnailUrl 独立 URL，快速加载）+ 时长角标 + 播放图标
9. 点击气泡：下载视频到本地缓存 → 打开全屏播放器

### 播放器

- 支持暂停/恢复、拖动进度条、横竖屏自动旋转
- 全屏模式，顶部返回按钮
- 复用发现域 `ImmersiveVideoViewer` 的设计模式

## 约束

- 视频大小 ≤100MB，时长 ≤10 分钟
- 压缩后格式统一为 mp4/H.264
- 非 WiFi 环境不自动下载视频（显示"点击下载"）
- 弱网入离线队列，网络恢复后自动重传
- 视频缓存独立 LRU 池 500MB

## 适用范围与约束

- **适用**：趣聊 1v1 私聊和群聊中发送/接收视频消息
- **不适用**：视频编辑（裁剪/滤镜）、实时视频通话、超长视频（>10min）
