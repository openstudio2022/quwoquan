# 语音播放与缓存（Voice Playback and Cache）

> **层级**：L4_story（隶属 L3 `voice-message`）
> **状态**：specified
> **依赖**：`voice-record-and-send`（消息模型已扩展）、`runtime-media/media-upload-and-storage`（CDN 下载）

## 背景与动机

语音消息端到端闭环的"接收侧"Story：从接收方看到语音消息气泡，到点击播放、听到完整语音。覆盖消息气泡 UI 渲染、音频下载与缓存、播放引擎、设备管理四个关键环节。

## 目标用户

- 趣聊用户（接收/播放语音消息的一方）

## 功能范围

### 语音消息气泡 UI

1. `type=audio` 消息渲染为独立语音气泡
2. 气泡组成：播放/暂停按钮（左侧）+ 波形可视化（中部）+ 时长显示 mm:ss（右侧）
3. 气泡宽度与时长正比：1s 最短（约 25% 屏宽）→ 120s 最长（约 70% 屏宽），线性插值
4. 播放状态视觉反馈：未播放（波形灰色）、播放中（波形动画 + 进度色彩填充）、已播放（波形主题色）
5. 未读语音标记：接收的语音消息未播放时显示红点标记
6. 发送中/失败状态：sending 显示上传进度环，failed 显示重试按钮
7. 所有视觉 token 使用 `AppSpacing`/`AppTypography`/`AppColors`

### 音频下载与缓存

8. 点击播放时，通过 `MediaDownloadCache` 下载音频文件
9. 已缓存文件直接从本地播放，不重新下载
10. 缓存策略：LRU，默认上限 200MB，超限淘汰最旧文件
11. 缓存目录使用 `getApplicationCacheDirectory()`，系统可清理
12. 下载进度展示（弱网时可见）

### 播放引擎

13. 集成 `just_audio` 包
14. 支持流式播放（边下边播，无需完全下载）
15. 播放/暂停/停止控制
16. 播放进度回调（100ms 精度），驱动波形进度动画
17. 同时只允许一个语音播放（点击新语音自动停止旧语音并重置进度）
18. 播放完成后自动重置进度到 0:00

### 设备管理

19. 集成 `audio_session` 包
20. 默认使用扬声器播放
21. 支持扬声器/听筒切换（贴耳自动切换听筒——通过距离传感器，后续增强）
22. 来电/音频中断时自动暂停播放，中断恢复后不自动续播

## 不做什么（Out of Scope）

- 变速播放（0.5x/1.5x/2x）——后续增强
- 连续播放（播完自动播下一条）——后续增强
- 语音转文字显示——后续增强
- 贴耳自动切换听筒——后续增强（需距离传感器）
- 蓝牙音频路由——后续增强

## 约束

### 弱网约束

| 场景 | 行为 |
|------|------|
| 强网 | 点击即播（流式播放，起播 ≤1s） |
| 弱网（<200kbps） | 显示下载进度条，流式播放（边下边播） |
| 极弱网（<50kbps） | 下载超时 30s 后显示"网络不佳，点击重试" |
| 断网 | 未缓存的语音显示"无法播放，请检查网络"；已缓存的正常播放 |

### 性能约束

| 指标 | 要求 |
|------|------|
| 起播延迟（缓存命中） | ≤ 200ms |
| 起播延迟（强网首次） | ≤ 1 秒 |
| 起播延迟（弱网首次） | ≤ 5 秒 |
| 气泡渲染 | 列表滚动 60fps 无卡顿（波形为静态 CustomPaint，不在滚动时动画） |
| 内存 | 播放器缓冲区 ≤ 2MB |
| 缓存 IO | 缓存写入不阻塞主线程 |

### 并发约束

- 同一时刻仅一个语音播放实例
- 下载与播放可并行（流式播放）
- 多个消息的下载可并行（最多 3 个，复用 MediaDownloadCache 队列）

## 适用范围与约束

- **适用**：趣聊 1v1 私聊和群聊中接收的语音消息播放
- **前置**：`voice-record-and-send` 已实现（消息模型含 media 字段）
- **不适用**：AI 助手 TTS 语音播放（需独立设计音频源和中断策略）

## 对标输入与吸收结论

| 对标 | 借鉴 | 不借鉴 |
|------|------|--------|
| 微信 | 气泡宽度正比时长；灰色→彩色播放进度；扬声器/听筒 | 红色未读圆点替换为主题色 |
| Telegram | 波形可视化美观；播放速度标签 | 圆形语音消息外观（与趣聊风格不一致） |
| Discord | 精细波形动画 | 过于复杂的波形渲染（性能成本） |
