# L3 规格：chat-list-ui-polish — 聊天列表 UI 打磨

> **层级**：L3_story（隶属 L2 `chat-experience-optimization`）
> **状态**：specified

## 0. 一句话定义

趣聊列表页 UI 体验对齐微信基线：Tab 文字真正居中、移除小趣助理会话条目、头像改为圆角方形、群聊采用九宫格组合头像。

## 1. 背景与动机

当前趣聊列表页存在四个体验断层：

1. **Tab 居中偏移**：`CenteredScrollableTabBar` 的 `trailingActions`（搜索+新建按钮）占据了右侧空间，导致"趣聊/同好"两个 Tab 文字的视觉中心被挤偏，不在页面真正中心。
2. **小趣助理冗余**：会话列表顶部置顶了小趣私人助理条目，但小趣已经在底部导航栏 C 位，无需在会话列表重复入口。
3. **头像形状不一致**：所有头像使用 `CircleAvatar`（圆形），而微信等主流 IM 均为圆角方形。
4. **群头像缺失**：群聊会话没有组合头像，仅显示单头像或占位，无法直观感知群聊身份。

## 2. 目标用户

- 趣聊日活用户（打开 App 后首个高频入口）

## 3. 功能范围

### 3.1 In-Scope

| 编号 | 功能 | 说明 |
|---|---|---|
| UI1 | Tab 居中修复 | "趣聊/同好"Tab 文字在页面水平居中，不受右侧按钮数量影响 |
| UI2 | 移除小趣助理条目 | 删除会话列表中小趣私人助理的置顶条目 |
| UI3 | 圆角方形头像 | 新建 `RoundedSquareAvatar` 通用组件，替换所有 `CircleAvatar` |
| UI4 | 群聊九宫格头像 | 新建 `GroupAvatarGrid` 组件，按成员数量自适应布局 |
| UI5 | 同好列表头像同步 | 同好 Tab 下的联系人头像也改为圆角方形 |

### 3.2 Out-of-Scope

- 会话列表数据加载/缓存（由 `chat-list-local-cache` 承担）
- 头像图片下载缓存策略（由 `chat-detail-avatar-display` 承担）
- 群管理相关交互（由 `chat-group-admin-govern` 承担）

## 4. 九宫格头像布局规则

按照有头像的成员数量（无头像成员跳过，有默认头像的计入）：

| 有效头像数 | 布局 | 说明 |
|---|---|---|
| 1 | 单个居中 | 仅一人有头像或所有人无头像时显示默认 |
| 2 | 上下居中，各一个 | 两人并排 |
| 3 | 品字：上一居中，下二 | |
| 4 | 2×2 四宫格 | |
| 5 | 上二居中，下三 | |
| 6 | 上三下三 | |
| 7 | 三行：上一居中，中三，下三 | |
| 8 | 三行：上二居中，中三，下三 | |
| ≥9 | 3×3 九宫格（取前9） | |

成员排序：按加入群聊的时间顺序（与群设置页成员列表一致）。

如果所有成员都没有设置头像（均为默认头像），则只显示一个默认占位。

## 5. 对标

| 维度 | 微信 | 趣聊目标 | 借鉴/不借鉴 |
|---|---|---|---|
| 头像形状 | 圆角方形 ~6px | 圆角方形 ~8px | 借鉴形状，圆角略大 |
| 群头像 | 九宫格，按加入序 | 九宫格，按加入序 | 完全借鉴 |
| Tab 居中 | 标题固定居中 | Tab 真正居中 | 借鉴思路 |
| 助理入口 | 无列表置顶 | 移除置顶 | 借鉴 |

## 6. 业务约束

- `RoundedSquareAvatar` 圆角值使用设计系统 token `AppSpacing.borderRadiusMd`（约 8px）
- 九宫格头像的每个子头像间距使用 `AppSpacing.one`（1px）
- 头像尺寸：会话列表 56px（28pt 半径等效）、同好列表 56px
- 九宫格整体尺寸与单头像一致（56×56），内部子头像按比例缩放
- 主题色蓝色（`AppColors.primaryColor`），其余保持现有页面元素布局

## 7. 非功能要求

- 九宫格头像纯前端渲染，不额外请求接口
- `GroupAvatarGrid` 支持任意数量成员输入，内部截取前 9 个有头像成员
- 性能：九宫格 Widget build 耗时 ≤2ms（不含图片解码）
