# L1 规格：聊天与会话

## 范围
- 会话列表/详情、消息投递、群聊创建、成员管理、会话设置与联系人。

## 功能说明
- 提供聊天核心链路（列表 -> 详情 -> 发送 -> 回执）并保证端云仓储切换无感。
- 支持群聊创建与成员管理，满足从圈子/联系人构建会话的业务场景。
- 统一会话治理能力（静音、置顶、已读、联系人检索）以保障用户体验一致性。

## 约束
- 端侧聊天页面数据源必须支持 mock/remote 无感切换。
- 会话与消息接口统一 `items` + `nextCursor`。
- 聊天链路必须透传 `X-Request-Id`、`X-Trace-Id`、`X-Client-Page-Id`、`X-Client-Session-Id`。

## 验收标准（L1 重点）
- A1：会话和消息全链路稳定可用。
- A2：聊天关键接口 p95/p99 达标。
- A7：契约字段不允许「猜字段」，端云强一致对齐。
- A8：仓储切换、契约、集成与 UAT 用例完整。
