# group-settings 设计方案

## 设计动因

群聊设置页当前同时承载：

- 成员查看/添加
- 群资料
- 会话级设置
- 清空记录
- 退出群聊

但随着“对象级治理下沉”“通话入口迁移到输入区 `+` 面板”被冻结，如果继续把更多动作塞进设置页，会让设置页失去边界。因此本设计聚焦两件事：

1. 明确群设置页只处理“群资料 + 会话配置 + 退出群聊”
2. 把治理与发起动作下沉到更贴近对象的位置

## 上游输入评审

- `group-settings/spec.md` 已冻结“唯一危险操作是退出群聊”
- `contact-and-session-governance/spec.md` 已冻结对象级举报/拉黑下沉
- `realtime-call/spec.md` 已冻结群聊通话入口放到输入区 `+` 面板

## 方案对比

### 对比 1：群聊设置页是否承载举报/拉黑

#### 方案 A：全部放进群设置页

**优点**：
- 用户在一个地方看到所有动作

**缺点**：
- 混淆对象：群、成员、消息
- 危险动作堆叠
- 与 1v1 设置页语义不一致

#### 方案 B：群设置页只做会话管理，对象级治理下沉（选定）

**优点**：
- 入口边界清晰
- 更符合操作对象语义
- 后续管理员治理扩展更稳

## 页面结构设计

### GS-1：成员区

保留：

- 成员头像网格
- 查看更多成员
- 添加成员入口

不直接提供：

- 举报成员
- 拉黑成员

这两类动作在成员卡片或成员主页上做对象级治理。

### GS-2：群资料区

保留：

- 群名
- 群二维码
- 群公告

当前不增加：

- 举报群
- 关联通话入口

### GS-3：会话配置区

保留：

- 免打扰
- 置顶
- 隐私屏蔽
- 聊天背景
- 清空聊天记录

这些动作仅影响当前用户的会话体验，不影响群对象本身。

### GS-4：危险操作区

唯一保留：

- `退出群聊`

不混入其他危险动作，保持用户心智稳定。

## 治理动作下沉设计

### 成员对象

从成员列表点进成员主页或成员卡片的 ActionSheet，可提供：

- 查看主页
- 举报用户
- 拉黑用户

### 消息对象

从群消息长按菜单可提供：

- 举报消息
- 其他消息级动作（引用、转发、复制等）

### 群对象

当前阶段不提供“举报群”与“拉黑群聊”。

如果后续需要“举报群”，应单独建模为群对象治理，不在当前群设置页临时追加入口。

## 与多人通话入口的边界

群聊通话入口统一从输入区 `+` 面板发起，不在以下位置出现：

- AppBar
- 群聊设置页

选择页规则：

- `<= 8 人`：默认全选（除自己）
- `> 8 人`：默认不选

## 关键设计决策

### KD-1：群设置页与对象级治理彻底分离（已定）

设置页只做“会话管理”，治理动作下沉到成员/消息对象。

### KD-2：危险操作最小化（已定）

群设置页危险操作仅保留“退出群聊”，避免用户在同一页面同时做退出、举报、拉黑等风险动作。

### KD-3：发起型动作下沉（已定）

群聊通话、群内扩展动作属于“发送/互动型”，统一走输入区 `+` 面板，不进入设置页。

## 适用场景与约束

- **适用**：普通群聊成员视角的群设置页
- **约束**：与 `contact-and-session-governance` 和 `realtime-call` 的对象边界一致
- **局限**：不覆盖管理员治理、禁言、踢人、审批等未来能力

## 未来演进

1. 管理员专属设置区（禁言、踢人、审批）
2. 群资料治理对象（如未来引入举报群）

## 全屏表单态 UI 工程方案（baseline 2026-03-29）

### 上游补充

- `group-settings/spec.md` §9 冻结 Inset Grouped、成员折叠、顶栏全局搜索入口与文案门禁。
- **无 metadata / codegen 变更**；不改变 `getGroupSettings` / `updateGroupSettings` 等键语义。

### 选型结论

| 方案 | 结论 |
|------|------|
| 每页手写 `Container` + 描边 | 否；与 iOS 设置分组不一致，维护分叉 |
| 统一 `SettingsInsetForm*` + `insetForm*` token | **是**；与 `chat-conversation` 内多页复用 |

### 组件与路径

| 用途 | 位置 |
|------|------|
| 全屏骨架、分组、行、分割线 | `lib/components/settings_form/settings_inset_form_page.dart` |
| 半屏更多操作 | `lib/components/settings_conversation/` |
| 顶栏搜索图标（与首页一致） | `GlobalTopBarIconButton` + `GlobalSearchLauncher` |

### 关键决策

- **KD-4**：全屏群相关设置页与全局「应用设置」类页面共享同一 Inset Grouped 语义，大屏留白由 `insetFormListHorizontalPadding` 统一。
- **KD-5**：全局搜索与页内列表过滤分离；本 baseline 仅冻结 **全局搜索启动器** 在群信息顶栏的样式与跳转，内嵌搜索另开 L3 时再定契约。

### 迁移顺序（实施切片）

1. `ChatSettingsPage`（已按 §9 对齐的可作为参考实现）
2. `GroupManagePage`
3. 其他 `lib/ui/settings/`、`lib/ui/circle/` 等全屏列表设置页（按需）
