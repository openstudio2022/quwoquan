# L2 规格：contact-and-session-governance — 联系人与会话治理

> **层级**：L2_feature（隶属 L1 `chat-conversation`）
> **状态**：specified

## 0. 一句话定义

面向趣聊的弱关系到强关系社交链路，定义 `陌生用户 → 关注用户 → 打招呼 → 正式会话 → 同好 → 密友` 的状态机、请求箱、升级入口和治理边界，避免会话列表被骚扰流污染，同时为实时语音/视频通话提供明确的关系门禁。

## 1. 背景与动机

当前聊天与用户关系只有“关注/私信”两级表达，存在三个核心问题：

1. 单向关注后若直接创建私聊，会使普通聊天列表被未回应关系污染，不符合“免骚扰”目标。
2. 互关、密友、正式会话之间没有清晰的升级路径，导致“何时可以消息、何时可以语音/视频”缺乏统一语义。
3. 群聊与 1v1 的治理动作边界不清晰，容易把“退出群聊”“拉黑用户”“举报消息/用户”混放在同一路径，增加误操作和认知负担。

本特性的职责是先把关系与会话治理语义冻结，再由 `realtime-call`、`group-settings` 等下游能力复用。

## 2. 目标用户

- 想从浏览、关注逐步发展为稳定互动关系的趣聊用户
- 希望先表达兴趣而非立即建立正式私聊的用户
- 希望控制陌生人打扰、又不完全关闭新关系机会的用户

## 3. 功能范围

### 3.1 In-Scope

| 编号 | 能力 | 说明 |
|------|------|------|
| G1 | 五态关系模型 | `我自己 / 陌生用户 / 关注用户 / 同好 / 密友` 五种主页与会话语义 |
| G2 | 打招呼请求箱 | 单向关注后可发送自定义打招呼，不进入普通聊天列表 |
| G3 | 回复建会话 | 对方回复打招呼后，自动升级为正式 1v1 会话 |
| G4 | 会话内加同好 | 已建立正式会话但尚未互关时，会话内可执行“加同好”升级 |
| G5 | 密友标记 | `密友 ⊂ 同好`，支持在添加同好时勾选或后续手动切换 |
| G6 | 关系降级 | 任一方取消关注后，同好失效；若是密友则自动失去密友身份 |
| G7 | 免骚扰治理 | 陌生人打招呼开关、频控、重复请求限制、对象级举报/拉黑 |
| G8 | 列表隔离 | 打招呼请求与正式会话分层展示，避免污染普通聊天列表 |

### 3.2 Out-of-Scope

- 基于共同圈子/共同兴趣的算法自动判定同好
- 群对象举报能力
- 群对象拉黑能力
- 同好/密友推荐排序算法
- 关系积分、勋章、亲密度数值化玩法

## 4. 关系对象模型

### 4.1 RelationshipState

| 状态 | 判定 | 说明 |
|------|------|------|
| self | viewerId == targetUserId | 我自己 |
| stranger | 无关注关系、无有效打招呼升级 | 陌生用户 |
| following_only | 单向关注 | 关注用户 |
| same_interest | 互关 | 同好，作为实时沟通门槛 |
| close_friend | 互关 + closeFriend=true | 密友，属于同好的子集 |

### 4.2 GreetingRequest

| 字段 | 类型 | 说明 |
|------|------|------|
| requestId | string | 请求唯一标识 |
| requesterId | string | 发起打招呼者 |
| targetUserId | string | 接收方 |
| content | string | 自定义打招呼内容 |
| status | enum | pending / replied / ignored / blocked / expired |
| createdAt | datetime | 创建时间 |
| repliedAt | datetime? | 回复时间 |
| promotedConversationId | string? | 升级后的正式会话 ID |

### 4.3 SessionPromotion

| 阶段 | 说明 |
|------|------|
| greeting_request | 仅存在于请求箱，不进入普通聊天列表 |
| replied_pending_upgrade | 已回复，已建立正式会话，但尚未升级为同好 |
| formal_conversation | 正式会话，支持正常收发消息 |
| same_interest_conversation | 正式会话 + 已是同好，解锁音视频 |
| close_friend_conversation | 正式会话 + 已是密友 |

## 5. 核心用户旅程

### 5.1 关系升级主链路

```text
陌生用户
  -> 关注用户
  -> 打招呼（自定义内容，进入请求箱）
  -> 对方回复
  -> 自动建立正式会话
  -> 会话内加同好
  -> 升级为同好
  -> 可选设为密友
```

### 5.2 降级链路

```text
密友 / 同好
  -> 任一方取消关注
  -> 降级为单向关注或陌生关系
  -> 自动失去密友标记
  -> 若已有正式会话则保留消息历史，但失去音视频入口
```

### 5.3 打招呼与会话边界

- 打招呼未被回复前：不创建正式会话，不进入普通聊天列表。
- 对方回复后：自动升级为正式 1v1 会话，进入普通聊天列表。
- 正式会话建立后但仍未互关：会话顶部显示“加同好”升级条。
- 升级为同好后：会话内解锁实时语音/视频入口。

## 6. 入口与交互基线

### 6.1 用户主页动作矩阵

| 关系状态 | 主动作 | 次动作 | 备注 |
|----------|--------|--------|------|
| 我自己 | 编辑资料 | 管理分身 | 不出现社交动作 |
| 陌生用户 | 关注 | 更多 | 不直接创建会话 |
| 关注用户 | 打招呼 | 已关注 | 打招呼允许自定义内容 |
| 同好 | 消息 / 视频 / 语音 | 更多 | 三个按钮等宽等间距 |
| 密友 | 消息 / 视频 / 语音 | 更多 + 已设为密友 | 三个按钮等宽等间距 |

### 6.2 会话页动作矩阵

| 会话阶段 | 顶部关系条 | 输入区 `+` 面板 | AppBar |
|----------|------------|----------------|--------|
| 仅打招呼请求 | 无正式会话 | 无 | 不进入普通聊天列表 |
| 已回复未同好 | 显示“加同好”升级条 | 仅异步消息相关能力 | 保持简洁 |
| 同好/密友会话 | 不显示升级条 | 解锁语音/视频通话 | 保持简洁，不放通话按钮 |

### 6.3 群聊治理边界

- 群聊设置页仅保留会话管理与 `退出群聊`。
- 不提供“拉黑群聊”。
- 当前阶段不提供“举报群”。
- 举报/拉黑下沉到对象级：
  - 成员卡片/成员主页：举报用户、拉黑用户
  - 消息长按菜单：举报消息

## 7. 业务约束

- `同好 = 互关`；不存在“互相回复但未互关即同好”的状态。
- `密友 ⊂ 同好`；非同好不能被标记为密友。
- 任一方取消关注后，`closeFriend` 自动失效。
- 打招呼支持自定义内容，但需受免骚扰设置和频控约束。
- 同一对用户在 `pending` 状态下只允许存在 1 条有效打招呼请求。
- 被对方屏蔽陌生人打招呼时，发送方不能绕过设置建立会话。
- 已建立正式会话并不自动代表已升级为同好。
- 对象级拉黑优先级高于所有社交升级动作；被拉黑后应阻断打招呼、加同好、音视频通话入口。

## 8. 非功能要求

### 8.1 免骚扰

- 打招呼默认进入请求箱，不污染普通聊天列表。
- 用户可配置“屏蔽陌生人打招呼”。
- 打招呼应有频控和去重机制，避免重复骚扰。

### 8.2 一致性

- 关系状态、主页按钮、会话入口、音视频权限必须共享同一份状态真相源。
- 打招呼回复 → 会话升级 → 加同好 → 密友切换，必须有完整状态流转和审计。

### 8.3 可观测

- 需记录打招呼发送率、回复率、升级同好率、加密友率、被屏蔽/被拉黑拦截率。

## 9. 与下游特性的边界

| 特性 | 依赖关系 | 本特性提供 |
|------|----------|------------|
| `realtime-call` | 消费关系门禁 | 是否可显示和发起 1v1 通话 |
| `group-settings` | 消费治理边界 | 群聊只保留退出群聊，不承担举报/拉黑群 |
| `user profile` | 消费关系状态 | 五态主页按钮矩阵 |
| `chat list/detail` | 消费请求箱与正式会话边界 | 请求箱不进普通聊天列表 |

## 10. 验收重点

### T1
- 关系状态枚举与降级规则正确
- 打招呼状态机、密友约束、阻断规则正确

### T2
- 五态主页按钮矩阵正确
- 会话顶部“加同好”关系条出现/消失正确
- 请求箱与普通聊天列表分层正确

### T3
- 打招呼发送、回复、建会话、加同好全链路正确
- 取消关注导致关系降级与密友失效正确
- 拉黑后阻断打招呼和音视频入口

### T4
- 真实用户旅程下，请求箱、正式会话、同好升级、音视频解锁体验连续稳定
