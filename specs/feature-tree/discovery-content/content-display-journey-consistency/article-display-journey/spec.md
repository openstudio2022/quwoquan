# L3 特性：article-display-journey（文章展示与阅读旅程）

## 背景与动机

当前文章展示存在三类结构性断裂：

1. 发现页文章 feed、圈子双列卡片、侵入式阅读器是三套分裂形态，封面、标题、摘要与正文没有统一的产品规则。
2. 沉浸式文章仍更像“post 放大态”，没有形成真正的全屏阅读器体验，顶部 chrome、页码、分页、封面参与方式与图片/视频浏览器也不一致。
3. 编辑态虽已具备分页与模板基础，但作者无法把“分发封面、书本风格、阅读壳层、真实书页气质”作为同一组可预览、可保存、可分发的展示设置来创作。

本场景的目标是冻结文章从“创作设置 -> 分发展示 -> 沉浸阅读 -> 作者页返回”的统一产品基线：作者在编辑器中选择封面与书本模板，文章在关注流和圈子频道以合适密度分发，进入后切换到真正的全侵入式书本阅读器，并在返回时保持同源状态与同源内容。

## 目标用户

- 在发现页文章频道浏览长文，并通过标题、封面和摘要快速判断是否进入阅读的用户。
- 在圈子频道双列流中高密度浏览文章，希望卡片在有图/无图两种情况下都保持精致辨识度的用户。
- 进入沉浸式文章阅读器后，希望获得接近一流图书阅读器的连续翻页与真实书页体验的用户。
- 创作文章时希望所见即所得，能直接控制封面、模板、纸张风格和阅读气质的作者。

## 核心问题

1. 文章在关注流、圈子频道、沉浸式阅读器中分别应承担什么信息密度和视觉职责。
2. 封面图是分发素材、阅读扉页素材，还是两者兼具，以及无封面时如何优雅降级。
3. “手帐 / 精装书 / 画册 / 科技手册”等真实书本风格如何在编辑态与阅读态中保持一致。
4. 文章页码、图片组页码、顶部返回导航、翻页动效如何形成统一阅读语言。

## 功能范围

### F1：关注流文章分发卡

- 关注流文章卡是“点击进入”的分发入口，不承担完整阅读。
- 有封面时使用“左封面 + 右标题 + 两行摘要”布局：
  - 左侧显示作者显式选择的文章封面。
  - 右侧标题最多 2 行，摘要最多 2 行。
  - 不显示页码，不显示模板纹理细节，不展示正文内图片数量。
- 无封面时使用纯文字卡：
  - 不保留空白图片区。
  - 标题最多 2 行，摘要最多 3 行。
  - 卡片上可显示轻量“文章”语义标签，但不能喧宾夺主。
- 整个卡片为统一热区，点击直接进入文章沉浸式阅读器。

### F2：圈子频道双列文章卡

- 圈子频道中的文章卡用于高密度浏览与审美辨识，必须适配双列瀑布流。
- 有封面时使用“上封面 + 下标题/摘要”的双列卡：
  - 封面承担频道内视觉抓手。
  - 标题最多 2 行；摘要最多 1 行。
  - 卡片高度允许因封面比例不同而产生轻量瀑布差异。
- 无封面时使用“高质感文字卡”：
  - 不强行预留图片区。
  - 标题最多 2 行；正文摘录 `3~5` 行。
  - 卡面可继承模板的弱纸张语义，如淡纸纹、淡页边、轻装订感，但不得重到影响信息阅读。
- 圈子双列文章卡点入后同样进入统一的文章沉浸式阅读器，不允许分叉到另一套“详情页”。

### F3：全侵入式文章阅读器

- 文章进入后切换到全屏阅读器语义，而不是图片/视频 post 放大语义。
- 阅读器必须满足：
  - 书页舞台左右占满屏幕，可见的是“书页实体边缘”，不是普通卡片外边距。
  - 顶部导航位于安全区内独立悬浮，正文页从导航下方开始，不得侵入顶部工具栏区域。
  - 默认只保留轻量阅读 chrome，不显示 post 式配文区或图片/视频式说明块。
- 页码统一规则：
  - 文章页码显示为 `当前页/总页数`。
  - 位置统一放在返回按钮之后，与图片组查看器保持相同导航语义。
  - 不再为文章使用右下角点状页码，也不允许文章与图片各用一套不同位置体系。
- 封面参与阅读的规则固定为：
  - 若文章存在封面，则封面进入沉浸式阅读器第一页。
  - 第一页不是“纯封面独占页”，而是“扉页式第一页”：上部展示封面，下部继续展示标题与正文开头。
  - 若无封面，则第一页直接从标题和正文开始。
- 阅读器默认进入纯阅读态；轻点页面后才显示顶部导航和底部互动层。

### F4：真实书页与翻页交互

- 文章沉浸式阅读器的翻页主语言是“真实书页翻动”，不是普通水平滑卡。
- 正式基线支持四角翻页热区：
  - 右上角 / 右下角：翻到下一页。
  - 左上角 / 左下角：翻回上一页。
- 拖拽页角时必须允许用户停留在中间状态；松手后按阈值自动完成翻页或回弹。
- 翻页反馈必须包含：
  - 页角掀起
  - 纸张背面
  - 翻页阴影
  - 下层页逐步露出
- 若系统开启减少动态效果或设备性能不足，允许降级为“页角预掀起 + 平滑翻页”，但视觉语义仍必须是书页，而不是回退成通用 `PageView` 卡片切换。

### F5：模板体系与书本风格

- 文章展示模板不是单纯颜色皮肤，而是完整的书本风格系统，必须同时作用于：
  - 编辑态纸面
  - 预览分页
  - 沉浸式阅读器
  - 关注流 / 圈子卡片的弱化气质表达
- 本期正式冻结 5 套模板：

| 模板 ID | 产品名 | 视觉定位 | 适合内容 |
|---------|--------|----------|----------|
| `journal` | 手帐札记 | 纸纹、胶带、贴纸、活页/手帐边缘 | 旅行、生活、校园、遇见 |
| `ritual` | 精装随笔 | 书脊、厚纸、精装边缘、克制衬线 | 人文、文学、记录、人物 |
| `diffuse` | 画册相簿 | 厚卡纸、画册边缘、图片优先留白 | 摄影、旅行、美食、运动 |
| `tech` | 科技手册 | 细网格、章节点、功能型页边 | 科技、AI、编程、车友改装 |
| `gentle` | 轻刊专栏 | 轻杂志感、柔和纸面、适度 editorial 节奏 | 生活、人文、情感、综合长文 |

- 圈子频道的模板默认推荐如下，作者可覆盖：

| 圈子频道 | 默认推荐 | 次推荐 |
|----------|----------|--------|
| 遇见 | `gentle` | `journal` |
| 校园 | `journal` | `ritual` |
| 车友 | `tech` | `diffuse` |
| 人文 | `ritual` | `gentle` |
| 生活 | `gentle` | `journal` |
| 运动 | `diffuse` | `tech` |
| 科技 | `tech` | `ritual` |
| 旅行 | `journal` | `diffuse` |
| 美食 | `diffuse` | `journal` |
| 推荐 | 跟随作者已选模板；未设置时使用频道推荐 |

### F6：编辑态与阅读态一致

- 编辑器必须把“封面、模板、字体、分页阅读效果”作为同一组展示设置提供给作者。
- 作者至少可配置：
  - 是否选择封面
  - 封面图片
  - 文章模板
  - 字体预设
- 编辑态必须做到：
  - 分页逻辑与阅读态一致
  - 模板纸张、边缘和版心节奏与阅读态一致
  - 模板切换后，预览页、扉页、列表卡片和圈子卡片的分发效果可预期
- 编辑态允许保留输入控件、工具栏和结构面板，但不能改变页面几何语义或把阅读态模板降级成“另一套 UI”。

### F7：同源状态与返回一致

- 关注流文章卡、圈子双列文章卡、文章沉浸式阅读器、作者详情页必须共用同一篇文章与同一作者状态。
- 进入作者详情页再返回时，关注态、点赞态、收藏态、评论数与分享数必须与阅读器和来源流保持一致。
- Mock/Remote 切换后，文章展示规则、模板规则和封面参与规则都必须保持一致。

## Out of Scope

- 新增独立的文章分享链路、文章付费链路或专属书架产品。
- 文章模板的用户自定义贴纸商店、字体商店与主题市场。
- 平板横屏双页展开的完全真实装帧模式；本期只要求大屏保持更舒适的阅读舞台与同一书本语义。
- 文章封面参与阅读的多模式切换；本期固定为“有封面即进入扉页式第一页”。
- 服务端 AI 自动选模板与自动选封面。

## 约束

- 主归属继续沿用 `discovery-content/content-display-journey-consistency/article-display-journey`，不新建平行文章展示 Journey。
- 文章展示设置必须纳入单一 presentation state，不允许关注流、圈子流、阅读器、编辑器各自维护第二套模板判断表。
- 封面必须来自作者显式选择；不得默认拿正文第一张图顶替封面。
- 图片组页码与文章页码可以共用同一导航位置语义，但同一时刻只允许存在一个主页码源。
- 模板推荐来自圈子频道，但作者最终选择优先；频道推荐不得强行覆盖作者设置。

## 对标输入与吸收结论

### 外部对标

| 对标 | 借鉴点 | 不借鉴点 |
|------|--------|----------|
| 微信读书 | 顶部返回与页码统一、沉浸阅读 chrome 克制 | 不照搬其纯文本阅读壳，不弱化图文模板表达 |
| Apple Books | 书页舞台、翻页实体感、模板与阅读气质统一 | 不照搬其系统级书架和购买体系 |
| 小红书长图文 | 列表分发效率、封面进入阅读的转场连续感 | 不照搬其“图文详情页”而放弃真正阅读器体验 |
| Notion / Medium | 编辑态与阅读态的一致排版心智 | 不照搬无实体书本气质的纯网页排版风格 |

### 吸收结论

- 文章必须从“详情页”升级为“阅读器”。
- 封面应兼做分发素材与阅读扉页素材，而不是二选一。
- 模板必须是 presentation system，而不是装饰皮肤。
- 编辑态必须与阅读态共享分页与模板语言，否则作者无法预期最终效果。

## 角色分工

- 产品：冻结三种展示方式、封面参与规则、模板体系、页码规则与翻页基线。
- 设计：冻结书页舞台、扉页布局、卡片比例、模板视觉 token 和动效基线。
- 客户端：实现模板一致渲染、封面分发、沉浸式阅读器、编辑态预览与大屏适配。
- 云端：在 metadata / projection 层补齐封面、模板、字体等展示字段与读写契约。
- 测试：建立 T1~T4 覆盖，重点验证分发卡、编辑预览、沉浸式阅读和返回一致性。

## 数据生命周期合同

- 文章正文与图片锚点仍属于文章内容真相源。
- 封面、模板、字体属于文章展示真相源，和正文一起保存在文章 draft / post payload 中。
- 作者在编辑器中修改封面或模板后，预览分页、关注流分发卡和圈子卡片都应立即反映最新 presentation state。
- 发布后的文章若修改封面或模板，所有来源卡片与阅读器均按最新 presentation state 渲染。

## 权限、分享与可见性边界

- 本次不新增文章权限模型，沿用 post 的公开 / 圈内可见 / 私密能力边界。
- 若文章所在圈子不可见，则圈子频道卡片不可展示，阅读器也不可从圈子流进入。
- 分享能力只复用现有 post 分享，不新增“导出电子书”或“生成外部书页链接”。

## 非功能目标

### SLO / KPI

- 文章卡点击到阅读器首帧可见：`p95 < 500ms`（本地已有数据场景）。
- 翻页拖拽响应：`p95 < 16ms` 单帧输入反馈。
- 翻页完成动画：`220ms ~ 420ms`，按模板阻尼略有差异。
- 分页重算后页码稳定率：相同容器尺寸下不得出现频繁跳页或页数抖动。
- 模板切换后编辑器预览刷新：`p95 < 150ms`。

### 弱网 / 性能 / 设备分层

- 弱网下阅读器至少保证已加载页面可继续翻阅，不因额外装饰或模板纹理阻塞正文阅读。
- 降级策略必须优先保正文与分页，再降翻页特效，再降边缘装饰；不得反向为了保动效牺牲可读性。
- `compact / regular / expanded` 三档屏幕都必须保持相同阅读语义；大屏可放大舞台与书页节奏，但不能变成另一套 IA。

## 迁移、灰度与回滚要求

- 迁移顺序固定：
  1. 冻结 PRD 与 acceptance
  2. 补文章 presentation settings 契约
  3. 打通编辑态模板设置
  4. 替换关注流和圈子卡片文章样式
  5. 切换文章沉浸式阅读器
- 灰度建议按模板能力与翻页能力分层：
  - G1：封面 + 分发卡
  - G2：沉浸式阅读器新壳层
  - G3：真实翻页与页角拖拽
- 回滚必须允许：
  - 阅读器回退到当前稳定分页壳
  - 分发卡回退到旧 article card
  - 模板设置保留但按默认模板渲染

## 验收重点

1. 关注流、圈子双列和沉浸式阅读器三种展示方式的职责边界与信息密度已冻结。
2. 封面是否参与阅读、如何参与阅读已冻结，不再在实现阶段反复裁决。
3. 顶部 `返回 + 页码` 已冻结为文章与图片组的统一阅读导航语言。
4. 模板体系、频道推荐与编辑态一致性已冻结，进入 `/design` 后无需再裁决“模板是皮肤还是书本系统”。
5. 翻页体验已明确为真实书页语言，并定义了可接受的降级边界。
