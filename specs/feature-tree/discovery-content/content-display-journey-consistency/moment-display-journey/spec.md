# L3 特性：moment-display-journey（微趣旅程）

## 功能说明

**微趣独立旅程**：微趣频道 → 侵入式媒体浏览器 → 作者详情，端到端数据源一致、交互状态跨页面同步、重入状态保持。按 `category=moment` 隔离。

- **行为基线**：作品侵入式浏览器作为统一行为基线；微趣点击图片/视频后进入同等交互能力的侵入式浏览器。
- **结构策略**：定义通用侵入式浏览器内核，`works` 与 `moment` 分别实例化装配层。
- **顶栏策略**：`works` 使用完整顶栏；`moment` 仅返回+更多。
- **底栏策略**：`works` 与 `moment` 复用同源同实现的互动底栏（作者、关注、赞、收藏、评论、分享）。

### 侵入式浏览器规格

| 能力 | 说明 |
|------|------|
| **通用内核** | `ImmersivePostViewerCore` 负责手势、分页、状态同步、回调分发、沉浸/纯净模式 |
| **场景实例化** | `WorksImmersiveViewer` 与 `MomentImmersiveViewer` 仅负责场景装配，不重复实现核心能力 |
| **feedPosts 支持** | 发现页微趣点击时必须传入 `feedPosts`，浏览器内可连续浏览微趣列表 |
| **浏览顺序** | 微趣场景：同帖横滑 + 跨帖纵滑；同一微趣内不混编图与视频 |
| **顶部工具栏** | `WorksTopBar`（完整）与 `MomentTopBar`（返回+更多）分离定义 |
| **底部工具栏** | 使用统一 `ImmersiveEngagementBar`（以作品行为为基线） |
| **文字展示** | 文字位于媒体底部，默认 3 行；超出显示「...全文」，点击后可展开并滚动 |

### 适用范围与约束

- **适用**：发现页微趣频道、侵入式媒体浏览器、作者详情联动。
- **约束**：按 `category=moment` 隔离；微趣场景顶栏必须为 backOnly 语义。
- **不适用**：作品频道原有业务语义不变，仅做结构收敛与组件抽取。

## 验收（摘要）

- A1：列表、媒体浏览器、作者详情数据与 post 一致。
- A2：关注/赞/收藏跨页面一致，重入后保持。
- A3：作者详情关注变更返回同步。
- A4：媒体浏览器支持 `feedPosts`，二维导航（横向=同微趣图，上下=微趣列表）。
- A5：文字底部展示，3 行折叠，超出「全文」可展开可滚动。
- A6：Mock/Remote 切换后正确。
- A7：微趣顶栏仅返回+更多，作品顶栏保持完整。
- A8：works/moment 复用同源底栏实现（操作顺序、计数格式、回调一致）。
- A9：微趣点击图/视频进入侵入式后，除顶栏外与作品行为一致。
- A10：回归通过（moment 图文、moment 视频、works 不回归）。
