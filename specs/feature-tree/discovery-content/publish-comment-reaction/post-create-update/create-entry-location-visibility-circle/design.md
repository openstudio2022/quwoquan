# Design: create-entry-location-visibility-circle

## 1. 设计目标
- 以最小改动打通“可发布”的创作入口，避免继续出现“UI 有入口、提交无效果”的断裂。
- 维持现有端云契约，优先复用已有字段与发布接口，不新增不必要 metadata。
- 为后续地图能力升级（POI 详情、逆地理编码、智能圈子推荐）保留扩展点。
- 能力设计为跨内容类型可复用（moment/photo/video/article）并可复用于其它媒体编辑页。

## 2. 方案对比与选型

### 方案 A：页面内联实现（快速）
- 在创作页直接拼接位置搜索、圈子选择、公开切换逻辑。
- 优点：开发快。
- 缺点：页面复杂度上升，跨页面复用差，测试成本高。

### 方案 B：独立选择器组件 + 发布状态聚合（选定）
- 位置选择、圈子选择拆为独立组件/弹层，创作页仅做状态聚合与回填展示。
- 公开设置与发布设置状态抽象为通用 `PublishSettings`，由各内容类型页面共享。
- 优点：复用性好、可测性好、后续扩展地图供应商更稳定。
- 缺点：初次实现文件数略增。

**选择理由**：当前需求明确会继续迭代发布前配置项，采用方案 B 可控制复杂度并降低后续变更风险。

## 3. 核心决策

### 3.1 地图供应商可配置切换
- 新增 `MapProviderType` 配置（baidu/alimap）。
- 端侧通过云侧 `integration-service` 访问位置能力，屏蔽供应商差异：`nearby()`、`search(keyword)`、`select(poi)`。
- 云侧采用主备供应商策略：主用失败自动尝试备用一次，两次失败才返回错误。
- 发布仅提交标准化结果（`location`, `locationName`），不泄露供应商特定字段。

### 3.2 公开性与圈子联动
- `isPublic=true` 时展示“选择要发布的圈子”。
- `isPublic=false` 时隐藏圈子区域，提交 payload 时强制清空 `circleIds`。
- UI 文案从“谁都可以看”调整为“选择要发布的圈子”，并放在公开设置下方。
- 上述语义在 moment/photo/video/article 四类发布入口保持一致。

### 3.3 回填与展示策略
- 位置：未选择显示“不显示位置”；已选择显示地理名（蓝色主色）。
- 圈子：多选后按名称逗号拼接，超过单行时尾部省略。
- 附近页加载态显示“正在搜索附近位置”；失败时以内联错误替换加载态，不弹权限对话框。
- 搜索页使用实时输入 + 防抖（约 280ms）；限流时静默保持最近成功结果，不打断输入。

### 3.4 Switch 与图标（UI 品质对齐）
- **Switch 统一使用 iOS 风格**：全部 Switch 采用 `CupertinoSwitch`，激活态为蓝色（`AppColors.primaryColor`），与高品质内容创作 App 定位一致。
- **图标替换**：是否公开、发布到圈子入口图标需匹配高品质定位，统一更换为更克制的 CupertinoIcons 或等效风格，避免与当前设计系统冲突。

### 3.5 圈子选择器布局
- **全屏页面**：圈子选择采用全屏页面（或占满高度的 modal），而非半屏 bottom sheet。
- **顶部工具栏**：居中显示「选择圈子」，左侧返回/关闭按钮。
- **底部工具栏**：固定底部栏，左「取消」、右「完成」双按钮。
- **暂不加搜索框**：本迭代不实现圈子搜索，后续演进再增加。
- **默认全选**：进入选择器时，默认勾选全部圈子；语义为「选择发布到哪些圈子」，不是「是否发布到圈子」。

### 3.6 选择发布圈子语义
- 是「选择要发布到哪些圈子」（多选，默认全部），而非「是否发布到圈子」（开关）。
- 公开时展示该入口；私密时隐藏且不提交 `circleIds`。

## 4. 端云契约对齐
- 复用现有 `CreatePost` 可写字段：`visibility`, `location`, `locationName`, `circleIds`。
- `visibility` 映射：`isPublic ? "public" : "private"`。
- `circleIds` 仅在公开时提交。
- 本迭代不新增后端路由；若圈子列表接口缺失，先对接现有仓储能力并提供空态。
- 四类内容在 `_buildPostPayload` 的内容特有字段之外，共享同一组发布设置映射逻辑。

## 5. 适用场景与约束
- 适用于 Flutter 创作页与图片/视频/文章编辑页的统一发布设置区域。
- 不适用于“更多功能按钮反馈”链路（本需求显式搁置）。
- 地图 SDK 依赖网络与密钥配置，需保证失败可降级。

## 6. 未来演进
- 增加 POI 历史与常用地点推荐。
- 圈子选择支持按最近互动排序与搜索（本迭代暂不加搜索框）。
- 发布前策略扩展：按圈子自动推荐可见范围和标签。
