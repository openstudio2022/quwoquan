# L4 特性：create-entry-location-visibility-circle

## 功能说明
- 打通创作入口中的发布前关键选项：`所在位置`、`是否公开`、`发布到圈子（多选）`。
- 该能力需被微趣、图片、视频、文章四类发布流程统一复用。
- 位置选择统一通过云侧 `integration-service` 获取（不直连地图 SDK），支持默认附近位置与地名搜索。
- 公开性与圈子分发联动：仅在公开时展示圈子选择，并将圈子选择结果写入发布 payload。

## 范围
- 端侧创作页与媒体编辑页的通用发布设置交互与状态管理改造。
- 新增位置选择全屏页与搜索全屏页（含附近列表、搜索、默认“不显示位置”）。
- 创作发布 payload 与既有云侧契约字段对齐：`visibility`、`location`、`locationName`、`circleIds`。
- 抽取可复用发布设置模块（位置/公开/圈子），供图片、视频、文章编辑页复用。
- 不包含“更多功能按钮反馈”闭环（本迭代明确排除）。

## 适用范围与约束
- 适用于微趣（moment）、图片（photo）、视频（video）、文章（article）的发布入口与对应编辑页。
- 地图服务切换通过配置生效，不在本任务内引入多套 UI 分支。
- 若地图服务不可用，必须允许用户回退为“不显示位置”继续发布；限流场景端侧静默保持最近成功结果。
- 圈子列表来源依赖现有圈子服务/接口；若无远端数据，需提供明确空态。

## 关键约束
- 默认值：不显示位置、公开。
- 选中位置后，创作页“所在位置”回填地理名，并使用蓝色主色显示。
- **位置选择流程与文案**：附近页加载态显示「正在获取结果」+ 动效；先检查/请求定位权限，未开启或未批准显示「请开启定位权限后重试」，永久拒绝显示「请在设置中为本应用开启定位权限」+「去设置」；云端错误按 `integration/location/errors.yaml` 映射（定位不可用/超时/服务异常），详见 design.md §3.3.1。
- 是否公开采用 Switch（iOS 风格，CupertinoSwitch），激活态蓝色；取消公开后，圈子选择区域隐藏且不提交 `circleIds`。
- 圈子入口文案：「发布的圈子」（简短）；无圈子时提示「加入圈子，发现同好」（有温度、引导性）。
- 圈子为「选择发布到哪些圈子」，默认全选；支持多选，回填以逗号分隔；超长单行省略显示。
- 圈子选择器：全屏页面，顶部工具栏居中显示「选择圈子」，底部工具栏「取消」「完成」双按钮；暂不加搜索框。
- **发现页可见性**：发现页（微趣、美图、视频、文章）仅展示 `visibility=public` 的内容；私密内容不出现在发现流。
- 图标与高品质内容创作定位匹配：是否公开、发布到圈子入口采用统一风格的图标。

## 验收标准（概要）
- A1：四类内容发布入口均可使用位置选择（附近+搜索）。
- A2：位置回填与主色高亮正确。
- A3：是否公开采用 iOS 风格 Switch（CupertinoSwitch），激活态蓝色，四类发布入口默认公开。
- A4：公开/非公开与圈子选择显示逻辑在四类内容中一致。
- A5：圈子多选回填与省略展示正确；选择器全屏、顶部居中「选择圈子」、底部取消+完成；默认全选；入口文案「发布的圈子」，无圈子时「加入圈子，发现同好」。
- A10：发现流仅返回 visibility=public 的内容（微趣/图片/视频/文章）；私密内容不出现在发现页。
- A6：四类内容发布 payload 字段正确映射到云侧契约。
- A7：异常与降级（地图失败、圈子空列表）可用。
- A8：复用能力在其它媒体编辑页可直接接入，相关测试通过并可进入 `/opsx-deliver`。
