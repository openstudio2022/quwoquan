# L1 特性：platform-ops-governance（运维横切）

## 功能说明
- 建立平台侧可观测、配置治理、服务治理、安全隐私、发布回滚的统一治理能力。
- 作为端云全链路“系统级质量与稳定性”兜底层。
- 作为统一 Web 门户 `ops-portal` 中 `Platform Ops` 工作域的特性树承载层。
- 冻结各领域的三类面拆分原则：`user-plane`、`platform-control-plane`、`product-control-plane`。

## 约束
- 所有服务必须接入统一治理策略，不允许按服务自定义核心口径。
- 语义 token、错误码、追踪头、治理参数必须标准化。
- 运维高风险变更必须具备灰度与回滚。
- 面向 `platform-ops` 的管理接口必须从统一控制面元数据生成，禁止手写临时 admin API。
- 三类面必须支持部署时任意组合：同进程、同 Pod 多容器、`seed-box` 共部署、独立 Deployment / Pod。
- 契约设计不得依赖当前部署拓扑，避免后续拆 Pod 返工。

## 验收重点
- A3：服务治理策略稳定生效
- A4：可观测统一且可检索
- A6：安全隐私策略可审计
- A8：治理链路自动化验证可复跑
- A7：控制面元数据、代码生成与部署组合约束一致
