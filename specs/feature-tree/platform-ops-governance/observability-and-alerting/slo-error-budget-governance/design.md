# Design: slo-error-budget-governance

## 设计动因

没有 SLO 门禁的灰度发布仍可能带病推进。需要以预算消耗驱动自动化决策。

## 关键设计

1. 指标集合：
   - HTTP 5xx 错误率
   - P95/P99 延迟
   - 依赖失败率（如 Redis connect error）
   - 配置版本维度稳定性（按 `CONFIG_VERSION` 打标签）
2. 阶段门禁窗口：
   - 每一灰度阶段固定观察时长
3. 判定动作：
   - 健康：继续下一阶段
   - 轻微异常：暂停并延长观察
   - 严重异常：触发一键回滚

## 与配置发布节点关系

- 本节点输出判定结果
- `risky-config-gray-release` 消费结果推进或回滚

## 约束

- 指标必须按配置版本打标签
- 所有判定结果都必须落审计事件
- 门禁必须能关联到具体实例组（stable/canary）以避免误判
