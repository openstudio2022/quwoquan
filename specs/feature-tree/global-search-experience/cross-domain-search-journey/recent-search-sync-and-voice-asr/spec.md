# L3 Scenario: recent-search-sync-and-voice-asr

## 节点定位

- `L1_capability`: `global-search-experience`
- `L2_journey`: `cross-domain-search-journey`
- `L3_scenario`: `recent-search-sync-and-voice-asr`

## 背景与动机

最近搜索和语音输入是搜索首页高频能力，但它们的生命周期与搜索结果本身完全不同。最新两段式搜索 UX 又新增了“最近在搜”的双列布局与记录管理态，因此需要单独冻结记录模型、管理语义以及与 `小趣搜` assistant 结果的边界。

## 目标用户

- 需要重复搜索记录关键词的用户。
- 更习惯说话而不是手打查询的移动端用户。

## 功能范围

- 最近搜索的记录字段、双列展示、展开、管理态、清理语义、本地与云同步边界。
- 语音 ASR 到文本 query 的输入语义。
- 语音权限失败时的降级策略。

## Out of Scope

- 原始音频长期存储。
- 语音语义理解与多轮语音助手能力。
- 问小趣会话存储本身。

## 约束

- 最近搜索由用户手动清除前持续保留。
- 最近搜索同步本地与云端，但本期不冻结自动过期时间。
- 语音入口只做 ASR 转搜索词，不进入 assistant 推理。
- `小趣搜` 不单独生成一条 AI query 记录记录。

## 对标输入与吸收结论

- 参考微信搜索首页中“最近搜索”和语音输入心智，并吸收记录管理态下的单条删除 / 全部清除交互。

## 角色分工

- `global-search-experience`: 负责记录模型、展示与语音写回搜索框。
- `assistant`: 只消费后续 assistant 会话，不消费普通搜索记录。

## 既有 Story 覆盖矩阵

| 既有能力 | 当前角色 |
|---|---|
| 聊天语音录制与发送 | 不复用为搜索能力，仅参考交互与权限处理 |
| 现有搜索原型中的本地查询 | 被正式最近搜索生命周期与记录管理态替代 |

## 数据生命周期合同

- 最近搜索记录字段：`query`、`launch_context`、`category_context`、`timestamp`。
- 用户可清空全部最近搜索，也可删除单条。
- `小趣搜` 只复用当前 query，不单独生成一条 AI query 记录。
- ASR 原始音频不写入最近搜索主模型。

## 小趣 / 权限 / 分享边界

- `小趣搜` 的 assistant continuation 与最近搜索分离。
- 麦克风权限被拒时，回退到手动输入。
- 本 Scenario 不定义分享。

## 非功能目标

- 最近搜索区块应在搜索首页即时可见，管理态切换不应阻塞主搜索路径。
- 语音转词成功后应快速写回搜索框并可直接继续搜索。

## 迁移、灰度与回滚要求

- 不保留旧的临时搜索记录逻辑。
- 若语音能力异常，不阻塞文本搜索主路径。

## 验收重点

1. 最近搜索生命周期与记录管理态明确。
2. `小趣搜` assistant continuation 与搜索记录分离。
3. 语音输入只承担 ASR 转词职责。
