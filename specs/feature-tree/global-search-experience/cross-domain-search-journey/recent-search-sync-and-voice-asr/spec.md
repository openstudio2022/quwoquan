# L3 Scenario: recent-search-sync-and-voice-asr

## 节点定位

- `L1_capability`: `global-search-experience`
- `L2_journey`: `cross-domain-search-journey`
- `L3_scenario`: `recent-search-sync-and-voice-asr`

## 背景与动机

最近搜索和语音输入是搜索首页高频能力，但它们的生命周期与搜索结果本身完全不同。PRD 需要单独冻结这两个子问题，避免后续与综合结果、问小趣 query 或 assistant 会话混淆。

## 目标用户

- 需要重复搜索历史关键词的用户。
- 更习惯说话而不是手打查询的移动端用户。

## 功能范围

- 最近搜索的记录字段、清理语义、本地与云同步边界。
- 语音 ASR 到文本 query 的输入语义。
- 语音权限失败时的降级策略。

## Out of Scope

- 原始音频长期存储。
- 语音语义理解与多轮语音助手能力。
- 问小趣会话存储本身。

## 约束

- 最近搜索由用户手动清除前持续保留。
- 最近搜索同步本地与云端，但本期不冻结自动过期时间。
- 语音入口只做 ASR 转搜索词，不进入 assistant 推理。
- 问小趣 query 不进入最近搜索。

## 对标输入与吸收结论

- 参考微信搜索首页中“最近搜索”和语音输入心智，但结合本产品明确语音只转词、不跑 AI。

## 角色分工

- `global-search-experience`: 负责历史模型、展示与语音写回搜索框。
- `assistant`: 只消费问小趣会话，不消费普通搜索历史。

## 既有 Story 覆盖矩阵

| 既有能力 | 当前角色 |
|---|---|
| 聊天语音录制与发送 | 不复用为搜索能力，仅参考交互与权限处理 |
| 现有搜索原型中的本地查询 | 被正式最近搜索生命周期替代 |

## 数据生命周期合同

- 最近搜索记录字段：`query`、`scope`、`facet`、`timestamp`。
- 用户可清空全部最近搜索，也可删除单条。
- 问小趣 query 只进入 assistant 对话，不进入最近搜索。
- ASR 原始音频不写入最近搜索主模型。

## 小趣 / 权限 / 分享边界

- 问小趣 query 与最近搜索分离。
- 麦克风权限被拒时，回退到手动输入。
- 本 Scenario 不定义分享。

## 非功能目标

- 最近搜索区块应在搜索首页即时可见。
- 语音转词成功后应快速写回搜索框并可直接继续搜索。

## 迁移、灰度与回滚要求

- 不保留旧的临时搜索历史逻辑。
- 若语音能力异常，不阻塞文本搜索主路径。

## 验收重点

1. 最近搜索生命周期明确。
2. 问小趣 query 与搜索历史分离。
3. 语音输入只承担 ASR 转词职责。
