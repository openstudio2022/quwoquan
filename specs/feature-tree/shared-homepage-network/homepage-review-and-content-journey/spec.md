# L2 Journey: homepage-review-and-content-journey

## 节点定位

- `L1_capability`: `shared-homepage-network`
- `L2_journey`: `homepage-review-and-content-journey`

该 Journey 冻结“用户如何围绕某个主页完成理解、比较、浏览内容与继续贡献内容”的完整链路。  
它承接主页总览、口碑摘要、内容聚合、问答聚合、相关群组入口与主页上下文下的发布入口，不承接认领审核与下线治理。

## 背景与动机

当前主页即使建出来，也容易停留在“有一个详情页”的层面，仍有四个旅程缺口：

1. **看不懂**：用户进入主页后，无法在首屏快速理解“这是什么、值不值得继续看、适不适合我”。
2. **看不全**：口碑、内容、问答与相关群组之间缺少统一组织，容易让主页变成零散模块堆砌。
3. **回不来**：用户从主页进入发布、提问、写口碑后，若没有主页上下文和回流语义，主页就无法形成长期沉淀。
4. **层级会串味**：主页要承接内容和口碑，但不能替代群组或内容域自己的详细治理。

本次 Journey 的目标，是让主页成为一个稳定的阅读与回流容器：

- 用户能在首屏理解该具体事物的核心信息和口碑摘要
- 用户能稳定浏览主页里的内容、问答和相关群组
- 用户能从主页上下文发起新的笔记、作品、提问或口碑
- 所有贡献都能回流主页，而不让主页侵入其他域边界

## 目标用户

- **浏览用户**：快速判断某个主页是否值得继续看、是否值得信任、是否与自己相关。
- **比较型用户**：通过评分摘要、内容和问答辅助决策。
- **贡献型用户**：在浏览主页后进一步发布笔记、作品、提问或口碑。
- **关系型用户**：从主页进一步进入相关群组，形成长期互动。

## 核心旅程

1. 用户从搜索、内容卡片、推荐列表或相关群组入口进入某个主页。
2. 用户在主页首屏看到名称、类型、位置/品牌/系列、评分摘要、关键亮点和主操作。
3. 用户进一步浏览主页的口碑、内容、问答与相关群组。
4. 用户可从主页上下文发起：
   - 发布笔记
   - 发布作品
   - 提问题
   - 写口碑
5. 用户发布完成后，内容回流到主页对应聚合面，并可继续浏览主页。

## 特性树拆分

本 Journey 冻结为 4 个 `L3_scenario` 方向：

| L3 Scenario | 负责的问题 | 归属域 |
|---|---|---|
| `homepage-overview-and-module-shell` | 主页总览、模块化骨架、首屏关键信息与独立降级 | `shared-homepage-network` |
| `homepage-review-read-and-score-summary` | 评分摘要、维度分、标签分布、精选口碑与口碑读取体验 | `shared-homepage-network` |
| `homepage-content-and-question-aggregation` | 笔记/作品/提问/口碑聚合，问答与内容区组织 | `shared-homepage-network` + `content` |
| `homepage-contextual-publish-entry` | 从主页进入发布动作并继承主页上下文 | `shared-homepage-network` + `content` |

## 功能范围

### In Scope

- 主页总览区、模块骨架和模块独立降级。
- 评分摘要、维度摘要、精选口碑与口碑入口。
- 主页内容区、问答区与相关群组区。
- 从主页上下文发起 `发布笔记 / 发布作品 / 提问题 / 写口碑`。
- 发布后回流主页对应聚合面的规则。

### Out of Scope

- 主页搜索和主页补充流程。
- 认领审核与维护台。
- 下线、恢复、归档与治理后台。
- 多主页比较器和复杂相关推荐图谱。
- 群组详细模板与群治理。

## 约束

- 主页首屏必须在不滚动或少量滚动内回答“这是什么、值不值得看、我接下来能做什么”。
- 主页模块必须独立加载和独立降级，单模块失败不影响其它模块渲染。
- 口碑摘要只读消费主页口碑模型，不在本 Journey 内重新定义评分写入协议。
- 主页中的相关群组只消费群组摘要，不承接群组详细治理。
- 从主页进入发布时，当前主页必须自动作为默认上下文带入。
- 主页的内容区和问答区是主页聚合面，不是内容域详情页的替代品。

## 对标输入与吸收结论

| 对标 | 借鉴点 | 本次吸收 |
|---|---|---|
| 大众点评 | 首屏先看评分、标签、精选内容，再深入浏览 | 吸收为主页总览 + 评分摘要 + 精选内容 |
| 汽车之家 | 车型页中口碑、资讯、论坛并存 | 吸收为主页内容/口碑/群组并存，但边界分层更清楚 |
| Booking/Airbnb | 酒店页信息、评价与行动入口集中 | 吸收为“首屏摘要 + 进入下一步动作”组合 |

## 角色分工

| 角色 | 职责 |
|---|---|
| `shared-homepage-network` | 主页总览、模块骨架、评分摘要、相关群组区、上下文发布入口 |
| `content` | 内容聚合、提问聚合、发布完成后的回流 |
| `circle` | 相关群组摘要与跳转 |

## 既有 Story 覆盖矩阵

| 当前实现 / 历史能力 | 处理方式 | Journey 内新归属 |
|---|---|---|
| 内容详情页中的零散对象信息卡 | 收敛为主页总览与摘要模块 | `homepage-overview-and-module-shell` |
| 现有内容域聚合流 | 保留内容域为来源，主页只做聚合消费 | `homepage-content-and-question-aggregation` |
| 主页入口中的临时“写口碑/发内容”跳转 | 统一为带主页上下文的发布入口 | `homepage-contextual-publish-entry` |

## 数据生命周期合同

- 主页总览与评分摘要只消费已发布主页及其已发布内容/口碑摘要。
- 主页中的内容与问答聚合以内容域为真相源，主页只做聚合展示。
- 主页上下文发布完成后，结果回流到主页聚合面，不反向修改主页主档。
- 口碑数量、评分摘要和内容数允许异步刷新，但需保证最终一致。

## 权限与异常边界

- 主页阅读默认遵从内容与群组现有可见性，不在本 Journey 再新增一套权限模型。
- 某个模块无数据时，应显示空态而不是整体错误页。
- 某个模块加载失败时，应显示模块级重试，不阻塞整个主页。
- 若主页上下文发布失败，用户应保留当前编辑内容并可重试。

## 非功能目标

### SLO

- 主页首屏骨架 `p95 < 300ms`
- 主页关键摘要（名称、评分、主操作）可见时间 `p95 < 1.2s`
- 口碑摘要或内容聚合首批结果 `p95 < 1.5s`

### KPI

- 主页首屏进入后 10 秒内有效互动率 `>= 60%`
- 从主页进入发布动作的链路完成率 `>= 80%`
- 主页内容回流展示成功率 `>= 99%`

### 弱网与恢复

- 弱网下优先显示总览骨架、评分摘要骨架和主操作。
- 非关键模块可延后加载，不阻塞首屏。
- 发布完成后的回流刷新失败时，允许用户手动下拉刷新，不影响内容已发布成功语义。

## 迁移、灰度与回滚要求

- baseline 不保留旧的主页碎片卡片并行治理，应统一收口到主页模块骨架。
- 若主页内容聚合、评分摘要或上下文发布入口出现重大稳定性问题，可回退到“主页只读摘要，不承接上下文发布”的保守方案。
- 不允许在无说明情况下，把主页重新退化为单一信息卡或纯目录页。

## 验收重点

1. 用户能通过主页首屏快速理解具体事物的核心信息和口碑摘要。
2. 用户能稳定浏览主页中的内容、问答和相关群组。
3. 用户能从主页上下文发起发布，并把结果回流到主页。
4. 主页作为聚合面成立，但没有侵入内容域与群组域的边界。
