# L3 Scenario: homepage-search-and-picker

## 节点定位

- `L1_capability`: `shared-homepage-network`
- `L2_journey`: `homepage-discovery-and-attach-journey`
- `L3_scenario`: `homepage-search-and-picker`

## 背景与动机

主页挂载的第一步不是写内容，而是先找到正确主页。  
如果搜索结果无法提供足够的区分信息，用户会在同名品牌、门店、景点或车型之间误选，直接破坏后续口碑与内容的可信度。

## 目标用户

- 在发布内容时需要快速关联主页的用户。
- 通过搜索直接寻找某个具体事物主页的用户。

## 功能范围

- 主页搜索输入与候选结果列表。
- 结果摘要：名称、类目、位置/品牌系列、封面、评分摘要。
- 主页选择器与选中后的回填结果。
- 最近搜索或默认推荐可作为增强项，但不改变主路径。

## Out of Scope

- 搜索不到主页时的补充主页流程。
- 主页详情页完整阅读体验。
- 发布写入与回流聚合。

## 约束

- 前台统一使用“主页”和具体类目名。
- 搜索结果必须能区分高频同名主页。
- 选择器必须支持返回当前上下文，而不是打断主流程。

## 对标输入与吸收结论

- 借鉴小红书与 Booking/Airbnb 的“搜索结果带足够摘要信息”。
- 吸收“先选对象，再进入后续动作”的低认知负担链路。

## 角色分工

- `shared-homepage-network`：主页搜索结果与选择器
- `search`：结果召回与排序
- `_shared` metadata：route、surface、request context

## 数据生命周期合同

- 搜索结果只展示已发布主页。
- 选择结果只在当前上下文内临时保留，直到用户确认下一步动作。

## 非功能目标

- 首批结果 `p95 < 800ms`
- 选择器可交互时间 `p95 < 1.2s`

## 迁移、灰度与回滚要求

- 不保留“自由文本写具体事物”的并行旧路径。
- 如搜索稳定性不足，可短期关闭主页挂载入口，但不能破坏主页本身浏览。

## 验收重点

1. 用户能找到并区分正确主页。
2. 主页选择器可稳定回填结果。
3. 结果摘要足以降低误挂载。
