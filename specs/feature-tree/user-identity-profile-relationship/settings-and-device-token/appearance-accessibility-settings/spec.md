# L3 规格：appearance-accessibility-settings — 外观与字号偏好同步设置

> **层级**：L3_story（隶属 L2 `settings-and-device-token`）
> **状态**：specified

## 0. 一句话定义

建立 `owner 默认值 -> 子账号继承 -> 子账号可覆写 -> 可同步全部账号` 的外观与字号偏好设置体系，并支持跨账号、跨端、弱网与离线补同步。

## 1. 背景与动机

在 `app-theme-infrastructure` 建立全局视觉运行时之后，用户还需要一个稳定、可解释、可同步的偏好模型，否则会出现：

1. **设置只在当前端临时生效**：切设备、重装或重新登录后外观和字号丢失。
2. **多子账号体验割裂**：owner 与子账号之间没有默认继承与覆写规则，用户难以理解“为什么这个账号跟那个账号不一样”。
3. **弱网与离线不可恢复**：本地修改后如果服务端未成功写入，界面与真实设置状态会分裂。
4. **缺少审计与冲突规则**：跨账号、跨端同时改外观时，没有统一的广播、回写和冲突解决策略。

本 Story 负责“设置值的模型、读写、继承、同步和审计”；视觉效果本身由兄弟 Story `app-theme-infrastructure` 承担。

## 2. 目标用户

- 单账号用户：希望主题和字号在当前设备与后续登录设备上保持一致。
- 多子账号用户：希望 owner 的外观默认值自动继承到子账号，同时允许某个子账号做个性化调整。
- 高频切号用户：希望在切换子账号、切换设备、弱网或离线后，外观设置仍可预测、可恢复。

## 3. 功能范围

### 3.1 In-Scope

| 编号 | 功能 | 说明 |
|---|---|---|
| SET1 | 偏好模型 | 定义外观与字号偏好模型，首发至少包含 `themeMode(system/light/dark)` 与 `fontSizePreset` |
| SET2 | owner 默认值 | owner 可读写自己的外观默认值，作为所有未覆写子账号的默认来源 |
| SET3 | 子账号继承 | 子账号未设置 override 时，自动继承 owner 默认值 |
| SET4 | 子账号局部覆写 | 子账号可以只覆盖自己的外观/字号偏好，不影响其他子账号 |
| SET5 | 同步全部账号 | 子账号修改时提供“同步所有账号”选项，默认勾选；勾选后写入 owner 默认值，并让全部子账号收敛到新的统一默认值 |
| SET6 | 本地优先与待同步标记 | 修改后先本地生效；离线或失败时保留待同步状态，联网后自动补同步 |
| SET7 | 跨端同步 | 同一 owner 体系内的设置在多设备间同步，正常网络 `p95 <= 3s`，弱网最终一致 `<= 10s` |
| SET8 | 冲突与审计 | 同一作用域采用 `last-write-wins`；记录作用域、操作者、是否全量同步、旧值、新值、时间、设备 |

### 3.2 Out-of-Scope

- 视觉 token、Cupertino-first 壳层与全量页面视觉刷新本身（由 `app-theme-infrastructure` 承担）
- 通知、隐私、设备 token 等其他设置项
- 跨 owner 账号族的设置共享
- 高级协同冲突合并策略（如字段级 merge、审批流）；首发仅采用 `last-write-wins`

## 4. 对标输入与吸收结论

| 输入 | 借鉴点 | 不借鉴点 | 本次吸收结论 |
|---|---|---|---|
| Apple HIG | 分组设置页、继承关系说明、状态可解释性、同步状态表达克制 | 仅依赖 iCloud/苹果生态的强绑定方案 | 作为设置页视觉与可解释性交互基线 |
| Instagram | 主题偏好作用于全局内容体验，切换后应立即稳定生效 | 选项层次较浅，不足以承载多账号继承 | 借鉴“设置一改，全局立刻可见”的体感标准 |
| 小红书 | 用户对阅读字号和内容舒适度敏感 | 字号设置入口分散、解释性一般 | 借鉴“字号直接影响阅读效率”的产品优先级 |
| 微信 | 多账号/高频切换场景对一致性要求高 | 设置体系偏工具化、缺少显式继承模型 | 借鉴切号后状态稳定与低认知负担 |
| 抖音 | 沉浸式内容页对主题和字号切换稳定性要求高 | 设置项展示较少，不适合作为复杂设置页模板 | 借鉴“设置不能破坏沉浸式页面稳定性” |

## 5. 约束

- 必须挂在 `settings-and-device-token` 下，以 metadata / API / audit 为真相源，禁止做成仅本地存储能力。
- 首发默认关系为：`subAccountOverride ?? ownerDefault ?? systemDefault`。
- 子账号修改时，“同步所有账号”默认勾选；取消勾选才允许只写当前子账号 override。
- 本地缓存仅用于加速与离线恢复，不能成为最终真相源。
- 本 Story 只定义设置值与同步规则，不负责页面视觉运行时实现；必须与 `app-theme-infrastructure` 通过清晰契约对接。

## 6. 角色分工

- 产品：冻结 owner/子账号继承与同步广播规则，确认默认交互文案与作用域说明。
- 架构：设计偏好模型、作用域、同步协议、离线补同步、审计与冲突解决策略。
- 开发：实现设置页入口、读写链路、pending 状态、广播更新、跨端恢复。
- 测试：覆盖 owner、子账号、同步全部账号、弱网、离线、冲突与恢复链路。
- 发布：统一生效、简单位为主，优先保证正确性与可回滚。

## 7. 非功能目标

- 当前账号本地生效：`<= 100ms`
- 全 app 主题/字号切换稳定完成：`<= 300ms`，不闪屏
- 跨账号/跨端同步：正常网络 `p95 <= 3s`
- 弱网最终一致：`<= 10s`
- 离线：本地先应用，恢复联网后自动补同步；失败时保留待同步标记
- 冲突：同一作用域 `last-write-wins`
- 子账号 override 只覆盖自己；勾选“同步所有账号”时写入 owner 默认值并广播
- 灰度：统一生效，简单位为主

## 8. 验收重点

### T1

- 偏好模型、作用域、同步语义、审计字段和 API 契约清晰且唯一
- 冲突与继承规则可通过单元/契约测试稳定验证

### T2

- 设置页明确展示“继承 owner 默认 / 当前账号已覆写 / 同步所有账号”状态
- 用户在任一账号修改主题/字号后，当前 UI 立即稳定反映结果

### T3

- owner 修改默认值可广播到未覆写子账号
- 子账号局部覆写仅影响自己
- 子账号勾选“同步所有账号”后可回写 owner 默认并广播
- 跨端登录、切号、断网恢复后状态一致

### T4

- 真机验证手机、平板、横屏、分屏、桌面窗口下的设置页与实际生效结果一致
- 弱网、断网、恢复联网、双端并发修改场景下最终结果符合 `last-write-wins` 和审计预期
