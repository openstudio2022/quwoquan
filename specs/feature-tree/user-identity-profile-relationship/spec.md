# L1 规格：用户身份画像与关系

## 范围
- 欢迎页入口、认证、画像快照、分身、关系网络、设置与设备 token。

## 功能说明
- 提供从欢迎页到登录后身份建立的统一入口流程，降低首启与回访断点。
- 提供用户画像、分身与关系网络能力，支撑推荐、聊天、助手等跨域场景。
- 提供系统设置与设备 token 注册能力，保障通知、隐私与多端一致。

## 约束
- 欢迎页（`welcome_screen.dart`）作为登录前后入口，必须归于本 L1。
- 认证与 profileUpdateProposal 状态流转采用强一致约束。
- persona 切换必须可追踪并透传到下游上下文。

## 验收标准（L1 重点）
- A1：欢迎页到认证再到用户资料链路完整。
- A3：身份与提案状态机具备幂等和回滚策略。
- A6：PII/SENSITIVE 字段分级、脱敏、加密与保留策略生效。
- A8：核心身份流程自动化测试覆盖。
