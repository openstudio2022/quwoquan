# 文章编辑器完全重构 — 功能规格（归纳版）

> **状态**：规格已冻结（含文内图环绕与尺寸），**可进入开发实施**；与 `design.md`、`acceptance.yaml`、`plan.yaml` 同源。  
> **范围**：沉浸文章创作路径（`EditorStartAction.write` + `CreateEditorKind.text` + 文章编辑壳），不含独立「图片滤镜编辑页」(`image_editor_page`)。

---

## 1. 文档目的

将前期讨论澄清的需求归纳为**单一真相源**，作为重构实现的验收基线。重构目标：**编辑态实现与阅读态一致的「卡片侵入式」所见即所得（WYSIWYG）**，并在 **iOS 原生画质、元数据驱动、横向质量门禁** 下交付可商用品质。

---

## 2. 核心设计原则

| 原则 | 说明 |
|------|------|
| **WYSIWYG** | 编辑画布上呈现的排版（标题层级、列表、图片与正文相对位置、对齐、行内样式）与发布后阅读态 **页面、结构与节奏一致**；禁止「仅改数据、编辑区仍为占位堆叠」的半成品。 |
| **精品长文默认成立** | 即使用户只输入正文、插图、偶尔设置标题，也应自动得到 **舒适页感、段落呼吸、图文节奏与层次感**；用户**不必**理解 H1/H2、段前距等术语。 |
| **双态单一分页真相** | 编辑态与预览态共用 **同一 page metrics 与 page slice 结果**；页面尺寸、页眉、页码、caption 占高与图片测量规则 **不得漂移**。 |
| **卡片侵入式阅读感** | 单页/单块内容以 **卡片级容器** 呈现（圆角、表面层级、适度留白），编辑时即感受到 **最终阅读时的块级节奏**，而非纯 TextField 灰底。 |
| **键盘与面板等价占位** | 各功能面板占据 **与系统键盘同阶高度**（可用快照高度 + 断点约束），切换时 **工具栏锚点不动**，避免布局跳动。 |
| **单一切换语义** | 打开任一面板时，对应工具项变为 **键盘图标**；点键盘回到文字输入。 |
| **元数据唯一真相** | 发布载荷中的块类型、布局枚举、富文本序列化格式以 **`contracts/metadata`** 为准，经 codegen 消费；禁止 UI 维护第二套 path/字段表。 |
| **高品质横切** | 遵守 `specs/02_IOS_NATIVE_FRONTEND_UX_SPEC.md`、语义 token（`AppSpacing` / `AppTypography` / `AppColors`）、`UITextConstants`+l10n；页面级改动同步 **page-horizontal-quality 矩阵与 P2 清单**。 |
| **iOS 创作器质感** | 顶栏、主 CTA、页材质与过渡需对齐高品质图片编辑器/沉浸创作壳，避免“功能对了但像表单”的低质感实现。 |

## 3. 信息架构总览

```
┌─────────────────────────────────────────────────────────────┐
│  [关闭]  标题   [编辑|预览]（紧凑，不占独立整行高度）  [草稿][下一步] │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   编辑态：纵向浏览可编辑纸页（页框/页眉/页码齐备，与预览同尺寸）             │
│   预览态：横向翻页沉浸阅读（与侵入式浏览器一致；前翻/回翻动效）             │
│   · 同一文档模型；同一分页切片；跨页图不纵向裁切                        │
│                                                             │
├─────────────────────────────────────────────────────────────┤
│ 图 │ emoji │ 样式 │ 序号 │ 排版  │  undo │ redo   （仅编辑态） │
└─────────────────────────────────────────────────────────────┘
         ↑ 左组 5 项          ↑ 分割条      ↑ 右组记录
```

**说明**：`[编辑|预览]` 为 **模式切换**（非「卡片 vs 瀑布流」）；实现上置于 **导航栏 trailing 区** 或与标题同行的 **紧凑分段控件 / 双图标**，**禁止**再使用占满一行的 `CupertinoSlidingSegmentedControl` 独立条作为顶栏第二行（与现网占位 IA 切割）。

## 4. 顶栏规格

| 元素 | 行为 |
|------|------|
| **关闭** | 退出前走草稿保存/确认策略（与现网一致或可强化）。 |
| **标题** | 展示当前模式文案（如「文章编辑」），走 l10n / `UITextConstants`；可选在预览态微调副标题或页码指示（design 定稿）。 |
| **模式切换（冻结）** | **编辑** ↔ **预览**：同一 `ArticleDocument` / **同一分页快照**；**编辑** = Word 式纵向多页滚动 + 可编辑；**预览** = **只读**，横向分页，**与侵入式媒体浏览器（`WorksImmersiveViewer` / `UnifiedMediaViewerPage`）展示品质与手势节奏对齐**，翻页带 **翻书向前翻/回翻** 动效（见 `design.md` §5.4）。控件 **紧凑**，置于 **导航栏右侧区域**，**不**单独占据一行垂直高度。 |
| **草稿** | 与现逻辑对齐（编辑态可用；预览态策略见 design）。 |
| **下一步** | 进入发布确认链；与顶栏材质、强调色、字号层级 **与当前沉浸创作壳及图片编辑流「下一步」可衔接**（iOS token 一致）。 |

**品质**：顶栏高度、分隔、模糊/材质、按钮热区与「图片编辑器 / 沉浸创作壳」一致，`下一步` 主 CTA 与其强调色、字重、位置关系一致；禁止新增未登记的第二套顶栏组合（参见 `page-layout-semantics`）。

**废弃**：原「卡片编辑 / 瀑布流编辑」顶栏第二行切换 **不再作为产品 IA**；若需缩略导航，归入编辑态内辅助能力（design 可选），不得复活为顶栏主切换。

## 5. 底栏规格（固定结构）

### 5.1 左组（五项，顺序固定）

1. **图片**：插入/替换当前上下文图片；可进入环绕子选项（通栏 / 左环绕 / 右环绕）。  
2. **Emoji**：表情面板。  
3. **样式**：行内样式（粗体、斜体、下划线、删除线，**多选**）、段落对齐（左 / 中 / 右 / 两端，**单选**）、文本层级（**正文 / 小标题 / 大标题**，见 §7）。  
4. **序号**：有序列表 / 无序列表；**层级最多 3 级**（缩进或子列表语义）。  
5. **排版**：**一个入口**，面板内 **两个 Tab**——**模版** | **字号**（含封面等模版相关能力可归入模版 Tab，与现能力对齐）。

### 5.2 分割条

- 左组末与右组首之间 **垂直分割条**（`separator` 语义色、hairline）。  
- 视觉上区分「内容编辑工具」与「记录操作」。

### 5.3 右组

- **Undo** / **Redo**：对 **当前编辑文档** 的撤销/重做（文本与结构变更均需入栈；与打开面板状态 **正交**——undo/redo **不强制关闭面板**，除非实现上冲突需在 design 说明）。  
- 不可用时 **置灰**；可用时符合对比度与触控热区（≥44pt）。

### 5.4 与键盘、面板的联动

- 点击左组某项：若当前为键盘输入 → **收起键盘**，展开对应面板（高度≈键盘区）。  
- 面板打开时，**仅该项**显示为 **键盘图标**；点击后 **关闭面板并恢复焦点到原输入上下文**。  
- 再次点击同一项（在未显示面板时）：可切换至该面板（与现 `_togglePanel` 语义一致）。

## 6. 各面板内容规格

### 6.1 样式面板（参考「多行四列」布局）

- **布局**：多个 **区块**；每区块 = **区段标签一行** + **一行四个等宽按钮**（或四槽位栅格）。  
- **行 1 — 行内样式**：粗体、斜体、下划线、删除线；**多选同时生效**于当前选区或后续输入。  
- **行 2 — 对齐**：左、中、右、两端；**单选**。  
- **行 3 — 文本层级**：**正文**、**小标题**、**大标题**；**单选**。面向用户**不得**暴露 `H1/H2/H3/H4` 术语，内部映射关系在 design 中冻结。  

**实现注意**：行内样式与对齐依赖 **富文本模型**（非纯 `CupertinoTextField` 单字符串）；规格层要求 **编辑与渲染共用同一套 span/块模型**。

### 6.2 序号面板

- 有序 / 无序切换；**嵌套层级 ≤ 3**（展示为缩进或层级 chip）。  
- 与样式面板中的「列表」不重复：此处为 **专项序号与层级**，样式面板可不再承载列表（避免重复，以 design 定稿为准）。

### 6.3 排版面板

- **Tab A — 模版**：模版缩略、封面选择等（与现有 `ArticleTemplatePreset` 能力对齐并合并入口）。  
- **Tab B — 字号**：字号档位 / 字体预设（与现有 `ArticleFontPreset` 对齐）。

### 6.4 Emoji 面板

- 与键盘同高、可滚动；最近使用 + 全量分类；选中插入至当前选区。

### 6.5 图片

- 选图、裁剪/编辑跳转策略与现网一致；**环绕方式变更必须在编辑画布上立即反映**（§8）。

## 7. 主编辑视口：WYSIWYG 与卡片侵入式

### 7.1 卡片侵入式

- 每个 **逻辑块**（段落、标题、列表项、图文组合等）使用 **统一卡片表面**（圆角、边框/阴影语义、内边距 token）。  
- **例外**：**文内图片位图本身**（含配图说明区域之上的**图块**）为 **矩形、无圆角**（见 §8.6），与卡片圆角可并存（图在卡内贴边或按 token 控制外框，但图像素区不裁圆角）。  
- **编辑态**即采用与 **预览/阅读** 相同或简化的 **同一套渲染组件**（或共享 layout delegate），避免「编辑简陋、预览华丽」双轨。

### 7.2 分页与自适应（流式 + 统一 page slice）

- **文档真相**：`ArticleDocumentData` 为唯一真相；**排版顺序**在文档流上连续计算（`ArticleFlowLayoutEngine`：`LayoutRun`/fragment 序列 + 测量高度），**禁止**将「横向预览页数」与「文内图张数」机械对齐。  
- **统一分页切片**：编辑态与预览态必须共用 **同一 page metrics 与 page slice 结果**；编辑只是在这些 page slices 上提供输入能力，预览则只读展示。  
- **页面边界**：若某元素（尤其通栏图）在当前页剩余高度内无法完整放置，则 **整块移到下一页**；编辑与预览页边界必须一致。  
- **页宽 / 页高**：页宽与页高均由统一 page metrics 决定；编辑态不得使用与预览不同的“无限高轻量页”作为终态。  
- **断点**：`compact / regular / expanded` 下内容区内边距用 `AppSpacing.responsiveValue`。

### 7.3 编辑态版式（Word 式 · 与预览同页）

- **滚动轴向**：**纵向** 连续列出所有 **可编辑纸页**；用户上下滑动浏览并编辑全文。  
- **页面壳**：编辑态每一页都必须具备 **明确页框、页眉、页脚/页码与固定页范围**；其 **页宽/页高/内容区** 与预览态一致，仅允许弱化沉浸背景和阴影强度。  
- **外框约束**：**禁止** 再给整列编辑区包一层粗外框；只保留 **单页纸张** 的细边框 + 轻阴影，形成“多页堆叠”质感。  
- **页码**：多页时强制显示 `n/m` 或等价本地化页码，编辑与预览一致。  
- **与块级卡片**：§7.1 块级节奏仍适用于正文块；编辑态终态不是“无限高轻量页”，而是“弱沉浸、可编辑、同页尺寸”的纸页。

### 7.4 预览态版式（侵入式媒体浏览器一致）

- **只读**：不弹出键盘、不触发正文 `TextField` 编辑；可选轻量 chrome（页码点、轻触显隐）与 `WorksImmersiveViewer` 节奏对齐。  
- **导航**：**横向**分页，**一页一屏**（或等价 viewport），手势与 **作品沉浸浏览器** 内横向翻图/翻帖的 **跟手度、阻尼、边缘提示** 同一品质条。  
- **动效**：翻至下一页 / 返回上一页时具备 **书本翻页感**（铰链/卷曲近似，禁止裸 `PageView` 无过渡硬切作为终态）；实现细节见 `design.md`。

### 7.5 跨页图（编辑与预览共用分页规则）

- 若某元素（尤其 **通栏图**）在当前页 **剩余高度内无法完整放置**，则 **整块移到下一页** 起始处展示；**禁止**一图纵向裁切跨两页。

### 7.6 ~~视图模式（顶栏切换）~~（已废止）

- 原 **卡片 / 瀑布流** 仅改变缩略导航的规格 **废止**；由 **§4 编辑/预览** 替代。

## 8. 文内图与环绕文字（统一规格，可开发）

> **坐标系**：凡称「半屏」「宽度占比」，均相对于 **主编辑内容区宽度**（`content area`）——即 **扣除页面/卡片水平内边距后的可排版宽度**，**不是** 全屏宽度、**不是** 含系统侧栏的 window 宽。实现时用 **LayoutBuilder / 内容区约束** 取宽，**禁止**用 `MediaQuery.sizeOf(context).width` 直接乘 0.5 作为唯一依据（除非已显式等同内容区）。

### 8.1 三种布局与环绕语义

| 布局枚举（与 metadata `imageLayout` 一致） | 语义 |
|------------------------------------------|------|
| **通栏** | 图片在内容区宽度方向 **占满**（或按模版为全宽），正文在图下/上按模版；**无**左右环绕列。 |
| **左环绕** | 图片在版式中偏左，**文字可占据图片的上方、下方与右侧**（三面环绕），**不是**「图独占左半屏、右侧空白」。 |
| **右环绕** | 对称：**文字可占据图片的上方、下方与左侧**。 |

- 图在 **当前光标之后** 插入，在文档结构上为 **独立图块**（块级）；**上方/下方** 与正文流衔接，且水平方向 **左右对齐、拉满内容区**（除环绕侧占用宽度外）。

### 8.2 默认尺寸（宽 → 高）

- **默认宽度**：文内图在左/右环绕模式下，**默认占用内容区宽度的 1/2**（50% of content area width）。  
- **高度**：在 **宽度已定** 的前提下，按 **图片固有宽高比** 计算显示高度（策略：**contain** 等比，具体见 `design.md`）；高度随 **换行与环绕列行数** 允许 **微调**，使 **环绕列文字与图块在竖直方向齐顶、齐底**（见 §8.5）。  
- **通栏** 模式宽度按 §8.1 表（占满内容区），高度仍按宽高比。

### 8.3 段落、换行与「环绕段」的划分

- **上文**：若光标 **上方** 同一段内 **尚未产生分段换行**（如仅软换行），则 **同一段落语义** 内可同时包含：**环绕侧正文** 与 **图块上方** 的通栏文字。  
- **若先分段（新段落）再插入图**：自 **新段落** 起应用左/右环绕规则。  
- **下文**：同理——根据 **是否新起一段** 决定与环绕区是否 **同段连续**；**换行输入** 时布局应 **自适应**。

### 8.4 竖直对齐与间距（行距 vs 段距）

- **与图块相邻的上下文文字** 与 **图块顶/底** 的间距规则：  
  - **同一行内自然折行、未形成新段落** → 使用 **行间距**（或等价「上行间距」）控制与图块的间距。  
  - **已分段**（新段落边界）→ 使用 **段落间距**。  
- **环绕列** 内文字：与 **图块** 在 **竖直方向** 对齐——**上方** 正文最后一行与 **图区顶**（或约定基线）对齐；**下方** 正文第一行与 **图区底** 对齐；若因行高/行数导致 **轻微不齐**，允许在 **保持默认宽与比例** 的前提下 **微调显示高度**（仅用于视觉对齐，见 §8.2）。

### 8.5 配图说明（小字）

- 若图 **下方有配图说明**，则 **「图块」在竖直方向的占用** = **图 + 说明区**（说明参与总高度）。  
- **空说明默认不占高度**；编辑态、预览态与流式测量必须遵循同一规则，禁止一端占位一端塌缩。  
- **下方接续的正文** 应对齐到 **配图说明的下沿**（即「底」对齐在 **说明之下**，而非仅图片下沿）。  
- 说明样式：小字、与正文区分、**居中对齐**；**可序列化** 字段在 metadata / `articleDocument` 中定义（与现有 `caption` 等字段对齐，见 codegen）。  
- **说明编辑入口**：优先从图片工具栏进入；不要求在画布上长期保留一个空 caption 输入框。

### 8.6 交互与视觉（文内图）

| 项 | 要求 |
|----|------|
| **点击图** | **选中图片并显示图片工具栏**（通栏 / 左图右文 / 右图左文 / 说明 / 删除）；再次触发编辑时进入 **图片编辑页**（独立编辑流；与 `image_editor_page` 等现网策略对齐）。 |
| **删除** | 图 **右上角** 提供 **删除** 控件（符合触控热区与无障碍）。 |
| **圆角** | **文内图无圆角**（矩形）；与 §7.1 卡片表面例外一致。 |

### 8.7 编辑态与阅读态一致

| 布局 | 编辑态要求 |
|------|------------|
| **通栏** | 与 §8.1 一致；正文相对位置与阅读投影一致。 |
| **左环绕 / 右环绕** | 编辑区必须为 **图文分栏的同一视觉结构**（如 `Row`：图 **按 §8.2 定宽** + `Expanded` 环绕正文，或与 `ArticleWrappedParagraph` **共享测量**），**禁止** 仅数据为 wrap、界面仍为「上图下文」。 |
| **阅读侧** | `ArticleWrappedParagraph` / 阅读态投影与编辑态 **共用布局或同一 delegate**，避免双轨。 |

### 8.8 数据与契约

- 布局枚举、图宽策略、**caption** 等字段以 **`contracts/metadata`** 为唯一真相源；`articleDocument` 内块与 assets 序列化与 `article_document_schema.yaml` 一致。  
- 发布载荷仍通过 `CreatePost` **writable_fields**；禁止在 UI 维护第二套字段表。

## 11. 非目标与风险（需在 design 中逐项回应）

- **非目标**：本文不规定独立图片滤镜编辑页的全部工具链；仅规定与文章插图相关的选图与布局。  
- **风险**：富文本 + undo/redo + 分页测量 **复杂度高**，需单独 **技术方案比选**（自研 vs 成熟库）与 **性能预算**（大文档滚动、重建）。  
- **风险**：H1 与全局标题字段是否合并，需产品一次性冻结。

## 12. 建议交付顺序（供 `plan.yaml` 切片）
