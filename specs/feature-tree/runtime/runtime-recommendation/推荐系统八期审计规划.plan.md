## P0：阻断级（数据管道断裂 / 核心指标失真）

### B-01: 社交召回查询与 MongoSink 写入 schema 不匹配（D5+D8 阻断）

- **问题**：`social_infra.go:GetFriendInteractedContent` 按 `eventType∈{like,favorite,share}` + `ts` 过滤/排序；而 `MongoSink.FlushEvents` 写入 `eventType=rec_engagement`（行为 action 在 `labels.action`）+ `createdAt`。**好友互动召回恒空。**
- **修复**：
  - 方案 A（推荐）：社交召回改读 `behavior_events` 集合（ContentReacted 时写入），按 `action∈{like,share}` + `createdAt` 查询
  - 方案 B：在 learning sink 写入时增加 `action` 顶层字段 + `ts` 别名
- **文件**：`social_infra.go`，可能 `behavior_service.go`
- **影响**：D5 +1.5, D8 +1.5

### B-02: depthDistribution 键语义 Go/Python 不一致（D7 阻断）

- **问题**：Go `UserFeatureVector.DepthDistribution` 为 `map[int]int`，JSON 序列化为 `"0".."4"`；Python `train.py` / `content_feed.py` 用 `depth_dist.get("L0")..get("L4")` — **离线训练深度分布特征全零。**
- **修复**：
  - Go 侧 `FeatureStore.GetFeatures` 在写入 `UserFeatureVector` 前将 int 键转为 `"L0".."L4"` 字符串
  - 或 Python 侧统一用 int 字符串键 `"0".."4"`
- **文件**：`recommend_feature.go`（或 `feature.go` 类型定义）+ `train.py` + `content_feed.py`
- **影响**：D7 +1.0

### B-03: UnifiedMediaViewerPage 旁路 engagement tracker（D1 阻断）

- **问题**：当 `MediaViewerExtra.dtoPosts` 不为空时，`app_router.dart` 走 `UnifiedMediaViewerPage`，而该页面**无任何** `ContentEngagementTracker` 调用。首页 Following、圈子创作、搜索结果、个人主页作品/微趣等主流路径**全部丢失深度行为数据。**
- **修复**：在 `UnifiedMediaViewerPage`（或其内部 `WorksImmersiveViewer`）中接入 `ContentEngagementTracker`：
  - 页面可见时 `trackContentEnter`（带 referralSource/feedRequestId/authorId/contentType）
  - 切换帖子时先 `trackContentExit` 再 `trackContentEnter`
  - 视频播放时 `trackPlayProgress`
  - 页面 dispose 时 `trackContentExit`
- **文件**：`unified_media_viewer_page.dart` + 可能 `works_immersive_viewer.dart`
- **影响**：D1 +1.5, D2 +0.5

### B-04: post_feed_contract_test 内容类型 "image" → "photo"（D10）

- **问题**：`post_feed_contract_test.go` 创建请求仍用 `"contentType":"image"`，与推荐管线 / ML 的 `photo` 不一致。
- **修复**：全文替换 `"image"` → `"photo"` 作为 contentType 值
- **文件**：`post_feed_contract_test.go`
- **影响**：D10 +0.5

## P1：归因准确度 + 特征准确度

### B-05: ArticleDetailPage 路由不接收归因参数（D2）

- **问题**：`app_router.dart` 中 `ArticleDetailPage` 路由仅传 `id`，不传 `referralSource` / `feedRequestId`。所有文章消费被错标为 `organicFeed` + `feedRequestId=null`。
- **修复**：
  1. 定义 `ArticleDetailPageRouteExtra`（含 referralSource + feedRequestId）
  2. 路由 builder 读取 extra 并传入 `ArticleDetailPage`
  3. 所有 push `articleDetail` 的调用方传入正确 extra
- **文件**：`app_router.dart`, `article_detail_page.dart`, `discovery_page.dart`, `profile_works_tab.dart`, `search_network_results_page.dart`, `section_creations.dart`
- **影响**：D2 +0.5

### B-06: Moment 帖在 following 分类下被错标为 photo（D1）

- **问题**：`PhotoDetailPage._resolvedContentType` 仅匹配 `'moment'` 和 `'profile_moment'`；首页 Following 传入 `category:'following'` 时返回 `photo`。
- **修复**：在 `_resolvedContentType` 中增加逻辑：若帖子本身的 `contentType` 或 `type` 字段表明 moment，则返回 `ContentType.moment`
- **文件**：`photo_detail_page.dart`
- **影响**：D1 +0.3

### B-07: reportSingle/impression 缺 feedRequestId（D2）

- **问题**：`discovery_page.dart._trackBehavior` / `_trackFeedImpression` 和 `home_page.dart` 的 `reportSingle(click)` 不携带 `feedRequestId`，与 Viewer 会话的 `navFeedRequestId` 割裂。
- **修复**：在 `_trackBehavior` / `_trackFeedImpression` 中传入当前 feed session 的 `feedRequestId`
- **文件**：`discovery_page.dart`, `home_page.dart`
- **影响**：D2 +0.5

### B-08: socialPrior 与 authorAffinity 重复消费 AuthorAffinities（D4）

- **问题**：`RuleScorer` 中 `socialPrior` 和 `authorAffinity` 都从 `user.AuthorAffinities` 读取同一个 map，导致作者信号被双重加权。
- **修复**：`socialPrior` 移除 `AuthorAffinities` 消费，只保留 `CircleTagAffinities` + `SocialInterestScore`
- **文件**：`scorer.go`
- **影响**：D4 +0.3

### B-09: rerank 探索/冷启动插入破坏去重约束（D5）

- **问题**：`rerank` 中探索和冷启动从 buffer 插入时未校验 `maxPerAuthor` / `maxPerType`；回填阶段的 `typeCount`/`authorCount` 不含已插入的探索/冷启动项。
- **修复**：探索/冷启动插入后更新 `typeCount`/`authorCount`；或插入前检查约束
- **文件**：`engine.go`（rerank 函数）
- **影响**：D5 +0.5

### B-10: OtherProfilePage trackAuthorProfileView 恒为 author_profile（D2）

- **问题**：无论从搜索、信息流、圈子还是私信进入，`referralSource` 恒为 `author_profile`。
- **修复**：定义 `OtherProfilePageRouteExtra` 含 `referralSource`；路由 builder 传入；各调用点传实际来源
- **文件**：`other_profile_page.dart`, `app_router.dart`, 各调用点
- **影响**：D2 +0.3

### B-11: HomepageDetailPage 从搜索进入用 entity_page 而非 search（D2）

- **问题**：`search_network_results_page.dart` 的 `push homepageDetail(id)` 未传 extra，默认 `entity_page`。
- **修复**：搜索结果页 push 时传 `HomepageDetailPageRouteExtra(referralSource: ReferralSource.search)`
- **文件**：`search_network_results_page.dart`
- **影响**：D2 +0.2

### B-12: feature_registry.yaml item 特征未在 train.py 消费（D7）

- **问题**：注册表声明 `aspectRatio`、`recallPath` 为 item 特征，但 `train.py` / `train_multiobjective.py` 的 `_extract_features` 未将其编入特征向量。
- **修复**：在 `_extract_features` 中加入 `aspectRatio`（数值）和 `recallPath`（编码为 one-hot 或 hash）
- **文件**：`train.py`, `train_multiobjective.py`, `content_feed.py`(serving)
- **影响**：D7 +0.5

### B-13: rec_engagement Context 缺 referralSource/contentType/authorId（D6）

- **问题**：`recEngagementContext` 只含 duration/recScore/tags/feedRequestId，离线分析无法按来源/类型/作者切分。
- **修复**：在 `recEngagementContext` helper 中增加 `referralSource`、`contentType`、`authorId` 参数
- **文件**：`learning.go`
- **影响**：D6 +0.3

### B-14: CircleDetailPage 路由无 referralSource 透传（D2）

- **问题**：`app_router.dart` 中 `CircleDetailPage` 不读取 extra，页面默认 `organicFeed`。
- **修复**：定义 `CircleDetailPageRouteExtra`；路由 builder 读取传入
- **文件**：`app_router.dart`, `circle_detail_page.dart`, `home_circles_hub_page.dart`
- **影响**：D2 +0.2

## P2：健壮性 + 合规 + 可观测

### B-15: learning.Event.Context map[string]any → 带类型结构（D9）

- **问题**：推荐侧学习事件上下文为 `map[string]any`，violates DDD 强类型约束。
- **修复**：定义 `RecImpressionContext` / `RecEngagementContext` 结构体，通过 `json.Marshal` 转为 `json.RawMessage` 存入 Event
- **文件**：`learning.go`, `runtime/learning/learning.go`（Event 结构）
- **影响**：D9 +0.5

### B-16: CascadeScorer 超时未调用 RecordModelTimeout（D6）

- **问题**：`RecordModelTimeout` 函数存在但未在 `CascadeScorer` 超时分支中调用。
- **修复**：在 cascade 超时 fallback 路径中调用 `RecordModelTimeout`
- **文件**：`scorer.go`
- **影响**：D6 +0.2

### B-17: favorite/comment 缺 per-type Prometheus label（D6）

- **问题**：click/like/share 有 per-type 原子计数和 Prometheus，但 favorite/comment 无。
- **修复**：增加 `recordFavoriteByType` / `recordCommentByType` helper 及对应 Prometheus counter
- **文件**：`metrics.go`
- **影响**：D6 +0.2

### B-18: verify_feature_consistency.py 不覆盖 train_embedding.py（D7）

- **问题**：双塔嵌入训练路径未纳入特征维数 / CONTENT_TYPE_MAP 一致性校验。
- **修复**：`check_content_type_maps` 和 `check_feature_dimensions` 增加对 `train_embedding.py` 的扫描
- **文件**：`verify_feature_consistency.py`
- **影响**：D7 +0.3

### B-19: trackContentEnter 不传 tags/entityRefs（D1）

- **问题**：`PhotoDetailPage` / `VideoDetailPage` 的 `trackContentEnter` 未传 `tags`/`entityRefs`，语义兴趣传播缺失。
- **修复**：从帖子数据中提取 tags 和 entityRefs 传入 tracker
- **文件**：`photo_detail_page.dart`, `video_detail_page.dart`
- **影响**：D1 +0.3

### B-20: PostCreated 在 RecommendFeatureProjector 为 no-op（D3）

- **问题**：用户发帖事件不更新任何用户特征向量，错失创作者偏好信号。
- **修复**：`PostCreated` 处理中将帖子标签/类型写入用户 TagAffinities 和 typeImpressions（低权重）
- **文件**：`recommend_feature.go`
- **影响**：D3 +0.3

### B-21: 特性树缺 plan.yaml（D10）

- **问题**：`specs/feature-tree/runtime/runtime-recommendation/` 下无 `plan.yaml`。
- **修复**：基于当前修复项生成 `plan.yaml`
- **文件**：`specs/feature-tree/runtime/runtime-recommendation/plan.yaml`
- **影响**：D10 +0.3
