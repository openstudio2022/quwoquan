## MODIFIED Requirements

### 需求：关键概念与命名

以下关键概念须在应用中统一使用，与设计文档一致。与助手相关的代码实体命名必须使用实体或分类名称（如 `assistant`、`personal_assistant`），逻辑判断必须使用固定 ID 常量，不得依赖可变展示名字符串，以避免产品命名变更造成系统行为回归。

| 概念 | 定义 | 用途 |
|------|------|------|
| **微趣** | 快速记录当下，降低记录门槛（照片九宫格、纯文字、短视频） | 创作入口抽屉第一区域；与「作品」区分 |
| **作品** | 精心打磨的图片/视频/文章内容 | 创作入口抽屉第二区域；主页创作 Tab |
| **创作** | 用户思考、筛选或排版的高质量产出 | 主页/圈子主 Tab 之一 |
| **互动** | 赞、评论、互访足迹、社群交流 | 主页/圈子主 Tab 之一 |
| **生活** | 结构化的个人生活图谱，含足迹、书影音、味蕾、爱物 | 主页/圈子主 Tab 之一 |
| **爱物** | 个人收藏、数码装备、心水好物（取代原「好物」） | 生活 Tab 子分类 |
| **足迹** | 旅行、地理标记 | 生活 Tab 子分类 |
| **书影音** | 电影、书籍、音乐等精神消费记录 | 生活 Tab 子分类 |
| **味蕾** | 美食、咖啡、餐厅打卡 | 生活 Tab 子分类 |
| **圈子** | 基于兴趣的社交组织 | 圈子频道、圈子主页 |
| **个人私人助手** | AI 助手核心能力集合，支持 App 内与外部渠道统一调用 | 趣聊、我的、内容详情页工具栏中心入口与外部网关 |
| **分身/身份** | 用户可切换的多重身份（Persona） | 我的主页 |

#### 场景：概念命名一致性

- **当** UI 展示上述概念时
- **则** 使用设计文档确定的术语；严禁使用已弃用的瞬间、随记、好物

#### 场景：逻辑层命名不依赖展示名

- **当** 助手名称或文案发生变更
- **则** 逻辑层因使用固定 ID 与实体命名而无需改动核心流程

### 需求：小趣入口与沉浸式工具栏

沉浸式内容（图片/视频/文章）详情页工具栏须将**个人私人助手入口**居中放置（星火图标 ✨）；左侧为点赞/收藏计数，右侧为评论/分享。滚动时作者头像吸顶。助手半屏面板高度 55vh–60vh。该入口在手机、平板、PC 上须保持一致交互语义，并允许根据设备能力执行本地或远程能力路由。

#### 场景：助手入口可见

- **当** 用户浏览内容详情时
- **则** 工具栏中心显示个人私人助手入口；点击可唤起助手半屏对话

#### 场景：多设备能力路由

- **当** 用户在不同设备形态（手机、平板、PC）使用助手入口
- **则** 系统可根据设备能力约束选择本地工具执行或远程节点转发执行
