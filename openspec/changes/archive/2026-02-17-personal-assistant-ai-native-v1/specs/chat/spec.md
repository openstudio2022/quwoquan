## MODIFIED Requirements

### 需求：趣聊频道与原型 MessagePage 一致

「趣聊」频道须与 MessagePage 一致：一级 Tab 为**消息**与**通讯**（或 contacts）。消息 Tab 下二级 Tab 为全部、@我、未读、密信等；通讯 Tab 下二级 Tab 为全部、圈子、好友、群聊等。系统须提供个人私人助手专属会话入口，并允许该会话由 `personal_assistant` 引擎驱动（而非固定 mock 回复）。

#### 场景：一级与二级 Tab

- **当** 用户处于「趣聊」频道
- **则** 一级 Tab 为消息/通讯；切换后二级 Tab 与列表内容随之更新（会话列表或联系人列表）

#### 场景：点击会话打开聊天详情

- **当** 用户在消息列表中点击某条会话
- **则** 打开该会话的聊天详情页，展示消息线程与底部输入栏

#### 场景：进入个人私人助手专属会话

- **当** 用户进入个人私人助手会话并发送消息
- **则** 会话通过 `personal_assistant` 引擎执行并返回真实结果

### 需求：聊天详情展示消息与输入区

聊天详情须展示消息列表（己方/对方气泡）、自动滚动到最新消息，以及底部输入工具栏。消息类型须包含文本、图片等，与 CHAT_FEATURES 及 Figma 一致。对于个人私人助手会话，系统须支持展示工具/技能 trace 摘要，帮助用户理解执行过程。

#### 场景：消息与输入区可见

- **当** 用户处于聊天详情页
- **则** 消息按正确对齐与样式展示，底部输入工具栏可见

#### 场景：展示工具或技能执行轨迹

- **当** 个人私人助手在回复中调用了工具或技能
- **则** 聊天详情可展示简要执行轨迹信息（如开始、结果、错误）
