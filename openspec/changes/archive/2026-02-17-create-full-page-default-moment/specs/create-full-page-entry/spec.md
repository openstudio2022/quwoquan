# create-full-page-entry

## ADDED Requirements

### Requirement: 创作入口直接进入全页且默认发微趣

系统 SHALL 在用户点击底部导航「创作」时直接导航到创作全页（如 `/create`），不先展示入口抽屉。创作页打开时 MUST 默认选中「发微趣」Tab，标题为「发微趣」。

#### Scenario: 点击创作进入全页

- **WHEN** 用户点击底部导航「创作」
- **THEN** 应用导航至创作全页（全屏路由），不弹出抽屉或弹窗

#### Scenario: 默认发微趣

- **WHEN** 创作全页首次展示且未带类型参数
- **THEN** 当前 Tab 为微趣（发微趣），AppBar 标题为「发微趣」

### Requirement: 创作页为全页形态且布局与图一一致

创作页 MUST 为全页（非弹窗、非底部抽屉）。顶部 AppBar 须包含：左侧关闭按钮（X）、中间标题（随当前 Tab 为「发微趣」/「发美图」/「发视频」/「写文章」）、右侧为草稿箱入口或发表按钮。底部须为四 Tab 栏：微趣、美图、视频、文章。中部为可滚动编辑区域。

#### Scenario: 全页布局与顶部栏

- **WHEN** 用户处于创作全页
- **THEN** 页面占满屏幕；顶部可见关闭按钮、居中标题、右侧草稿箱（无内容时）或发表（有内容时）

#### Scenario: 底部四 Tab

- **WHEN** 用户处于创作全页
- **THEN** 底部 Tab 栏展示四个 Tab：微趣、美图、视频、文章；点击可切换编辑类型且标题随之更新

### Requirement: 发微趣编辑能力完整

发微趣 Tab 须提供：文本输入占位符「这一刻的想法...」、媒体添加区（如 + 占位，支持添加图片）、所在位置、提醒谁看、谁可以看（默认公开）。图片添加后 SHALL 支持进入图片编辑（如 `/create/edit-image`），编辑结果回写当前内容。

#### Scenario: 发微趣占位与选项

- **WHEN** 用户选中发微趣 Tab
- **THEN** 可见「这一刻的想法...」占位、媒体添加区、所在位置/提醒谁看/谁可以看入口，谁可以看默认展示「公开」

#### Scenario: 发微趣图片编辑

- **WHEN** 用户在发微趣中添加图片并触发编辑
- **THEN** 可进入图片编辑页，编辑完成后返回并更新当前微趣内容

### Requirement: 美图 / 视频 / 文章编辑能力与操作支持

美图、视频、文章三个 Tab SHALL 分别提供对应编辑区域（标题、描述、媒体/封面等），并支持草稿、退出确认、定时自动保存（如 10 秒）、图片编辑（美图/文章）等操作，与现有 CreatePage 及迁移任务 CR2/CR3 行为一致。

#### Scenario: 各 Tab 编辑与保存

- **WHEN** 用户在美图/视频/文章 Tab 编辑内容
- **THEN** 支持草稿保存、有内容时关闭提示确认、自动保存与图片编辑等与原型一致的操作

### Requirement: 创作页使用语义 token

创作页内所有颜色、间距、文案 SHALL 使用 AppColors、AppSpacing、UITextConstants。禁止硬编码。

#### Scenario: 创作页 token

- **WHEN** 创作页被构建
- **THEN** 样式与文案仅来自设计系统常量
