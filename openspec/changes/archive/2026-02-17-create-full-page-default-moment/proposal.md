# 创作全页入口与默认发微趣

## Why

当前点击底部导航「创作」会先弹出入口抽屉（CreateEntrySheet），用户需在六入口中再选一次才能进入创作页，与原型图一不一致。图一要求创作入口直接进入全页创作界面，默认即为「发微趣」，底部四 Tab（微趣 / 美图 / 视频 / 文章）统一承载四种创作类型，且所有操作（如图片编辑、所在位置、提醒谁看、谁可以看）均需支持并与原型 1:1 一致。本变更将创作流程改为「点击创作 → 全页创作页、默认发微趣」，并逐一实现四类创作的完整能力与迁移任务对齐。

## What Changes

- **创作入口**：点击「创作」不再打开入口抽屉，改为直接导航到创作全页（如 `/create`），不再使用 `/create-entry` 作为首步。
- **默认 Tab**：创作页打开时默认选中「发微趣」（微趣 Tab），标题为「发微趣」。
- **页面形态**：创作页为全页（非弹窗、非底部抽屉），顶部 AppBar（左侧关闭 X、中间标题「发微趣」/「发美图」等、右侧草稿箱/发表）、中部可滚动编辑区、底部四 Tab 栏（微趣、美图、视频、文章），与图一布局一致。
- **发微趣**：占位符「这一刻的想法...」、媒体添加区（+ 添加图片等）、所在位置、提醒谁看、谁可以看（默认公开）；支持图片选择与图片编辑等操作。
- **美图 / 视频 / 文章**：逐一实现各 Tab 的编辑能力，包含图片编辑、草稿、退出确认、10 秒自动保存等，与现有 CreatePage 及迁移任务 CR2/CR3 对齐。
- **迁移任务同步**：更新 figma-prototype-full-migration 的 tasks.md（或等价迁移文档），将「创作入口为全页、默认发微趣、四 Tab 与全部操作」明确为 1:1 复制原型的创作实现，确保与 CreatePage.tsx / CreateEntrySheet 等原型行为一致。

## Capabilities

### New Capabilities

- `create-full-page-entry`: 创作入口直接进入全页、默认发微趣；创作页为全页形态，顶部栏（关闭/标题/草稿箱）、底部四 Tab（微趣、美图、视频、文章），发微趣/美图/视频/文章四类编辑与全部操作（图片编辑、位置、提醒、可见范围等）1:1 与原型一致。

### Modified Capabilities

- `main-nav`: 底部导航「创作」的点击目标由「打开入口抽屉」改为「直接进入创作全页」（如 `/create`），不再依赖 `/create-entry` 作为唯一入口。

## Impact

- **路由**：`/create-entry` 可保留但不再作为创作主入口；底部导航「创作」改为 `context.go('/create')` 或等价全页路由。
- **CreatePage**：保留并强化为全页创作容器；默认 `initialType` 或默认 Tab 为微趣（发微趣）；标题、草稿箱、四 Tab 文案（微趣、美图、视频、文章）与图一一致。
- **CreateEntrySheet**：可保留供其他场景（如从发现页某入口）复用，或后续移除；不作为底部「创作」点击的默认路径。
- **迁移文档**：figma-prototype-full-migration 的 tasks.md 中创作相关任务（CR1/CR2/CR3）需补充或调整为：入口全页、默认发微趣、四 Tab 及图片编辑等操作 1:1 实现。
