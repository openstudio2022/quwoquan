# create-flow

## 新增需求

### 需求：创作入口抽屉包含两个区域与六个入口

创作入口抽屉须包含两个区域，与原型代码 CreateEntrySheet 一致：**微趣**（三个入口：照片、文字、视频）与**作品**（三个入口：图片、文章、视频）。区域标题为「微趣·记录生活此刻感悟」与「作品·精心创作优质内容」。布局与文案须与 Figma 及当前原型一致。

#### 场景：抽屉展示六个入口

- **当** 创作入口抽屉打开时
- **则** 用户可见「微趣」下的照片/文字/视频与「作品」下的图片/文章/视频，且区域标题与入口副标题（如相册选择、纯文字、短视频、大图展示、长文图文、精美视频）正确

#### 场景：选择入口后打开对应编辑器

- **当** 用户点击六个入口之一
- **则** 抽屉关闭，并以全屏形式展示创作页（CreatePage）对应 Tab 的编辑器：微趣对应随记类（moment/照片/文字/视频），作品对应图片/文章/视频类

### 需求：创作页为四 Tab 统一编辑容器

创作页（CreatePage）须提供四个 Tab：随记（moment）、图片（photo）、视频（video）、文章（article）。从入口抽屉选择某一类型后，创作页以对应 initialTab 打开。草稿、退出确认、保存与丢弃行为须与原型一致。

#### 场景：创作页 Tab 与内容

- **当** 用户从入口抽屉选择任一类进入创作页
- **则** 创作页展示对应 Tab 的编辑区域（如 MomentEditorCard、ImageEditor、VideoEditorCard、ArticleEditorCard）；用户可切换 Tab 编辑不同类型

### 需求：入口抽屉高度与行为符合设计

抽屉须符合规定的最大高度（如 67vh），支持点击遮罩或关闭按钮关闭，并使用规定的动效（如 300ms ease-out）。顶部须有指示条与关闭按钮。

#### 场景：抽屉正确关闭

- **当** 用户点击遮罩或关闭按钮
- **则** 抽屉按规定的动效关闭

### 需求：已废弃概念

设计文档 v3 已废弃瞬间、随记、好物标签；创作入口统一为微趣（快速记录）与作品（精心创作）两类。创作页内不再使用瞬间/随记作为独立 Tab 名称。

#### 场景：术语一致性

- **当** 创作流程 UI 展示时
- **则** 使用微趣/作品，严禁使用已弃用的瞬间、随记、好物

### 需求：创作流程使用语义 token

创作入口抽屉与创作页内所有 UI 须使用 AppColors、AppSpacing、UITextConstants。禁止使用硬编码值。

#### 场景：创作流程中的 token

- **当** 入口抽屉与创作页被构建时
- **则** 所有样式与文案使用设计系统常量
