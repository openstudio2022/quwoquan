# app-global

## 新增需求

### 需求：产品整体定位

趣我圈 App 须遵循以下产品定位（参考 InformationArchitecture v3.1、CONTENT_SPECIFICATION v3.0、DESIGN_SPECIFICATION v3.1、CIRCLE_DESIGN v2.0、PROFILE_PAGE_DESIGN v3.0 等设计文档）：

- **核心理念**（来源：InformationArchitecture v3.1）：**「以兴趣为半径，画出我们的交集。」** 趣我圈不仅仅是一个社交应用，它是一个基于「兴趣锚点」与「智能助手」的共生系统。用户通过创作内容、记录生活来沉淀个人资产，通过「小趣」AI 助手实现上下文感知的高效互动。
- **小趣定位**：小趣图标本意为「星火 (Spark)」，象征灵感的瞬间闪现与兴趣的高光点亮。Slogan：「让兴趣闪亮 (Make Interests Shine)」。拟人化名称为「智多星」。交互形态：悬浮球、工具栏中心入口、半屏对话面板。
- **内容体系**（来源：CONTENT_SPECIFICATION v3.0）：采用「创作-互动-生活」三位一体逻辑，通过小趣进行上下文串联，平衡「深度内容沉淀」与「高频社交反馈」。创作：精心编排的图片/视频/文章；互动：赞、评论、访问足迹、社群交流；生活：足迹、书影音、味蕾、爱物。
- **圈子定位**（来源：CIRCLE_DESIGN v2.0）：子频道含创作、讨论、生活、聊天。生活展示圈子成员共同标记的兴趣点与爱物。
- **已弃用概念**（来源：设计文档）：❌ 瞬间 (Moments) → 已由微趣或直接归类至创作；❌ 随记 (Notes) → 已合并至文章或生活记录；❌ 好物 (Goods) → 太商品化，已整合进「爱物」或通过 AI 自动识别。

#### 场景：产品定位一致性

- **当** 设计与实现功能时
- **则** 功能与交互符合上述定位，不偏离核心价值；严禁使用已弃用概念

### 需求：关键概念与命名

以下关键概念须在应用中统一使用，与设计文档一致：

| 概念 | 定义 | 用途 |
|------|------|------|
| **微趣** | 快速记录当下，降低记录门槛（照片九宫格、纯文字、短视频） | 创作入口抽屉第一区域；与「作品」区分 |
| **作品** | 精心打磨的图片/视频/文章内容 | 创作入口抽屉第二区域；主页创作 Tab |
| **创作** | 用户思考、筛选或排版的高质量产出 | 主页/圈子主 Tab 之一 |
| **互动** | 赞、评论、互访足迹、社群交流 | 主页/圈子主 Tab 之一 |
| **生活** | 结构化的个人生活图谱，含足迹、书影音、味蕾、爱物 | 主页/圈子主 Tab 之一 |
| **爱物** | 个人收藏、数码装备、心水好物（取代原「好物」） | 生活 Tab 子分类 |
| **足迹** | 旅行、地理标记 | 生活 Tab 子分类 |
| **书影音** | 电影、书籍、音乐等精神消费记录 | 生活 Tab 子分类 |
| **味蕾** | 美食、咖啡、餐厅打卡 | 生活 Tab 子分类 |
| **圈子** | 基于兴趣的社交组织 | 圈子频道、圈子主页 |
| **小趣/XiaoQu** | AI 智能助手，智多星，星火图标 | 趣聊、我的、内容详情页工具栏中心入口 |
| **分身/身份** | 用户可切换的多重身份（Persona） | 我的主页 |

#### 场景：概念命名一致性

- **当** UI 展示上述概念时
- **则** 使用设计文档确定的术语；严禁使用已弃用的瞬间、随记、好物

### 需求：五大频道与信息架构

应用须提供五大频道（来源：InformationArchitecture v3.1）：**发现**（兴趣内容分发中心，推荐 + 图片/视频/文章）、**圈子**（兴趣社群，二级圈子与多分身）、**创作**（统一发布入口，微趣与作品）、**趣聊**（即时通讯，含小趣专属对话）、**我的**（个人资产与分身管理）。

#### 场景：频道与入口

- **当** 用户处于主应用框架
- **则** 底部导航展示五个频道；创作点击打开入口抽屉；发现与圈子按设计组织

### 需求：主页统一 3-Tab 架构

作者主页、圈子主页、我的主页须采用统一的 **3-Tab 架构**（来源：PROFILE_PAGE_DESIGN v3.0、CONTENT_SPECIFICATION）：**创作**（全部、图片、视频、文章；弃用瞬间、随记）、**互动**（收到的赞、发出的评论、互访）、**生活**（足迹、书影音、味蕾、爱物；弃用好物橱窗）。使用胶囊样式二级 Tab，支持响应式横滚。

#### 场景：主页结构一致性

- **当** 用户查看作者、圈子或我的主页
- **则** 主 Tab 为创作/互动/生活；创作子分类为全部/图片/视频/文章；生活子分类为足迹/书影音/味蕾/爱物

### 需求：小趣入口与沉浸式工具栏

沉浸式内容（图片/视频/文章）详情页工具栏须将**小趣入口**居中放置（星火图标 ✨）；左侧为点赞/收藏计数，右侧为评论/分享。滚动时作者头像吸顶。小趣半屏面板高度 55vh–60vh。

#### 场景：小趣入口可见

- **当** 用户浏览内容详情时
- **则** 工具栏中心显示小趣入口；点击可唤起小趣半屏对话

### 需求：头像设计系统（来源：AVATAR_DESIGN_SYSTEM.md）

头像须遵循以下形状与尺寸语义，确保个人与圈子的视觉区分：

- **个人/作者/AI 头像 = 圆形**：完美圆形（border-radius: 50%），代表个体、独立性。小趣头像与用户头像尺寸保持 1:1，统一圆形。
- **圈子/群组头像 = 圆角正方形**：圆角正方形（border-radius: 20%），代表组织、容器。参考 Discord、Slack、小红书等。

**尺寸语义**（响应式：移动端/平板/桌面）：

| 语义 | 个人头像 | 圈子头像 | 使用场景 |
|------|----------|----------|----------|
| xs | 24/28/32px | 24/28/32px | 评论列表、小型引用、小型圈子标签 |
| sm | 32/36/40px | 32/36/40px | 主要列表项、消息列表、圈子列表 |
| md | 40/44/48px | 40/44/48px | 帖子作者、卡片头部、圈子卡片 |
| lg | 56/64/72px | 56/64/72px | 用户资料预览、圈子详情页 |
| xl | 72/80/88px | 72/80/88px | 个人资料页顶部、圈子主页头部 |

**扩展语义**：avatar-container、avatar-badge（认证徽章）、avatar-status-indicator（在线状态）、avatar-container-gradient（活跃圈子渐变边框）。若扩展则须在 AppSpacing 或 DesignSemanticConstants 中补充 avatar-user-*、avatar-circle-* 等常量。

#### 场景：头像形状区分

- **当** 展示作者、用户、小趣头像时
- **则** 使用圆形；展示圈子、群组头像时使用圆角正方形

#### 场景：头像尺寸语义

- **当** 构建帖子卡片、消息列表、主页等
- **则** 使用对应场景的 avatar-user-* 或 avatar-circle-* 语义尺寸，禁止硬编码 px

### 需求：语义 token 与编码规范

全应用须遵守 quwoquan_app 编码规范：禁止硬编码；所有颜色、间距、文案、类型须使用 AppColors、AppSpacing、UITextConstants、ContentTypeConstants、DesignSemanticConstants；导入使用包引用 `package:quwoquan_app/...`。若扩展语义则须在对应常量中补充。头像须使用头像设计系统语义。

#### 场景：规范一致性

- **当** 实现或修改功能时
- **则** 符合上述规范；flutter analyze 无违规
