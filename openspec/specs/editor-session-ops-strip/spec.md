# editor-session-ops-strip

## Purpose

图片编辑器中可复用的系统级会话操作条，位于图片内容区底部，承载会话内撤回/重做与原图对比能力；与工具面板底部 X/✓ 分层清晰，多工具共用同一组件与视觉规范。

## ADDED Requirements

### Requirement: 会话操作条基础能力

编辑器 SHALL 提供可复用的会话操作条组件，位于图片内容区底部，至少包含 `undo/redo` 与 `compare` 两类操作入口。组件 MUST 支持按工具能力开关启用或隐藏对应操作，不得强制所有工具显示全部按钮。

#### Scenario: 工具按能力启用操作

- **WHEN** HSL 或局部工具会话声明支持 `undo/redo + compare`
- **THEN** 会话操作条同时显示撤回、重做与对比按钮

#### Scenario: 工具禁用某项能力

- **WHEN** 某工具会话仅声明支持 `compare`
- **THEN** 会话操作条仅显示对比按钮，不显示撤回和重做

### Requirement: 会话级撤回与重做

会话操作条中的撤回与重做 SHALL 仅作用于当前工具会话内记录栈，不得直接修改全局编辑记录。点击 `X` 时系统 MUST 回滚到进入当前工具会话的快照；点击 `✓` 时系统 MUST 仅提交会话最终状态为全局记录单步记录。

#### Scenario: 会话内撤回

- **WHEN** 用户在局部或 HSL 会话中连续调节后点击撤回
- **THEN** 仅回退当前工具会话内最近一步，不影响其他工具既有结果

#### Scenario: 会话提交为单步

- **WHEN** 用户在工具会话中多次调节后点击 `✓`
- **THEN** 全局记录新增一条当前工具结果记录，而非逐次调节记录

### Requirement: 对比基线语义

会话操作条的对比基线 MUST 由工具会话在进入时提供，并在会话生命周期内保持稳定。对比 SHALL 以「进入当前工具会话时的图像状态」作为基线；按下或切换对比时，系统 MUST 临时旁路当前工具会话效果，并在释放或再次切换后恢复。对比行为 SHALL 只影响预览层显示，不得直接改写工具参数。

#### Scenario: 对比显示会话基线

- **WHEN** 用户在局部或 HSL 会话中点击或按住对比按钮
- **THEN** 图片显示进入当前工具会话时的基线状态，不显示当前临时调节

#### Scenario: 对比结束恢复当前效果

- **WHEN** 用户释放按压或关闭对比
- **THEN** 图片恢复到当前工具会话的实时调节效果

#### Scenario: 对比不改参数

- **WHEN** 用户多次触发对比操作
- **THEN** 工具参数与会话操作栈保持不变

### Requirement: 布局与视觉一致性

会话操作条 SHALL 固定在图片内容区底部，并与工具面板底部 `X / ✓` 分层清晰；语义高度与底栏一致，保证对比图标与勾勾垂直对齐。图标尺寸、触达热区、颜色语义 MUST 使用共享设计 token，保证多工具视觉一致。安全区适配 MUST 覆盖不同机型底部 inset。按钮状态（可用/禁用）必须提供可辨识的亮度层级反馈。

#### Scenario: 布局分层不冲突

- **WHEN** 工具面板同时展示 `X / ✓` 与系统级会话操作条
- **THEN** 两者不重叠、不遮挡图片核心区域，且在不同机型安全区内布局稳定

#### Scenario: 安全区适配

- **WHEN** 设备存在底部安全区（如全面屏）
- **THEN** 会话操作条与 `X / ✓` 操作栏不重叠，且按钮可完整点击

#### Scenario: 禁用态可辨识

- **WHEN** 会话内无可撤回步骤时
- **THEN** 撤回按钮显示禁用态并不可触发，同时视觉上明显弱化
