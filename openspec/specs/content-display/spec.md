# content-display

## 新增需求

### 需求：帖子卡片与各内容类型布局

帖子卡片须采用与 Figma 一致的布局与样式（缩略图、标题、作者、统计、类型标识）。卡片须可在发现与圈子 feed 中复用。**作者头像**使用圆形（avatar-user-md，帖子作者场景）；**圈子标识**使用圆角正方形（avatar-circle-sm 或 md）。内容类型为图片、视频、文章等；已弃用瞬间、随记、好物作为独立展示概念。

#### 场景：按类型展示卡片布局

- **当** 某条帖子在 feed 中渲染时
- **则** 该帖子类型的卡片布局与 Figma 一致

#### 场景：Feed 布局

- **当** 展示多条帖子时
- **则** feed 使用规定的布局（瀑布流/网格）与间距

### 需求：沉浸式媒体查看器含小趣入口

当用户打开图片或视频帖子时，应用须展示全屏沉浸式查看器。工具栏须将**小趣入口居中放置**（星火图标）；左侧点赞/收藏计数，右侧评论/分享。支持滑动/关闭。与 DESIGN_SPECIFICATION v3.1 及 InformationArchitecture v3.1 一致。

#### 场景：查看器与小趣入口

- **当** 用户点击图片或视频帖子
- **则** 沉浸式查看器打开，工具栏中心显示小趣入口；点击可唤起小趣半屏面板

#### 场景：查看器关闭

- **当** 用户关闭查看器
- **则** 查看器关闭，用户回到前一屏

### 需求：文章详情含小趣入口

当用户打开文章帖子时，应用须展示全屏文章详情。滚动时作者信息吸顶；工具栏含小趣入口。与设计文档一致。

#### 场景：文章详情与小趣

- **当** 用户点击文章帖子
- **则** 打开文章详情页，工具栏含小趣入口

### 需求：内容展示使用语义 token

所有帖子卡片、沉浸式查看器与文章详情须使用 AppColors、AppSpacing、UITextConstants、ContentTypeConstants。禁止使用硬编码值。

#### 场景：内容展示的 token

- **当** 内容展示相关组件被构建时
- **则** 所有样式与文案使用设计系统常量
